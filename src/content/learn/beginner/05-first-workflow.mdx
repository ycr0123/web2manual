## 실전 워크플로우

이 레슨에서는 실제 개발 상황에서 Claude Code를 활용하는 세 가지 핵심 워크플로우를 살펴봅니다.

## 워크플로우 1: 새 기능 구현

### 예시 시나리오
사용자 프로필 업데이트 API 엔드포인트를 추가해야 합니다.

### 단계별 접근

**1단계: 요구사항 설명**

<CLIExample
  command='claude "사용자 프로필을 업데이트하는 REST API 엔드포인트를 추가해줘. PUT /api/users/:id를 사용하고, 이름, 이메일, 아바타 URL을 수정할 수 있어야 해. 입력 검증도 포함해줘."'
  description="Claude Code에게 명확한 요구사항을 전달합니다."
/>

**2단계: Claude Code의 분석**

Claude Code는 다음 작업을 수행합니다:
1. 기존 API 구조 파악 (라우터, 컨트롤러, 모델)
2. 일관된 스타일로 코드 작성
3. 유효성 검사 로직 추가
4. 테스트 코드 작성

**3단계: 변경사항 검토 및 승인**

Claude Code가 변경할 파일들과 내용을 보여주면 검토합니다.

**4단계: 테스트 실행**

  command='claude "방금 추가한 엔드포인트의 테스트를 실행해줘"'
  description="테스트를 실행하여 구현이 올바른지 확인합니다."

## 워크플로우 2: 버그 수정

### 효과적인 버그 리포팅

버그를 수정하려면 Claude Code에게 다음 정보를 제공하세요:

1. **오류 메시지**: 정확한 오류 메시지나 스택 트레이스
2. **재현 방법**: 버그를 재현하는 단계
3. **기대 동작**: 어떻게 동작해야 하는지
4. **실제 동작**: 현재 어떻게 동작하는지

### 실습 예시

```
사용자 로그인 시 다음 오류가 발생합니다:

오류: TypeError: Cannot read property 'email' of undefined
위치: auth/middleware.js:42

재현 방법:
1. POST /api/auth/login 요청
2. 올바른 자격증명으로 로그인
3. 때때로 이 오류가 발생함

기대 동작: 정상적으로 JWT 토큰을 반환해야 함
```

<CLIExample
  command='claude "auth/middleware.js의 42번째 줄에서 Cannot read property email of undefined 오류가 발생합니다. 로그인 시 간헐적으로 발생하는데, 원인을 찾아 수정해줘."'
  description="버그 정보와 함께 수정 요청을 합니다."
/>

### 버그 수정 체크리스트

- [ ] 버그 재현 테스트 케이스 작성
- [ ] 근본 원인 파악
- [ ] 수정 적용
- [ ] 수정 후 테스트 통과 확인
- [ ] 유사한 패턴의 다른 버그 존재 여부 확인

### 코드 품질 개선

  description="특정 파일에 대한 코드 리뷰를 요청합니다."

Claude Code는 다음 항목들을 검토합니다:

| 검토 항목 | 설명 |
|-----------|------|
| 성능 | 불필요한 리렌더링, 메모화 기회 |
| 가독성 | 변수명, 함수명, 주석 |
| 보안 | XSS 취약점, 입력 검증 |
| 타입 안전성 | TypeScript 타입 오류 |
| 테스트 가능성 | 의존성 주입, 순수 함수 |

</Callout>

## 워크플로우 팁

### 효과적인 프롬프트 작성

**구체적으로 작성하기:**

나쁜 예: "코드 고쳐줘"

좋은 예: "users.ts의 getUserById 함수가 사용자가 없을 때 null을 반환하는 대신 NotFoundException을 던지도록 수정해줘"

**컨텍스트 제공하기:**

```
우리 프로젝트는 Express + TypeScript를 사용하고, 오류 처리는
HttpException 클래스를 통해 이루어집니다.
```

**단계적으로 요청하기:**

한 번에 너무 많은 것을 요청하지 말고, 단계적으로 작업을 진행하세요.

<Collapsible title="유용한 Claude Code 명령어 모음">
인터랙티브 모드에서 사용할 수 있는 유용한 내장 명령어들:

- `/help` - 사용 가능한 명령어 목록 표시
- `/clear` - 컨텍스트 초기화
- `/exit` - Claude Code 종료
- `/model` - 사용할 모델 변경
- `/status` - 현재 세션 상태 확인

이 명령어들은 `/` 슬래시로 시작하며 Claude Code의 동작을 제어합니다.
</Collapsible>
