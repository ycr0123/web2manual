## 인터랙티브 모드 이해

Claude Code는 두 가지 실행 모드를 제공합니다: **인터랙티브 모드**와 **비인터랙티브 모드**.

### 인터랙티브 모드

<CLIExample
  command="claude"
  description="아무 인자 없이 claude를 실행하면 인터랙티브 모드가 시작됩니다."
/>

인터랙티브 모드의 특징:
- REPL(Read-Eval-Print Loop) 방식으로 동작
- 이전 대화 컨텍스트 유지
- 연속된 작업 수행 가능
- 실시간 피드백 및 수정 가능

### 비인터랙티브 모드

<CLIExample
  command='claude -p "이 프로젝트의 테스트를 모두 실행하고 결과를 요약해줘"'
  description="-p 또는 --print 플래그로 단일 명령을 실행하고 종료합니다."
/>

비인터랙티브 모드의 특징:
- CI/CD 파이프라인에 적합
- 스크립트에서 자동화
- 단일 작업 완료 후 자동 종료
- 결과를 표준 출력으로 반환

## 효과적인 인터랙티브 세션 운영

### 세션 시작 시 컨텍스트 설정

인터랙티브 세션을 시작할 때, 첫 번째 메시지에서 충분한 컨텍스트를 제공하세요:

```
> 오늘 사용자 인증 모듈을 리팩토링할 예정입니다.
현재 코드는 JWT를 사용하고, Express 프레임워크를 사용합니다.
TypeScript로 작성되어 있으며, Jest로 테스트합니다.
시작 전에 현재 auth 디렉토리 구조를 파악해줘.
```

### 연속 작업 흐름

인터랙티브 모드의 큰 장점은 이전 작업의 맥락을 이어받는 것입니다:

```
> auth/middleware.ts를 읽어줘

# Claude Code가 파일 내용을 읽고 설명

> 좋아, 이 파일에서 토큰 검증 로직을 분리해서
  별도의 tokenValidator.ts 파일로 만들어줘

# Claude Code가 이전 파일 내용을 기반으로 분리 작업 수행

> 이제 tokenValidator.ts에 대한 단위 테스트를 작성해줘

# Claude Code가 방금 만든 파일의 내용을 알고 테스트 작성
```

### 멀티라인 입력

복잡한 요청은 여러 줄로 작성할 수 있습니다. 줄 끝에 `\`를 입력하거나, 따옴표 안에서 Enter를 누르세요:

```
> 다음 요구사항으로 새 API 엔드포인트를 만들어줘:
  - POST /api/orders
  - 요청 본문: { items: [{ productId, quantity }], address }
  - 재고 확인 후 주문 생성
  - 결제 서비스 호출
  - 주문 확인 이메일 발송
```

## 유용한 인터랙티브 명령어

### 내장 슬래시 명령어

| 명령어 | 기능 |
|--------|------|
| `/help` | 사용 가능한 명령어 목록 |
| `/clear` | 대화 히스토리 초기화 |
| `/exit` | 세션 종료 |
| `/model` | AI 모델 변경 |
| `/status` | 현재 세션 상태 |
| `/history` | 명령어 히스토리 |
| `/compact` | 컨텍스트 압축 |

<Callout type="tip">
`/compact` 명령어는 대화 히스토리를 압축하여 컨텍스트 공간을 절약합니다. 긴 세션에서 토큰 한도에 도달하기 전에 사용하세요.
</Callout>

### 컨텍스트 유지 전략

긴 작업 세션에서 컨텍스트를 효율적으로 관리하는 방법:

**1. 중간 요약 요청**
```
> 지금까지 한 작업을 간략히 요약해줘.
  다음 단계로 넘어가기 전에 현재 상태를 정리하고 싶어.
```

**2. 단계별 커밋**
작업 중 정기적으로 Git 커밋을 생성하세요. Claude Code에게 커밋을 요청할 수 있습니다:
```
> 지금까지 변경사항을 Git에 커밋해줘.
  커밋 메시지는 컨벤셔널 커밋 형식으로 작성해줘.
```

**3. 작업 체크포인트 설정**
```
> 현재까지 완료된 작업 목록을 정리해줘.
  아직 남은 작업도 함께 나열해줘.
```

## 실전 예시: 풀스택 기능 구현

인터랙티브 모드를 사용한 실제 기능 구현 흐름을 살펴봅니다.

### 시나리오: 댓글 기능 추가

```
1단계 - 기존 코드 파악:
> 현재 프로젝트에서 게시글(posts) 관련 코드 구조를 파악해줘

2단계 - 백엔드 구현:
> 게시글에 댓글을 추가할 수 있는 API를 구현해줘.
  Comment 모델, CRUD API, 그리고 기본 테스트 포함.

3단계 - 검토 및 수정:
> 댓글 삭제 시 본인 것만 삭제할 수 있도록 권한 검사를 추가해줘

4단계 - 프론트엔드:
> React 컴포넌트로 댓글 목록과 댓글 작성 폼을 구현해줘.
  기존 UI 스타일에 맞게 작성해줘.

5단계 - 통합 테스트:
> E2E 테스트를 작성해줘. 댓글 작성, 조회, 삭제 시나리오 포함.
```

<Callout type="note">
이처럼 단계별로 나누어 작업하면 각 단계를 검토하고 수정할 수 있어, 최종 결과물의 품질이 높아집니다. Claude Code는 이전 단계의 코드를 알고 있으므로 일관된 스타일로 구현합니다.
</Callout>

## 세션 관리 베스트 프랙티스

### 세션 시작 전 준비

- [ ] 프로젝트 루트 디렉토리로 이동
- [ ] CLAUDE.md 파일 최신화
- [ ] Git 상태 확인 (`git status`)
- [ ] 최근 변경사항 커밋 여부 확인

### 세션 중 관리

- [ ] 주요 작업 완료 시 Git 커밋
- [ ] 컨텍스트가 길어지면 `/compact` 또는 `/clear` 사용
- [ ] 복잡한 요청은 단계별로 분리

### 세션 종료 전 확인

- [ ] 모든 테스트 통과 확인
- [ ] 변경사항 Git 커밋
- [ ] 다음 세션을 위한 메모 작성 (CLAUDE.md 업데이트)

<Collapsible title="비인터랙티브 모드 활용 예시">
CI/CD 파이프라인이나 자동화 스크립트에서 Claude Code를 활용하는 예시:

```bash
#!/bin/bash
# 코드 품질 리뷰 자동화

# 변경된 파일 목록 가져오기
CHANGED_FILES=$(git diff --name-only HEAD~1)

# 각 파일에 대해 Claude Code로 리뷰
for file in $CHANGED_FILES; do
  if [[ $file == *.ts ]] || [[ $file == *.tsx ]]; then
    claude -p "다음 파일을 코드 리뷰해줘. 보안 이슈, 버그, 개선점을 JSON으로 반환해: $file"
  fi
done
```

이처럼 `-p` 플래그를 사용하면 Claude Code를 스크립트에서 도구로 활용할 수 있습니다.
</Collapsible>
