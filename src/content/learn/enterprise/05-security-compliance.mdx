## 보안 모델 개요

Claude Code는 엔터프라이즈 환경의 엄격한 보안 요구사항을 충족하도록 설계되었습니다. 이 레슨에서는 Claude Code의 보안 아키텍처, 데이터 처리 방식, 그리고 주요 컴플라이언스 프레임워크와의 정합성을 다룹니다.

### 보안 아키텍처

Claude Code의 보안은 다층적인 접근 방식을 따릅니다:

| 계층 | 보안 메커니즘 | 설명 |
|-----|-------------|------|
| 전송 계층 | TLS 1.3 | 모든 통신 암호화 |
| 인증 계층 | API Key / OAuth / SSO | 다중 인증 방식 지원 |
| 권한 계층 | 최소 권한 원칙 | 필요한 권한만 부여 |
| 실행 계층 | 샌드박스 | 파일 시스템 접근 제한 |
| 데이터 계층 | ZDR(Zero Data Retention) | 데이터 비보관 정책 |

<Callout type="note">
Claude Code는 사용자가 명시적으로 승인한 작업만 수행합니다. 파일 읽기, 쓰기, 명령 실행 등 모든 작업은 사용자의 확인을 거칩니다. 이 Permission System은 Claude Code의 핵심 보안 메커니즘입니다.
</Callout>

### 권한 제어 시스템

Claude Code의 권한 시스템은 세밀한 제어를 지원합니다:

```json
// .claude/settings.json - 권한 설정 예시
{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Bash(npm test)",
      "Bash(npm run lint)",
      "Bash(git diff)",
      "Bash(git log)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(curl *)",
      "Bash(wget *)",
      "Bash(sudo *)"
    ]
  }
}
```

## 데이터 처리 방침

### 데이터 흐름

Claude Code를 사용할 때 데이터가 어떻게 처리되는지 이해하는 것이 중요합니다:

1. **입력 데이터**: 사용자 프롬프트, 코드 파일 내용, 실행 결과
2. **전송**: TLS 1.3으로 암호화된 HTTPS 연결
3. **처리**: Anthropic 서버(또는 클라우드 제공자)에서 추론 실행
4. **응답**: 생성된 결과를 암호화하여 반환
5. **보관**: API 사용 시 데이터를 모델 학습에 사용하지 않음

```bash
# 데이터 전송 확인 (디버깅용)
# Claude Code가 전송하는 데이터를 로컬에서 확인
claude --verbose "hello" 2>&1 | grep -i "request\|payload"
```

<Callout type="tip">
민감한 코드를 다루는 경우, 클라우드 제공자(Bedrock, Vertex AI) 경로를 사용하면 데이터가 해당 리전 내에서만 처리되어 추가적인 데이터 레지던시 보장을 받을 수 있습니다.
</Callout>

### 제로 데이터 리텐션(ZDR)

Anthropic API를 통한 데이터 처리에 적용되는 ZDR 정책:

- API로 전송된 프롬프트와 응답은 모델 학습에 사용되지 않음
- 요청 처리 후 데이터는 즉시 삭제
- 남용 방지를 위한 제한적 로깅만 유지 (30일 이내 삭제)
- 클라우드 제공자 경로 사용 시 해당 제공자의 보안 정책 추가 적용

## 컴플라이언스 프레임워크

### 주요 규정 준수 현황

| 프레임워크 | 상태 | 적용 영역 |
|----------|------|----------|
| SOC 2 Type II | 준수 | 데이터 보안, 가용성 |
| GDPR | 준수 | EU 개인정보 보호 |
| HIPAA | 지원 | 의료 데이터 보호 |
| CCPA | 준수 | 캘리포니아 소비자 보호 |
| ISO 27001 | 준수 | 정보보안 관리 |

<Callout type="warning">
HIPAA 컴플라이언스가 필요한 경우, 반드시 Anthropic과 BAA(Business Associate Agreement)를 체결하고, 적절한 클라우드 제공자 경로를 사용해야 합니다. 일반 API 경로로는 HIPAA 요구사항을 충족할 수 없습니다.
</Callout>

### GDPR 준수

EU 지역에서 Claude Code를 사용하는 조직을 위한 가이드:

```bash
# 데이터 처리 활동 기록 (ROPA - Records of Processing Activities)
# 다음 항목을 문서화해야 합니다:

# 1. 처리 목적: 코드 분석, 생성, 리뷰
# 2. 데이터 범주: 소스 코드, 개발 문서
# 3. 처리 근거: 정당한 이익 또는 동의
# 4. 보관 기간: 세션 기간 중만 처리, 이후 삭제
# 5. 제3자 공유: Anthropic (데이터 처리자)
```

### 감사 로그 설정

엔터프라이즈 환경에서는 AI 사용 이력을 추적하는 감사 로그가 필수입니다:

```bash
# 감사 로그 활성화 설정
# settings.json
{
  "telemetry": {
    "enabled": true,
    "level": "audit",
    "destination": "siem"
  }
}

# 감사 로그에 기록되는 정보:
# - 사용자 ID
# - 타임스탬프
# - 수행된 작업 유형
# - 접근한 파일 목록
# - 실행된 명령어
```

<CLIExample
  command="claude config set telemetry.enabled true"
  description="텔레메트리 및 감사 로그 수집을 활성화합니다"
/>

## 데이터 프라이버시 설정

### 민감 데이터 보호

Claude Code에 전송하지 않아야 하는 데이터 유형:

- 개인 식별 정보(PII): 주민등록번호, 여권 번호 등
- 금융 정보: 신용카드 번호, 계좌 정보
- 의료 정보: 환자 기록, 진단 정보
- 자격 증명: 비밀번호, API 키, 암호화 키

```bash
# .claudeignore 파일로 민감 파일 제외
# Claude Code가 읽지 못하도록 차단

# .claudeignore
.env
.env.*
*.pem
*.key
credentials/
secrets/
**/config/production.json
```

<Callout type="danger">
`.claudeignore` 파일을 설정하지 않으면 Claude Code가 프로젝트 내 모든 파일을 읽을 수 있습니다. 민감한 설정 파일, 인증서, 키 파일은 반드시 `.claudeignore`에 등록하세요.
</Callout>
