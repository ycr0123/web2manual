# 웹에서 Claude Code 사용

> 보안 클라우드 인프라에서 Claude Code 작업을 비동기적으로 실행합니다.

> 웹에서의 Claude Code는 현재 리서치 프리뷰 단계입니다.

## 웹에서의 Claude Code란?

웹에서의 Claude Code를 사용하면 개발자가 Claude 앱에서 Claude Code를 시작할 수 있습니다. 다음과 같은 경우에 특히 유용합니다:

* **질문 답변**: 코드 아키텍처 및 기능 구현 방식에 대해 질문합니다.
* **버그 수정 및 반복 작업**: 자주 개입할 필요 없이 명확하게 정의된 작업을 처리합니다.
* **병렬 작업**: 여러 버그 수정을 동시에 진행합니다.
* **로컬 머신에 없는 저장소**: 로컬에 체크아웃하지 않은 코드를 작업합니다.
* **백엔드 변경**: Claude Code가 테스트를 작성한 후 해당 테스트를 통과하는 코드를 작성합니다.

Claude Code는 Claude iOS 앱에서도 사용할 수 있어, 이동 중에 작업을 시작하고 진행 상황을 모니터링할 수 있습니다.

로컬과 원격 개발을 자유롭게 전환할 수 있습니다. `&` 접두사를 사용해 터미널에서 웹으로 작업을 전송하거나, 웹 세션을 터미널로 가져와 로컬에서 계속 작업할 수 있습니다.

## 웹에서의 Claude Code를 사용할 수 있는 사람은?

웹에서의 Claude Code는 다음 사용자를 대상으로 리서치 프리뷰로 제공됩니다:

* **Pro 사용자**
* **Max 사용자**
* **Team 사용자**
* **Enterprise 사용자** (프리미엄 시트 또는 Chat + Claude Code 시트 보유자)

## 시작하기

1. claude.ai/code 방문
2. GitHub 계정 연결
3. 저장소에 Claude GitHub 앱 설치
4. 기본 환경 선택
5. 코딩 작업 제출
6. diff 뷰에서 변경 사항 검토, 댓글로 반복 수정, 그런 다음 pull request 생성

## 작동 방식

웹에서 Claude Code 작업을 시작하면:

1. **저장소 클로닝**: 저장소가 Anthropic이 관리하는 가상 머신에 클로닝됩니다.
2. **환경 설정**: Claude가 코드와 함께 보안 클라우드 환경을 준비합니다.
3. **네트워크 구성**: 설정에 따라 인터넷 접근이 구성됩니다.
4. **작업 실행**: Claude가 코드를 분석하고, 변경 사항을 적용하고, 테스트를 실행하고, 작업을 검토합니다.
5. **완료**: 작업이 완료되면 알림을 받고 변경 사항으로 PR을 생성할 수 있습니다.
6. **결과**: 변경 사항이 브랜치에 푸시되어 pull request 생성 준비가 완료됩니다.

## diff 뷰로 변경 사항 검토

diff 뷰를 사용하면 pull request를 생성하기 전에 Claude가 변경한 내용을 정확히 확인할 수 있습니다. "PR 생성"을 클릭하여 GitHub에서 변경 사항을 검토하는 대신, 앱에서 직접 diff를 확인하고 변경 사항이 준비될 때까지 Claude와 반복 작업할 수 있습니다.

Claude가 파일을 변경하면 추가 및 삭제된 줄 수를 나타내는 diff 통계 표시기(예: `+12 -1`)가 나타납니다. 이 표시기를 선택하면 왼쪽에 파일 목록, 오른쪽에 각 파일의 변경 사항이 표시되는 diff 뷰어가 열립니다.

diff 뷰에서 다음을 수행할 수 있습니다:

* 파일별로 변경 사항 검토
* 특정 변경 사항에 댓글을 달아 수정 요청
* 확인한 내용을 바탕으로 Claude와 계속 반복 작업

이를 통해 임시 PR을 생성하거나 GitHub으로 전환하지 않고도 여러 차례 피드백을 주고받으며 변경 사항을 다듬을 수 있습니다.

## 웹과 터미널 간 작업 이동

웹에서 작업을 시작하고 터미널에서 계속하거나, 터미널에서 웹으로 작업을 전송할 수 있습니다. 웹 세션은 노트북을 닫아도 유지되며, Claude iOS 앱을 포함해 어디서든 모니터링할 수 있습니다.

> 세션 전환은 단방향입니다. 웹 세션을 터미널로 가져올 수 있지만, 기존 터미널 세션을 웹으로 푸시할 수는 없습니다. `&` 접두사는 현재 대화 컨텍스트로 *새* 웹 세션을 생성합니다.

### 터미널에서 웹으로

Claude Code 내에서 메시지를 `&`로 시작하면 웹에서 실행할 작업을 전송합니다:

```
& Fix the authentication bug in src/auth/login.ts
```

이렇게 하면 현재 대화 컨텍스트로 claude.ai에 새 웹 세션이 생성됩니다. 작업이 클라우드에서 실행되는 동안 로컬에서 계속 작업할 수 있습니다. `/tasks`를 사용해 진행 상황을 확인하거나, claude.ai 또는 Claude iOS 앱에서 세션을 열어 직접 상호작용할 수 있습니다. 거기서 Claude를 조종하거나, 피드백을 제공하거나, 다른 대화처럼 질문에 답할 수 있습니다.

커맨드라인에서 직접 웹 세션을 시작할 수도 있습니다:

```bash
claude --remote "Fix the authentication bug in src/auth/login.ts"
```

#### 백그라운드 작업을 위한 팁

**로컬에서 계획하고, 원격에서 실행하기**: 복잡한 작업의 경우, 웹으로 작업을 전송하기 전에 계획 모드에서 Claude와 함께 접근 방식을 협의하세요:

```bash
claude --permission-mode plan
```

계획 모드에서 Claude는 파일을 읽고 코드베이스를 탐색하는 것만 가능합니다. 계획에 만족하면 자율 실행을 위해 웹으로 전송합니다:

```
& Execute the migration plan we discussed
```

이 패턴을 통해 전략에 대한 통제권을 유지하면서 Claude가 클라우드에서 자율적으로 실행하도록 할 수 있습니다.

**병렬로 작업 실행하기**: 각 `&` 명령은 독립적으로 실행되는 자체 웹 세션을 생성합니다. 여러 작업을 시작하면 별도의 세션에서 동시에 실행됩니다:

```
& Fix the flaky test in auth.spec.ts
& Update the API documentation
& Refactor the logger to use structured output
```

`/tasks`로 모든 세션을 모니터링하세요. 세션이 완료되면 웹 인터페이스에서 PR을 생성하거나, 세션을 터미널로 가져와 계속 작업할 수 있습니다.

### 웹에서 터미널로

웹 세션을 터미널로 가져오는 방법은 여러 가지가 있습니다:

* **`/teleport` 사용**: Claude Code 내에서 `/teleport`(또는 `/tp`)를 실행하면 웹 세션의 인터랙티브 선택기가 표시됩니다. 커밋되지 않은 변경 사항이 있는 경우 먼저 스태시할 것인지 묻는 메시지가 표시됩니다.
* **`--teleport` 사용**: 커맨드라인에서 `claude --teleport`를 실행하면 인터랙티브 세션 선택기가 열리고, `claude --teleport <session-id>`를 사용하면 특정 세션으로 바로 이동할 수 있습니다.
* **`/tasks` 사용**: `/tasks`를 실행하여 백그라운드 세션을 확인한 다음, `t`를 눌러 세션으로 이동합니다.
* **웹 인터페이스 사용**: "Open in CLI"를 클릭하여 터미널에 붙여넣을 수 있는 명령을 복사합니다.

세션을 텔레포트하면 Claude는 올바른 저장소에 있는지 확인하고, 원격 세션의 브랜치를 가져와 체크아웃한 후, 전체 대화 기록을 터미널에 로드합니다.

#### 텔레포트 요구 사항

텔레포트는 세션을 재개하기 전에 다음 요구 사항을 확인합니다. 요구 사항이 충족되지 않으면 오류가 표시되거나 문제를 해결하라는 메시지가 나타납니다.

| 요구 사항 | 세부 내용 |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| 깨끗한 git 상태 | 작업 디렉토리에 커밋되지 않은 변경 사항이 없어야 합니다. 필요한 경우 텔레포트가 변경 사항을 스태시할 것인지 묻습니다. |
| 올바른 저장소 | 포크가 아닌 동일한 저장소의 체크아웃에서 `--teleport`를 실행해야 합니다. |
| 브랜치 사용 가능 | 웹 세션의 브랜치가 원격에 푸시되어 있어야 합니다. 텔레포트가 자동으로 가져와 체크아웃합니다. |
| 동일한 계정 | 웹 세션에 사용된 것과 동일한 Claude.ai 계정으로 인증되어 있어야 합니다. |

### 세션 공유

세션을 공유하려면 아래 계정 유형에 따라 공개 범위를 설정하세요. 그런 다음 세션 링크를 그대로 공유합니다. 공유 세션을 열면 수신자는 로드 시점의 최신 세션 상태를 볼 수 있지만, 수신자 페이지는 실시간으로 업데이트되지 않습니다.

#### Enterprise 또는 Teams 계정에서 공유

Enterprise 및 Teams 계정의 경우 공개 범위 옵션은 **비공개(Private)**와 **팀(Team)** 두 가지입니다. 팀 공개로 설정하면 Claude.ai 조직의 다른 구성원이 세션을 볼 수 있습니다. 저장소 접근 확인은 기본적으로 활성화되어 있으며, 수신자 계정에 연결된 GitHub 계정을 기반으로 합니다. 계정 표시 이름은 접근 권한이 있는 모든 수신자에게 표시됩니다. Slack의 Claude 세션은 자동으로 팀 공개로 공유됩니다.

#### Max 또는 Pro 계정에서 공유

Max 및 Pro 계정의 경우 공개 범위 옵션은 **비공개(Private)**와 **공개(Public)** 두 가지입니다. 공개로 설정하면 claude.ai에 로그인한 모든 사용자가 세션을 볼 수 있습니다.

공유하기 전에 세션에 민감한 내용이 없는지 확인하세요. 세션에는 비공개 GitHub 저장소의 코드와 자격 증명이 포함될 수 있습니다. 기본적으로 저장소 접근 확인은 활성화되어 있지 않습니다.

Settings > Claude Code > Sharing settings에서 저장소 접근 확인을 활성화하거나 공유 세션에서 이름을 숨길 수 있습니다.

## 클라우드 환경

### 기본 이미지

일반적인 툴체인과 언어 생태계가 미리 설치된 범용 이미지를 빌드하고 유지 관리합니다. 이 이미지에는 다음이 포함됩니다:

* 주요 프로그래밍 언어 및 런타임
* 공통 빌드 도구 및 패키지 매니저
* 테스팅 프레임워크 및 린터

#### 사용 가능한 도구 확인

환경에 미리 설치된 항목을 확인하려면 Claude Code에 다음을 실행하도록 요청하세요:

```bash
check-tools
```

이 명령은 다음을 표시합니다:

* 프로그래밍 언어 및 버전
* 사용 가능한 패키지 매니저
* 설치된 개발 도구

#### 언어별 설정

범용 이미지에는 다음에 대한 사전 구성된 환경이 포함됩니다:

* **Python**: Python 3.x (pip, poetry 및 일반 과학 라이브러리 포함)
* **Node.js**: 최신 LTS 버전 (npm, yarn, pnpm, bun 포함)
* **Ruby**: 버전 3.1.6, 3.2.6, 3.3.6 (기본값: 3.3.6, gem, bundler, rbenv 포함)
* **PHP**: 버전 8.4.14
* **Java**: OpenJDK (Maven 및 Gradle 포함)
* **Go**: 최신 안정 버전 (모듈 지원 포함)
* **Rust**: Rust 툴체인 (cargo 포함)
* **C++**: GCC 및 Clang 컴파일러

#### 데이터베이스

범용 이미지에는 다음 데이터베이스가 포함됩니다:

* **PostgreSQL**: 버전 16
* **Redis**: 버전 7.0

### 환경 구성

웹에서 Claude Code 세션을 시작하면 내부적으로 다음이 진행됩니다:

1. **환경 준비**: 저장소를 클로닝하고 초기화를 위해 구성된 Claude 훅을 실행합니다. 저장소는 GitHub 저장소의 기본 브랜치로 클로닝됩니다. 특정 브랜치를 체크아웃하려면 프롬프트에서 지정할 수 있습니다.

2. **네트워크 구성**: 에이전트의 인터넷 접근을 구성합니다. 기본적으로 인터넷 접근은 제한되지만, 필요에 따라 인터넷 접근 없음 또는 전체 인터넷 접근으로 환경을 구성할 수 있습니다.

3. **Claude Code 실행**: Claude Code가 작업을 완료하기 위해 실행되며, 코드를 작성하고, 테스트를 실행하고, 작업을 검토합니다. 웹 인터페이스를 통해 세션 전반에 걸쳐 Claude를 안내하고 조종할 수 있습니다. Claude는 `CLAUDE.md`에 정의된 컨텍스트를 존중합니다.

4. **결과**: Claude가 작업을 완료하면 브랜치를 원격에 푸시합니다. 해당 브랜치에 대한 PR을 생성할 수 있습니다.

> Claude는 환경에서 사용 가능한 터미널 및 CLI 도구를 통해서만 작동합니다. 범용 이미지에 미리 설치된 도구와 훅 또는 종속성 관리를 통해 추가로 설치한 도구를 사용합니다.

**새 환경 추가**: 현재 환경을 선택하여 환경 선택기를 열고, "Add environment"를 선택합니다. 환경 이름, 네트워크 접근 수준, 설정할 환경 변수를 지정할 수 있는 대화 상자가 열립니다.

**기존 환경 업데이트**: 현재 환경을 선택한 다음, 환경 이름 오른쪽의 설정 버튼을 선택합니다. 환경 이름, 네트워크 접근, 환경 변수를 업데이트할 수 있는 대화 상자가 열립니다.

**터미널에서 기본 환경 선택**: 여러 환경이 구성된 경우 `/remote-env`를 실행하여 `&` 또는 `--remote`로 터미널에서 웹 세션을 시작할 때 사용할 환경을 선택합니다. 환경이 하나뿐인 경우 이 명령은 현재 구성을 표시합니다.

> 환경 변수는 `.env` 형식의 키-값 쌍으로 지정해야 합니다. 예를 들면:
>
> ```
> API_KEY=your_api_key
> DEBUG=true
> ```

### 종속성 관리

사용자 지정 환경 이미지와 스냅샷은 아직 지원되지 않습니다. 임시 방편으로 SessionStart 훅을 사용하여 세션 시작 시 패키지를 설치할 수 있습니다. 이 방법에는 알려진 제한 사항이 있습니다.

자동 종속성 설치를 구성하려면 저장소의 `.claude/settings.json` 파일에 SessionStart 훅을 추가하세요:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/scripts/install_pkgs.sh"
          }
        ]
      }
    ]
  }
}
```

`scripts/install_pkgs.sh`에 해당 스크립트를 생성합니다:

```bash
#!/bin/bash

# Only run in remote environments
if [ "$CLAUDE_CODE_REMOTE" != "true" ]; then
  exit 0
fi

npm install
pip install -r requirements.txt
exit 0
```

실행 권한 부여: `chmod +x scripts/install_pkgs.sh`

#### 환경 변수 유지

SessionStart 훅은 `CLAUDE_ENV_FILE` 환경 변수에 지정된 파일에 기록하여 이후 Bash 명령에서 사용할 환경 변수를 유지할 수 있습니다. 자세한 내용은 훅 참조의 SessionStart 훅을 참고하세요.

#### 종속성 관리 제한 사항

* **모든 세션에서 훅 실행**: SessionStart 훅은 로컬 및 원격 환경 모두에서 실행됩니다. 훅을 원격 세션에만 적용하도록 범위를 제한하는 훅 구성은 없습니다. 로컬 실행을 건너뛰려면 위에 표시된 것처럼 스크립트에서 `CLAUDE_CODE_REMOTE` 환경 변수를 확인하세요.
* **네트워크 접근 필요**: 설치 명령은 패키지 레지스트리에 접근하기 위해 네트워크가 필요합니다. 환경이 "인터넷 없음" 접근으로 구성된 경우 이 훅은 실패합니다. "제한됨"(기본값) 또는 "전체" 네트워크 접근을 사용하세요. 기본 허용 목록에는 npm, PyPI, RubyGems, crates.io와 같은 공통 레지스트리가 포함됩니다.
* **프록시 호환성**: 원격 환경의 모든 아웃바운드 트래픽은 보안 프록시를 통과합니다. 일부 패키지 매니저는 이 프록시와 올바르게 작동하지 않습니다. Bun이 대표적인 예입니다.
* **매 세션 시작 시 실행**: 훅은 세션이 시작되거나 재개될 때마다 실행되므로 시작 지연이 발생합니다. 재설치 전에 종속성이 이미 설치되어 있는지 확인하여 설치 스크립트를 빠르게 유지하세요.

#### GitHub 프록시

보안을 위해 모든 GitHub 작업은 모든 git 상호작용을 투명하게 처리하는 전용 프록시 서비스를 통해 이루어집니다. 샌드박스 내에서 git 클라이언트는 사용자 지정으로 빌드된 범위 지정 자격 증명을 사용하여 인증합니다. 이 프록시는:

* GitHub 인증을 안전하게 관리합니다. git 클라이언트는 샌드박스 내에서 범위 지정 자격 증명을 사용하며, 프록시는 이를 확인하고 실제 GitHub 인증 토큰으로 변환합니다.
* 안전을 위해 git push 작업을 현재 작업 브랜치로 제한합니다.
* 보안 경계를 유지하면서 원활한 클로닝, 페칭, PR 작업을 가능하게 합니다.

#### 보안 프록시

환경은 보안 및 남용 방지를 위해 HTTP/HTTPS 네트워크 프록시 뒤에서 실행됩니다. 모든 아웃바운드 인터넷 트래픽은 이 프록시를 통과하며, 다음을 제공합니다:

* 악성 요청으로부터의 보호
* 속도 제한 및 남용 방지
* 보안 강화를 위한 콘텐츠 필터링

## 보안 및 격리

웹에서의 Claude Code는 강력한 보안을 보장합니다:

* **격리된 가상 머신**: 각 세션은 Anthropic이 관리하는 격리된 VM에서 실행됩니다.
* **네트워크 접근 제어**: 기본적으로 네트워크 접근이 제한되며, 비활성화할 수 있습니다.

> 네트워크 접근이 비활성화된 상태에서 실행할 때도 Claude Code는 Anthropic API와 통신할 수 있으며, 이를 통해 데이터가 격리된 Claude Code VM 외부로 나갈 수 있습니다.

* **자격 증명 보호**: git 자격 증명이나 서명 키와 같은 민감한 자격 증명은 Claude Code가 있는 샌드박스 내에 절대 저장되지 않습니다. 인증은 범위 지정 자격 증명을 사용하는 보안 프록시를 통해 처리됩니다.
* **보안 분석**: PR 생성 전에 격리된 VM 내에서 코드를 분석하고 수정합니다.

## 가격 및 속도 제한

웹에서의 Claude Code는 계정 내 다른 모든 Claude 및 Claude Code 사용량과 속도 제한을 공유합니다. 여러 작업을 병렬로 실행하면 그에 비례하여 더 많은 속도 제한을 소비합니다.

## 제한 사항

* **저장소 인증**: 동일한 계정으로 인증된 경우에만 웹에서 로컬로 세션을 이동할 수 있습니다.
* **플랫폼 제한**: 웹에서의 Claude Code는 GitHub에 호스팅된 코드에서만 작동합니다. GitLab 및 기타 비 GitHub 저장소는 클라우드 세션에서 사용할 수 없습니다.
