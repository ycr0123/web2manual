# Claude Code 설치 및 설정

> 개발 머신에서 Claude Code를 설치하고 인증하여 사용을 시작합니다.

## 시스템 요구 사항

- **운영 체제**:
  - macOS 13.0 이상
  - Windows 10 1809 이상 또는 Windows Server 2019 이상
  - Ubuntu 20.04 이상
  - Debian 10 이상
  - Alpine Linux 3.19 이상 (추가 의존성 필요)
- **하드웨어**: RAM 4 GB 이상
- **네트워크**: 인터넷 연결 필요
- **셸**: Bash 또는 Zsh에서 가장 잘 동작
- **위치**: Anthropic 지원 국가

### 추가 의존성

- **ripgrep**: 보통 Claude Code에 포함됩니다. 검색이 실패하면 검색 문제 해결을 참고하세요.
- **Node.js 18 이상**: 지원 중단된 npm 설치에만 필요합니다.

## 설치

**네이티브 설치 (권장)**

macOS, Linux, WSL:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Windows PowerShell:

```powershell
irm https://claude.ai/install.ps1 | iex
```

Windows CMD:

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

네이티브 설치는 최신 버전을 유지하기 위해 백그라운드에서 자동으로 업데이트됩니다.

**Homebrew**

```sh
brew install --cask claude-code
```

Homebrew 설치는 자동 업데이트되지 않습니다. 주기적으로 `brew upgrade claude-code`를 실행하세요.

**WinGet**

```powershell
winget install Anthropic.ClaudeCode
```

WinGet 설치는 자동 업데이트되지 않습니다. 주기적으로 `winget upgrade Anthropic.ClaudeCode`를 실행하세요.

설치 프로세스가 완료되면 프로젝트로 이동하여 Claude Code를 시작하세요:

```bash
cd your-awesome-project
claude
```

> 설치 후 `claude doctor`를 실행하여 설치 유형과 버전을 확인하세요.

### 플랫폼별 설정

**Windows**: Claude Code를 네이티브로 실행하거나 (Git Bash 필요) WSL 내에서 실행합니다. WSL 1과 WSL 2 모두 지원되지만, WSL 1은 제한된 지원만 제공하며 Bash 도구 샌드박싱 같은 기능은 지원하지 않습니다.

**Alpine Linux 및 기타 musl/uClibc 기반 배포판**:

Alpine 및 기타 musl/uClibc 기반 배포판의 네이티브 설치 프로그램에는 `libgcc`, `libstdc++`, `ripgrep`이 필요합니다. 배포판의 패키지 관리자를 사용하여 설치한 다음 `USE_BUILTIN_RIPGREP=0`을 설정하세요.

Alpine에서:

```bash
apk add libgcc libstdc++ ripgrep
```

### 인증

#### 개인 사용자

1. **Claude Pro 또는 Max 플랜** (권장): Claude의 Pro 또는 Max 플랜을 구독하면 Claude Code와 웹용 Claude를 포함하는 통합 구독을 사용할 수 있습니다. 하나의 계정에서 계정을 관리하고 Claude.ai 계정으로 로그인합니다.
2. **Claude Console**: Claude Console을 통해 연결하고 OAuth 프로세스를 완료합니다. Anthropic Console에서 활성 결제가 필요합니다. 사용량 추적 및 비용 관리를 위해 "Claude Code" 워크스페이스가 자동으로 생성됩니다.

#### 팀 및 조직

1. **Teams 또는 Enterprise용 Claude** (권장): Teams 또는 Enterprise용 Claude를 구독하면 중앙 집중식 결제, 팀 관리, Claude Code와 웹용 Claude 모두에 대한 접근 권한을 얻을 수 있습니다.
2. **팀 결제가 포함된 Claude Console**: 팀 결제로 공유 Claude Console 조직을 설정합니다. 팀원을 초대하고 사용량 추적을 위한 역할을 할당합니다.
3. **클라우드 제공자**: 기존 클라우드 인프라로 배포하기 위해 Amazon Bedrock, Google Vertex AI, Microsoft Foundry를 사용하도록 Claude Code를 구성합니다.

### 특정 버전 설치

네이티브 설치 프로그램은 특정 버전 번호나 릴리스 채널(`latest` 또는 `stable`)을 허용합니다.

최신 버전 설치 (기본값):

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

안정 버전 설치:

```bash
curl -fsSL https://claude.ai/install.sh | bash -s stable
```

특정 버전 번호 설치:

```bash
curl -fsSL https://claude.ai/install.sh | bash -s 1.0.58
```

**Windows PowerShell - 특정 버전:**

```powershell
& ([scriptblock]::Create((irm https://claude.ai/install.ps1))) 1.0.58
```

### 바이너리 무결성 및 코드 서명

- 모든 플랫폼의 SHA256 체크섬이 릴리스 매니페스트에 게시됩니다
- 서명된 바이너리가 다음에 배포됩니다:
  - macOS: "Anthropic PBC"가 서명하고 Apple이 공증
  - Windows: "Anthropic, PBC"가 서명

## NPM 설치 (지원 중단)

NPM 설치는 지원이 중단되었습니다. 가능하면 네이티브 설치 방법을 사용하세요. 기존 npm 설치를 네이티브로 마이그레이션하려면 `claude install`을 실행하세요.

**전역 npm 설치**

```sh
npm install -g @anthropic-ai/claude-code
```

> 권한 문제와 보안 위험을 초래할 수 있으므로 `sudo npm install -g`를 사용하지 마세요.

## Windows 설정

**옵션 1: WSL 내 Claude Code**

- WSL 1과 WSL 2 모두 지원됩니다
- WSL 2는 보안 강화를 위한 샌드박싱을 지원합니다. WSL 1은 샌드박싱을 지원하지 않습니다.

**옵션 2: Git Bash를 사용하는 네이티브 Windows의 Claude Code**

- Windows용 Git 필요
- 이식 가능한 Git 설치의 경우 `bash.exe` 경로를 지정하세요:

```powershell
$env:CLAUDE_CODE_GIT_BASH_PATH="C:\Program Files\Git\bin\bash.exe"
```

## Claude Code 업데이트

### 자동 업데이트

Claude Code는 최신 기능과 보안 수정을 항상 유지하기 위해 자동으로 업데이트됩니다.

- **업데이트 확인**: 시작 시 및 실행 중 주기적으로 수행
- **업데이트 프로세스**: 백그라운드에서 자동으로 다운로드 및 설치
- **알림**: 업데이트가 설치되면 알림이 표시됩니다
- **업데이트 적용**: 다음에 Claude Code를 시작할 때 업데이트가 적용됩니다

> Homebrew 및 WinGet 설치는 자동 업데이트되지 않습니다. 수동으로 업데이트하려면 `brew upgrade claude-code` 또는 `winget upgrade Anthropic.ClaudeCode`를 사용하세요.

### 릴리스 채널 구성

`autoUpdatesChannel` 설정으로 자동 업데이트와 `claude update` 모두에 대해 Claude Code가 따르는 릴리스 채널을 구성합니다:

- `"latest"` (기본값): 릴리스되는 즉시 새 기능 수신
- `"stable"`: 보통 약 1주일 된 버전 사용, 주요 회귀가 있는 릴리스 건너뜀

`/config` → **자동 업데이트 채널**을 통해 구성하거나 settings.json 파일에 추가하세요:

```json
{
  "autoUpdatesChannel": "stable"
}
```

### 자동 업데이트 비활성화

셸 또는 settings.json 파일에서 `DISABLE_AUTOUPDATER` 환경 변수를 설정하세요:

```bash
export DISABLE_AUTOUPDATER=1
```

### 수동 업데이트

```bash
claude update
```

## Claude Code 제거

### 네이티브 설치

**macOS, Linux, WSL:**

```bash
rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude
```

**Windows PowerShell:**

```powershell
Remove-Item -Path "$env:USERPROFILE\.local\bin\claude.exe" -Force
Remove-Item -Path "$env:USERPROFILE\.local\share\claude" -Recurse -Force
```

**Windows CMD:**

```batch
del "%USERPROFILE%\.local\bin\claude.exe"
rmdir /s /q "%USERPROFILE%\.local\share\claude"
```

### Homebrew 설치

```bash
brew uninstall --cask claude-code
```

### WinGet 설치

```powershell
winget uninstall Anthropic.ClaudeCode
```

### NPM 설치

```bash
npm uninstall -g @anthropic-ai/claude-code
```

### 구성 파일 정리 (선택 사항)

> **경고**: 구성 파일을 제거하면 모든 설정, 허용된 도구, MCP 서버 구성, 세션 기록이 삭제됩니다.

**macOS, Linux, WSL:**

```bash
# 사용자 설정 및 상태 제거
rm -rf ~/.claude
rm ~/.claude.json

# 프로젝트별 설정 제거 (프로젝트 디렉토리에서 실행)
rm -rf .claude
rm -f .mcp.json
```

**Windows PowerShell:**

```powershell
# 사용자 설정 및 상태 제거
Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force
Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force

# 프로젝트별 설정 제거 (프로젝트 디렉토리에서 실행)
Remove-Item -Path ".claude" -Recurse -Force
Remove-Item -Path ".mcp.json" -Force
```

**Windows CMD:**

```batch
REM 사용자 설정 및 상태 제거
rmdir /s /q "%USERPROFILE%\.claude"
del "%USERPROFILE%\.claude.json"

REM 프로젝트별 설정 제거 (프로젝트 디렉토리에서 실행)
rmdir /s /q ".claude"
del ".mcp.json"
```
