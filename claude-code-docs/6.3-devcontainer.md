# Dev Container 설정

> 일관되고 안전한 환경이 필요한 팀을 위한 Claude Code 개발 컨테이너에 대해 알아봅니다.

참조 [devcontainer 설정](https://github.com/anthropics/claude-code/tree/main/.devcontainer)과 관련 [Dockerfile](https://github.com/anthropics/claude-code/blob/main/.devcontainer/Dockerfile)은 그대로 사용하거나 필요에 맞게 커스터마이즈할 수 있는 미리 구성된 개발 컨테이너를 제공합니다. 이 devcontainer는 Visual Studio Code [Dev Containers 확장](https://code.visualstudio.com/docs/devcontainers/containers) 및 유사한 도구와 함께 작동합니다.

컨테이너의 강화된 보안 조치(격리 및 방화벽 규칙)를 통해 무인 작동을 위한 권한 프롬프트를 우회하는 `claude --dangerously-skip-permissions`를 실행할 수 있습니다.

> **경고**: devcontainer는 상당한 보호를 제공하지만 모든 공격에 완전히 면역인 시스템은 없습니다.
> `--dangerously-skip-permissions`로 실행하면 devcontainer는 악성 프로젝트가 Claude Code 자격 증명을 포함하여 devcontainer에서 접근 가능한 모든 것을 유출하는 것을 막을 수 없습니다.
> devcontainer는 신뢰할 수 있는 저장소로 개발할 때만 사용하는 것을 권장합니다.
> 항상 좋은 보안 관행을 유지하고 Claude의 활동을 모니터링하세요.

## 주요 기능

* **프로덕션 준비된 Node.js**: 필수 개발 의존성이 있는 Node.js 20 기반
* **보안 설계**: 필요한 서비스에만 네트워크 접근을 제한하는 커스텀 방화벽
* **개발자 친화적 도구**: git, 생산성 향상 기능이 있는 ZSH, fzf 등 포함
* **원활한 VS Code 통합**: 미리 구성된 확장 기능 및 최적화된 설정
* **세션 지속성**: 컨테이너 재시작 간에 명령어 기록과 구성 보존
* **어디서나 작동**: macOS, Windows, Linux 개발 환경과 호환

## 4단계로 시작하기

1. VS Code와 Remote - Containers 확장 설치
2. [Claude Code 참조 구현](https://github.com/anthropics/claude-code/tree/main/.devcontainer) 저장소 클론
3. VS Code에서 저장소 열기
4. 메시지가 표시되면 "컨테이너에서 다시 열기" 클릭 (또는 명령 팔레트 사용: Cmd+Shift+P → "Remote-Containers: Reopen in Container")

## 구성 분석

devcontainer 설정은 세 가지 주요 컴포넌트로 구성됩니다:

* [**devcontainer.json**](https://github.com/anthropics/claude-code/blob/main/.devcontainer/devcontainer.json): 컨테이너 설정, 확장 기능, 볼륨 마운트 제어
* [**Dockerfile**](https://github.com/anthropics/claude-code/blob/main/.devcontainer/Dockerfile): 컨테이너 이미지와 설치된 도구 정의
* [**init-firewall.sh**](https://github.com/anthropics/claude-code/blob/main/.devcontainer/init-firewall.sh): 네트워크 보안 규칙 설정

## 보안 기능

컨테이너는 방화벽 구성으로 다층 보안 접근 방식을 구현합니다:

* **정밀한 접근 제어**: 허용 목록의 도메인으로만 발신 연결 제한 (npm 레지스트리, GitHub, Claude API 등)
* **허용된 발신 연결**: 방화벽이 발신 DNS 및 SSH 연결을 허용합니다
* **기본 거부 정책**: 다른 모든 외부 네트워크 접근 차단
* **시작 시 검증**: 컨테이너 초기화 시 방화벽 규칙 검증
* **격리**: 메인 시스템과 분리된 안전한 개발 환경 생성

## 커스터마이즈 옵션

devcontainer 구성은 필요에 맞게 조정할 수 있도록 설계되었습니다:

* 워크플로우에 따라 VS Code 확장 추가 또는 제거
* 다른 하드웨어 환경을 위한 리소스 할당 수정
* 네트워크 접근 권한 조정
* 셸 구성과 개발자 도구 커스터마이즈

## 예시 활용 사례

### 안전한 클라이언트 작업

devcontainer를 사용하여 서로 다른 클라이언트 프로젝트를 격리하고 코드와 자격 증명이 환경 간에 섞이지 않도록 합니다.

### 팀 온보딩

새 팀원들이 필요한 모든 도구와 설정이 미리 설치된 완전히 구성된 개발 환경을 몇 분 안에 갖출 수 있습니다.

### 일관된 CI/CD 환경

CI/CD 파이프라인에서 devcontainer 구성을 미러링하여 개발 및 프로덕션 환경이 일치하도록 합니다.
