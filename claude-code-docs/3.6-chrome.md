# Chrome에서 Claude Code 사용 (베타)

> Claude Code를 Chrome 브라우저에 연결하여 웹 앱 테스트, 콘솔 로그 디버깅, 폼 자동화 등을 수행합니다.

Claude Code는 Claude in Chrome 브라우저 extension과 통합되어 CLI 또는 VS Code extension에서 브라우저 자동화 기능을 제공합니다. 코드를 빌드한 다음, 컨텍스트를 전환하지 않고도 브라우저에서 바로 테스트하고 디버깅할 수 있습니다.

Claude는 브라우저 작업을 위해 새 탭을 열고 브라우저의 로그인 상태를 공유하므로, 이미 로그인된 모든 사이트에 접근할 수 있습니다. 브라우저 동작은 실시간으로 눈에 보이는 Chrome 창에서 실행됩니다. Claude가 로그인 페이지나 CAPTCHA를 만나면 일시 중지하고 수동으로 처리해 달라고 요청합니다.

> Chrome 통합은 베타 상태이며 현재 Google Chrome과 Microsoft Edge에서 동작합니다. Brave, Arc 및 기타 Chromium 기반 브라우저는 아직 지원되지 않습니다. WSL(Windows Subsystem for Linux)도 지원되지 않습니다.

## 기능

Chrome이 연결된 상태에서 브라우저 동작과 코딩 작업을 하나의 워크플로우로 연결할 수 있습니다.

* **라이브 디버깅**: 콘솔 오류와 DOM 상태를 직접 읽고, 원인이 된 코드를 즉시 수정합니다.
* **디자인 검증**: Figma 목업으로 UI를 빌드한 후 브라우저에서 열어 일치 여부를 확인합니다.
* **웹 앱 테스트**: 폼 유효성 검사 테스트, 시각적 회귀 확인, 사용자 플로우 검증을 수행합니다.
* **인증이 필요한 웹 앱**: API 커넥터 없이 Google Docs, Gmail, Notion 또는 로그인된 모든 앱과 상호작용합니다.
* **데이터 추출**: 웹 페이지에서 구조화된 정보를 가져와 로컬에 저장합니다.
* **작업 자동화**: 데이터 입력, 폼 작성, 다중 사이트 워크플로우 등 반복적인 브라우저 작업을 자동화합니다.
* **세션 녹화**: 브라우저 상호작용을 GIF로 녹화하여 문서화하거나 공유합니다.

## 사전 요구사항

Chrome과 함께 Claude Code를 사용하기 전에 다음이 필요합니다.

* Google Chrome 또는 Microsoft Edge 브라우저
* Chrome Web Store에서 두 브라우저 모두 사용 가능한 Claude in Chrome extension 버전 1.0.36 이상
* Claude Code 버전 2.0.73 이상
* Anthropic 직접 플랜(Pro, Max, Teams, 또는 Enterprise)

> Chrome 통합은 Amazon Bedrock, Google Cloud Vertex AI, Microsoft Foundry 등 서드파티 제공업체를 통해서는 사용할 수 없습니다. Claude를 서드파티 제공업체를 통해서만 이용하는 경우, 이 기능을 사용하려면 별도의 claude.ai 계정이 필요합니다.

## CLI에서 시작하기

### 1단계: Chrome으로 Claude Code 실행

`--chrome` 플래그를 사용하여 Claude Code를 시작합니다.

```bash
claude --chrome
```

기존 세션 내에서 `/chrome`을 실행하여 Chrome을 활성화할 수도 있습니다.

### 2단계: Claude에게 브라우저 사용 요청

다음 예시는 페이지로 이동하여 상호작용하고 결과를 보고하는 작업을 터미널이나 에디터에서 직접 수행합니다.

```
Go to code.claude.com/docs, click on the search box,
type "hooks", and tell me what results appear
```

언제든지 `/chrome`을 실행하여 연결 상태를 확인하거나, 권한을 관리하거나, extension을 재연결할 수 있습니다.

VS Code에서의 사용은 VS Code의 브라우저 자동화를 참고하세요.

### 기본값으로 Chrome 활성화

매 세션마다 `--chrome`을 전달하지 않으려면 `/chrome`을 실행하고 "Enabled by default"를 선택하세요.

VS Code extension에서는 Chrome extension이 설치되어 있으면 Chrome을 항상 사용할 수 있으며 추가 플래그가 필요하지 않습니다.

> CLI에서 Chrome을 기본값으로 활성화하면 브라우저 도구가 항상 로드되므로 컨텍스트 사용량이 늘어납니다. 컨텍스트 소비가 증가하는 경우 이 설정을 비활성화하고 필요할 때만 `--chrome`을 사용하세요.

### 사이트 권한 관리

사이트 수준 권한은 Chrome extension에서 상속됩니다. Chrome extension 설정에서 권한을 관리하여 Claude가 탐색, 클릭, 입력할 수 있는 사이트를 제어하세요.

## 예시 워크플로우

다음 예시들은 브라우저 동작과 코딩 작업을 결합하는 일반적인 방법을 보여줍니다. `/mcp`를 실행하고 `claude-in-chrome`을 선택하면 사용 가능한 전체 브라우저 도구 목록을 확인할 수 있습니다.

### 로컬 웹 애플리케이션 테스트

웹 앱을 개발할 때 Claude에게 변경 사항이 올바르게 동작하는지 확인하도록 요청합니다.

```
I just updated the login form validation. Can you open localhost:3000,
try submitting the form with invalid data, and check if the error
messages appear correctly?
```

Claude는 로컬 서버로 이동하여 폼과 상호작용하고 관찰한 결과를 보고합니다.

### 콘솔 로그로 디버깅

Claude는 콘솔 출력을 읽어 문제 진단을 도울 수 있습니다. 로그가 방대할 수 있으므로 모든 콘솔 출력을 요청하기보다 특정 패턴을 찾아달라고 지정하세요.

```
Open the dashboard page and check the console for any errors when
the page loads.
```

Claude는 콘솔 메시지를 읽고 특정 패턴이나 오류 유형을 필터링할 수 있습니다.

### 폼 자동 입력

반복적인 데이터 입력 작업의 속도를 높입니다.

```
I have a spreadsheet of customer contacts in contacts.csv. For each row,
go to the CRM at crm.example.com, click "Add Contact", and fill in the
name, email, and phone fields.
```

Claude는 로컬 파일을 읽고, 웹 인터페이스를 탐색하여 각 레코드의 데이터를 입력합니다.

### Google Docs에 내용 초안 작성

API 설정 없이 Claude를 사용하여 문서에 직접 작성합니다.

```
Draft a project update based on the recent commits and add it to my
Google Doc at docs.google.com/document/d/abc123
```

Claude는 문서를 열고 에디터를 클릭하여 내용을 입력합니다. Gmail, Notion, Sheets 등 로그인된 모든 웹 앱에서 동작합니다.

### 웹 페이지에서 데이터 추출

웹사이트에서 구조화된 정보를 가져옵니다.

```
Go to the product listings page and extract the name, price, and
availability for each item. Save the results as a CSV file.
```

Claude는 페이지로 이동하여 내용을 읽고 데이터를 구조화된 형식으로 정리합니다.

### 다중 사이트 워크플로우 실행

여러 웹사이트에 걸친 작업을 조율합니다.

```
Check my calendar for meetings tomorrow, then for each meeting with
an external attendee, look up their company website and add a note
about what they do.
```

Claude는 여러 탭에서 작업하며 정보를 수집하고 워크플로우를 완료합니다.

### 데모 GIF 녹화

브라우저 상호작용의 공유 가능한 녹화본을 만듭니다.

```
Record a GIF showing how to complete the checkout flow, from adding
an item to the cart through to the confirmation page.
```

Claude는 상호작용 시퀀스를 녹화하고 GIF 파일로 저장합니다.

## 문제 해결

### Extension이 감지되지 않는 경우

Claude Code에 "Chrome extension not detected"가 표시되면:

1. `chrome://extensions`에서 Chrome extension이 설치되고 활성화되어 있는지 확인하세요.
2. `claude --version`을 실행하여 Claude Code가 최신 버전인지 확인하세요.
3. Chrome이 실행 중인지 확인하세요.
4. `/chrome`을 실행하고 "Reconnect extension"을 선택하여 연결을 다시 설정하세요.
5. 문제가 지속되면 Claude Code와 Chrome을 모두 재시작하세요.

Chrome 통합을 처음 활성화하면 Claude Code가 네이티브 메시징 호스트 구성 파일을 설치합니다. Chrome은 시작 시 이 파일을 읽으므로, 첫 번째 시도에서 extension이 감지되지 않으면 Chrome을 재시작하여 새 구성을 적용하세요.

연결이 여전히 실패하면 다음 위치에 호스트 구성 파일이 존재하는지 확인하세요.

Chrome의 경우:

* **macOS**: `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Linux**: `~/.config/google-chrome/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Windows**: Windows 레지스트리의 `HKCU\Software\Google\Chrome\NativeMessagingHosts\` 확인

Edge의 경우:

* **macOS**: `~/Library/Application Support/Microsoft Edge/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Linux**: `~/.config/microsoft-edge/NativeMessagingHosts/com.anthropic.claude_code_browser_extension.json`
* **Windows**: Windows 레지스트리의 `HKCU\Software\Microsoft\Edge\NativeMessagingHosts\` 확인

### 브라우저가 응답하지 않는 경우

Claude의 브라우저 명령이 작동하지 않으면:

1. 모달 다이얼로그(alert, confirm, prompt)가 페이지를 차단하고 있는지 확인하세요. JavaScript 다이얼로그는 브라우저 이벤트를 차단하여 Claude가 명령을 받을 수 없게 합니다. 다이얼로그를 수동으로 닫고 Claude에게 계속하라고 알려주세요.
2. Claude에게 새 탭을 열고 다시 시도하도록 요청하세요.
3. `chrome://extensions`에서 Chrome extension을 비활성화했다가 다시 활성화하여 재시작하세요.

### 장시간 세션 중 연결이 끊기는 경우

Chrome extension의 서비스 워커는 장시간 세션 중 유휴 상태가 될 수 있으며, 이로 인해 연결이 끊어집니다. 비활성 기간 후 브라우저 도구가 작동하지 않으면 `/chrome`을 실행하고 "Reconnect extension"을 선택하세요.

### Windows 전용 문제

Windows에서는 다음과 같은 문제가 발생할 수 있습니다.

* **명명된 파이프 충돌(EADDRINUSE)**: 다른 프로세스가 동일한 명명된 파이프를 사용하는 경우 Claude Code를 재시작하세요. Chrome을 사용 중인 다른 Claude Code 세션을 모두 닫으세요.
* **네이티브 메시징 호스트 오류**: 시작 시 네이티브 메시징 호스트가 충돌하면 Claude Code를 다시 설치하여 호스트 구성을 재생성해 보세요.

### 자주 발생하는 오류 메시지

가장 자주 발생하는 오류와 해결 방법입니다.

| 오류 | 원인 | 해결 방법 |
| ------------------------------------ | ------------------------------------------------ | --------------------------------------------------------------- |
| "Browser extension is not connected" | 네이티브 메시징 호스트가 extension에 도달할 수 없음 | Chrome과 Claude Code를 재시작한 후 `/chrome`을 실행하여 재연결 |
| "Extension not detected" | Chrome extension이 설치되지 않았거나 비활성화됨 | `chrome://extensions`에서 extension을 설치하거나 활성화 |
| "No tab available" | 탭이 준비되기 전에 Claude가 동작을 시도함 | Claude에게 새 탭을 만들고 재시도하도록 요청 |
| "Receiving end does not exist" | Extension 서비스 워커가 유휴 상태가 됨 | `/chrome`을 실행하고 "Reconnect extension" 선택 |

## 관련 항목

* VS Code에서 Claude Code 사용: VS Code extension의 브라우저 자동화
* CLI 레퍼런스: `--chrome`을 포함한 커맨드라인 플래그
* 일반적인 워크플로우: Claude Code를 활용하는 더 많은 방법
* 데이터 및 개인정보 보호: Claude Code의 데이터 처리 방식
* Claude in Chrome 시작 가이드: 단축키, 예약, 권한을 포함한 Chrome extension 전체 문서
