# VS Code에서 Claude Code 사용

> VS Code용 Claude Code 확장을 설치하고 구성합니다. 인라인 diff, @-언급, 계획 검토, 키보드 단축키로 AI 코딩 지원을 받으세요.

VS Code 확장 프로그램은 IDE에 직접 통합된 Claude Code의 네이티브 그래픽 인터페이스를 제공합니다. 이것이 VS Code에서 Claude Code를 사용하는 권장 방법입니다.

확장 프로그램을 사용하면 Claude의 계획을 수락하기 전에 검토하고 편집하고, 편집 내용이 이루어질 때 자동 수락하고, 선택 영역에서 특정 줄 범위로 파일을 @-언급하고, 대화 기록에 접근하고, 별도의 탭이나 창에서 여러 대화를 열 수 있습니다.

## 사전 요구 사항

* VS Code 1.98.0 이상
* Anthropic 계정 (확장 프로그램을 처음 열 때 로그인). Amazon Bedrock이나 Google Vertex AI와 같은 서드파티 공급자를 사용하는 경우 서드파티 공급자 사용을 참조하세요.

> 확장 프로그램에는 CLI (커맨드라인 인터페이스)가 포함되어 있으며, 고급 기능을 위해 VS Code의 통합 터미널에서 접근할 수 있습니다. 자세한 내용은 VS Code 확장 vs. Claude Code CLI를 참조하세요.

## 확장 프로그램 설치

IDE에 직접 설치하려면 링크를 클릭하세요:

* VS Code용 설치: `vscode:extension/anthropic.claude-code`
* Cursor용 설치: `cursor:extension/anthropic.claude-code`

또는 VS Code에서 `Cmd+Shift+X` (Mac) 또는 `Ctrl+Shift+X` (Windows/Linux)를 눌러 확장 프로그램 보기를 열고 "Claude Code"를 검색한 다음 **설치**를 클릭하세요.

> 설치 후 확장 프로그램이 나타나지 않으면 VS Code를 다시 시작하거나 명령 팔레트에서 "Developer: Reload Window"를 실행하세요.

## 시작하기

설치 후 VS Code 인터페이스를 통해 Claude Code를 사용할 수 있습니다:

### 1단계: Claude Code 패널 열기

VS Code 전체에서 스파크 아이콘은 Claude Code를 나타냅니다.

Claude를 여는 가장 빠른 방법은 **편집기 도구 모음** (편집기 오른쪽 상단)에서 스파크 아이콘을 클릭하는 것입니다. 이 아이콘은 파일이 열려 있을 때만 나타납니다.

Claude Code를 여는 다른 방법:

* **명령 팔레트**: `Cmd+Shift+P` (Mac) 또는 `Ctrl+Shift+P` (Windows/Linux), "Claude Code"를 입력하고 "Open in New Tab"과 같은 옵션 선택
* **상태 표시줄**: 창 오른쪽 하단의 **✱ Claude Code**를 클릭. 파일이 열려 있지 않아도 작동합니다.

패널을 처음 열면 **Claude Code 알아보기** 체크리스트가 표시됩니다. **보여주기**를 클릭하여 각 항목을 진행하거나 X로 닫으세요. 나중에 다시 열려면 확장 프로그램 → Claude Code의 VS Code 설정에서 **온보딩 숨기기**를 해제하세요.

Claude 패널을 VS Code의 어디든 끌어다 위치를 변경할 수 있습니다. 자세한 내용은 워크플로우 사용자 정의를 참조하세요.

### 2단계: 프롬프트 보내기

코드나 파일에 대해 도움을 요청하세요. 작동 방식 설명, 문제 디버깅, 변경 사항 적용 등 무엇이든 가능합니다.

> Claude는 선택한 텍스트를 자동으로 볼 수 있습니다. `Option+K` (Mac) / `Alt+K` (Windows/Linux)를 눌러 `@file.ts#5-10`과 같은 @-언급 참조를 프롬프트에 삽입하세요.

### 3단계: 변경 사항 검토

Claude가 파일을 편집하려 할 때 원본과 제안된 변경 사항의 나란히 비교를 보여주고 권한을 요청합니다. 수락, 거부하거나 대신 수행할 작업을 Claude에게 알릴 수 있습니다.

Claude Code로 할 수 있는 더 많은 아이디어는 일반 워크플로우를 참조하세요.

> 기본 사항에 대한 안내 투어를 보려면 명령 팔레트에서 "Claude Code: Open Walkthrough"를 실행하세요.

## 프롬프트 상자 사용

프롬프트 상자는 여러 기능을 지원합니다:

* **권한 모드**: 프롬프트 상자 하단의 모드 표시기를 클릭하여 모드를 전환합니다. 일반 모드에서 Claude는 각 작업 전에 권한을 요청합니다. Plan 모드에서 Claude는 변경하기 전에 수행할 작업을 설명하고 승인을 기다립니다. 자동 수락 모드에서 Claude는 묻지 않고 편집합니다. VS Code 설정의 `claudeCode.initialPermissionMode`에서 기본값을 설정합니다.
* **명령 메뉴**: `/`를 클릭하거나 입력하여 명령 메뉴를 엽니다. 파일 첨부, 모델 전환, 확장 사고 토글, 플랜 사용량 보기 (`/usage`) 등의 옵션이 있습니다. 사용자 정의 섹션에서 MCP 서버, 훅, 메모리, 권한, 플러그인에 접근할 수 있습니다. 터미널 아이콘이 있는 항목은 통합 터미널에서 열립니다.
* **컨텍스트 표시기**: 프롬프트 상자는 Claude의 컨텍스트 창 중 얼마나 사용 중인지 보여줍니다. Claude는 필요할 때 자동으로 압축하거나 `/compact`를 수동으로 실행할 수 있습니다.
* **확장 사고**: Claude가 복잡한 문제를 더 오래 추론할 수 있게 합니다. 명령 메뉴 (`/`)를 통해 켜세요. 자세한 내용은 확장 사고를 참조하세요.
* **멀티라인 입력**: 보내지 않고 새 줄을 추가하려면 `Shift+Enter`를 누르세요. 이는 질문 다이얼로그의 "기타" 자유 텍스트 입력에서도 작동합니다.

### 파일 및 폴더 참조

@-언급을 사용하여 특정 파일이나 폴더에 대한 컨텍스트를 Claude에게 제공합니다. 파일이나 폴더 이름 뒤에 `@`을 입력하면 Claude가 해당 내용을 읽고 질문에 답하거나 변경할 수 있습니다. Claude Code는 퍼지 매칭을 지원하여 부분 이름을 입력하여 원하는 것을 찾을 수 있습니다:

```
> Explain the logic in @auth (fuzzy matches auth.js, AuthService.ts, etc.)
> What's in @src/components/ (include a trailing slash for folders)
```

대용량 PDF의 경우 전체 파일 대신 특정 페이지를 읽도록 Claude에게 요청할 수 있습니다: 단일 페이지, 1-10페이지와 같은 범위, 또는 3페이지 이후와 같은 열린 범위.

편집기에서 텍스트를 선택하면 Claude가 강조 표시된 코드를 자동으로 볼 수 있습니다. 프롬프트 상자 푸터에 선택된 줄 수가 표시됩니다. `Option+K` (Mac) / `Alt+K` (Windows/Linux)를 눌러 파일 경로와 줄 번호가 포함된 @-언급 (예: `@app.ts#5-10`)을 삽입하세요. 선택 표시기를 클릭하여 Claude가 강조 표시된 텍스트를 볼 수 있는지 토글합니다 - 눈-슬래시 아이콘은 선택이 Claude에게 숨겨진 것을 의미합니다.

프롬프트 상자에 파일을 끌어다 놓으면서 `Shift`를 누르면 첨부 파일로 추가됩니다. 첨부 파일의 X를 클릭하면 컨텍스트에서 제거됩니다.

### 이전 대화 재개

Claude Code 패널 상단의 드롭다운을 클릭하여 대화 기록에 접근합니다. 키워드로 검색하거나 시간별로 탐색할 수 있습니다 (오늘, 어제, 지난 7일 등). 대화를 클릭하면 전체 메시지 기록과 함께 재개됩니다. 세션 재개에 대한 자세한 내용은 일반 워크플로우를 참조하세요.

### Claude.ai에서 원격 세션 재개

웹에서 Claude Code를 사용하는 경우 VS Code에서 해당 원격 세션을 직접 재개할 수 있습니다. 이를 위해 Anthropic Console이 아닌 **Claude.ai 구독**으로 로그인해야 합니다.

**단계:**

1. **이전 대화 열기**: Claude Code 패널 상단의 **이전 대화** 드롭다운을 클릭합니다.
2. **원격 탭 선택**: 다이얼로그에 두 탭이 표시됩니다: 로컬 및 원격. **원격**을 클릭하여 claude.ai의 세션을 봅니다.
3. **재개할 세션 선택**: 원격 세션을 탐색하거나 검색합니다. 세션을 클릭하면 다운로드하여 로컬에서 대화를 계속합니다.

> 원격 탭에는 GitHub 저장소로 시작된 웹 세션만 표시됩니다. 재개하면 대화 기록이 로컬로 로드되고 변경 사항은 claude.ai에 다시 동기화되지 않습니다.

## 워크플로우 사용자 정의

설치 후 Claude 패널을 위치 변경하거나, 여러 세션을 실행하거나, 터미널 모드로 전환할 수 있습니다.

### Claude의 위치 선택

Claude 패널을 VS Code의 어디든 끌어다 위치를 변경할 수 있습니다. 패널의 탭이나 제목 표시줄을 잡아 다음으로 끌어다 놓으세요:

* **보조 사이드바**: 창의 오른쪽. 코딩하는 동안 Claude가 표시됩니다.
* **기본 사이드바**: 탐색기, 검색 등의 아이콘이 있는 왼쪽 사이드바.
* **편집기 영역**: 파일과 함께 탭으로 Claude를 엽니다. 사이드 작업에 유용합니다.

> 주요 Claude 세션에는 사이드바를 사용하고 사이드 작업을 위해 추가 탭을 여세요. Claude는 선호하는 위치를 기억합니다. Claude가 기본적으로 오른쪽에 있기 때문에 활동 표시줄에 스파크 아이콘은 Claude 패널이 왼쪽에 도킹될 때만 나타납니다. 따라서 편집기 도구 모음 아이콘을 사용하여 Claude를 여세요.

### 여러 대화 실행

명령 팔레트에서 **Open in New Tab** 또는 **Open in New Window**를 사용하여 추가 대화를 시작합니다. 각 대화는 자체 기록과 컨텍스트를 유지하여 다양한 작업을 병렬로 처리할 수 있습니다.

탭을 사용할 때 스파크 아이콘의 작은 색점은 상태를 나타냅니다: 파란색은 권한 요청이 대기 중, 주황색은 탭이 숨겨진 동안 Claude가 완료된 것을 의미합니다.

### 터미널 모드로 전환

기본적으로 확장 프로그램은 그래픽 채팅 패널을 엽니다. CLI 스타일 인터페이스를 선호하는 경우 터미널 사용 설정을 열고 체크박스를 선택하세요.

VS Code 설정 (`Cmd+,` Mac 또는 `Ctrl+,` Windows/Linux)을 열고 확장 프로그램 → Claude Code로 이동하여 **터미널 사용**을 체크할 수도 있습니다.

## 플러그인 관리

VS Code 확장 프로그램은 플러그인 설치 및 관리를 위한 그래픽 인터페이스를 포함합니다. 프롬프트 상자에 `/plugins`를 입력하여 **플러그인 관리** 인터페이스를 엽니다.

### 플러그인 설치

플러그인 다이얼로그에는 두 탭이 표시됩니다: **플러그인**과 **마켓플레이스**.

플러그인 탭에서:

* **설치된 플러그인**이 상단에 활성화 또는 비활성화 토글 스위치와 함께 표시됩니다
* 구성된 마켓플레이스의 **사용 가능한 플러그인**이 아래에 표시됩니다
* 이름이나 설명으로 플러그인을 검색하여 필터링합니다
* 사용 가능한 플러그인에서 **설치**를 클릭합니다

플러그인을 설치할 때 설치 범위를 선택합니다:

* **사용자를 위해 설치**: 모든 프로젝트에서 사용 가능 (사용자 범위)
* **이 프로젝트용 설치**: 프로젝트 협업자와 공유 (프로젝트 범위)
* **로컬에 설치**: 이 저장소에서만 본인만 사용 (로컬 범위)

### 마켓플레이스 관리

**마켓플레이스** 탭으로 전환하여 플러그인 소스를 추가하거나 제거합니다:

* GitHub 저장소, URL, 또는 로컬 경로를 입력하여 새 마켓플레이스 추가
* 새로 고침 아이콘을 클릭하여 마켓플레이스의 플러그인 목록 업데이트
* 휴지통 아이콘을 클릭하여 마켓플레이스 제거

변경 후 배너가 표시되어 변경 사항을 적용하기 위해 Claude Code를 다시 시작하라는 메시지가 나타납니다.

> VS Code에서의 플러그인 관리는 내부적으로 동일한 CLI 명령어를 사용합니다. 확장 프로그램에서 구성한 플러그인과 마켓플레이스는 CLI에서도 사용 가능하며 그 반대도 마찬가지입니다.

플러그인 시스템에 대한 자세한 내용은 플러그인 및 플러그인 마켓플레이스를 참조하세요.

## Chrome으로 브라우저 작업 자동화

Claude를 Chrome 브라우저에 연결하여 VS Code를 벗어나지 않고 웹 앱을 테스트하고, 콘솔 로그로 디버그하고, 브라우저 워크플로우를 자동화하세요. 이를 위해 Claude in Chrome 확장 프로그램 버전 1.0.36 이상이 필요합니다.

프롬프트 상자에 `@browser`를 입력하고 Claude가 수행할 작업을 입력하세요:

```
@browser go to localhost:3000 and check the console for errors
```

첨부 파일 메뉴를 열어 새 탭 열기나 페이지 내용 읽기와 같은 특정 브라우저 도구를 선택할 수도 있습니다.

Claude는 브라우저 작업을 위해 새 탭을 열고 브라우저의 로그인 상태를 공유하므로 이미 로그인한 모든 사이트에 접근할 수 있습니다.

설정 지침, 전체 기능 목록, 문제 해결에 대해서는 Chrome과 함께 Claude Code 사용을 참조하세요.

## VS Code 명령어 및 단축키

명령 팔레트 (`Cmd+Shift+P` Mac 또는 `Ctrl+Shift+P` Windows/Linux)를 열고 "Claude Code"를 입력하여 Claude Code 확장 프로그램의 모든 사용 가능한 VS Code 명령어를 확인하세요.

일부 단축키는 "포커스"된 (키보드 입력을 받는) 패널에 따라 다릅니다. 커서가 코드 파일에 있으면 편집기가 포커스됩니다. 커서가 Claude의 프롬프트 상자에 있으면 Claude가 포커스됩니다. 이 둘 사이를 전환하려면 `Cmd+Esc` / `Ctrl+Esc`를 사용하세요.

> 이것들은 확장 프로그램을 제어하기 위한 VS Code 명령어입니다. 모든 내장 Claude Code 명령어가 확장 프로그램에서 사용 가능한 것은 아닙니다. 자세한 내용은 VS Code 확장 vs. Claude Code CLI를 참조하세요.

| 명령어                    | 단축키                                                 | 설명                                                                          |
| -------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| 입력 포커스                | `Cmd+Esc` (Mac) / `Ctrl+Esc` (Windows/Linux)             | 편집기와 Claude 간 포커스 토글                                               |
| 사이드바에서 열기           | -                                                        | 왼쪽 사이드바에서 Claude 열기                                                      |
| 터미널에서 열기           | -                                                        | 터미널 모드에서 Claude 열기                                                     |
| 새 탭에서 열기            | `Cmd+Shift+Esc` (Mac) / `Ctrl+Shift+Esc` (Windows/Linux) | 편집기 탭으로 새 대화 열기                                             |
| 새 창에서 열기         | -                                                        | 별도의 창에서 새 대화 열기                                                         |
| 새 대화           | `Cmd+N` (Mac) / `Ctrl+N` (Windows/Linux)                 | 새 대화 시작 (Claude가 포커스되어 있어야 함)                                             |
| @-언급 참조 삽입 | `Option+K` (Mac) / `Alt+K` (Windows/Linux)               | 현재 파일과 선택 영역에 대한 참조 삽입 (편집기가 포커스되어 있어야 함) |
| 로그 표시                  | -                                                        | 확장 프로그램 디버그 로그 보기                                                            |
| 로그아웃                     | -                                                        | Anthropic 계정에서 로그아웃                                                   |

## 설정 구성

확장 프로그램에는 두 가지 유형의 설정이 있습니다:

* **VS Code의 확장 프로그램 설정**: VS Code 내에서 확장 프로그램의 동작 제어. `Cmd+,` (Mac) 또는 `Ctrl+,` (Windows/Linux)로 열고 확장 프로그램 → Claude Code로 이동합니다. `/`를 입력하고 **일반 구성**을 선택하여 설정을 열 수도 있습니다.
* **`~/.claude/settings.json`의 Claude Code 설정**: 확장 프로그램과 CLI 간에 공유됩니다. 허용된 명령어, 환경 변수, 훅, MCP 서버에 사용합니다. 자세한 내용은 설정을 참조하세요.

> `"$schema": "https://json.schemastore.org/claude-code-settings.json"`을 `settings.json`에 추가하면 VS Code에서 모든 사용 가능한 설정에 대한 자동 완성 및 인라인 유효성 검사를 받을 수 있습니다.

### 확장 프로그램 설정

| 설정                           | 기본값   | 설명                                                                                           |
| --------------------------------- | --------- | ----------------------------------------------------------------------------------------------------- |
| `selectedModel`                   | `default` | 새 대화의 모델. `/model`로 세션별 변경.                                        |
| `useTerminal`                     | `false`   | 그래픽 패널 대신 터미널 모드에서 Claude 실행                                             |
| `initialPermissionMode`           | `default` | 승인 프롬프트 제어: `default` (매번 묻기), `plan`, `acceptEdits`, 또는 `bypassPermissions`   |
| `preferredLocation`               | `panel`   | Claude가 열리는 위치: `sidebar` (오른쪽) 또는 `panel` (새 탭)                                            |
| `autosave`                        | `true`    | Claude가 읽거나 쓰기 전에 파일 자동 저장                                                    |
| `useCtrlEnterToSend`              | `false`   | Enter 대신 Ctrl/Cmd+Enter로 프롬프트 전송                                                   |
| `enableNewConversationShortcut`   | `true`    | Cmd/Ctrl+N으로 새 대화 시작 활성화                                                         |
| `hideOnboarding`                  | `false`   | 온보딩 체크리스트 숨기기 (학사 모자 아이콘)                                                   |
| `respectGitIgnore`                | `true`    | 파일 검색에서 .gitignore 패턴 제외                                                        |
| `environmentVariables`            | `[]`      | Claude 프로세스의 환경 변수 설정. 공유 구성은 Claude Code 설정 사용. |
| `disableLoginPrompt`              | `false`   | 인증 프롬프트 건너뜀 (서드파티 공급자 설정용)                                         |
| `allowDangerouslySkipPermissions` | `false`   | 모든 권한 프롬프트 우회. **극도의 주의가 필요합니다.**                                          |
| `claudeProcessWrapper`            | -         | Claude 프로세스를 실행하는 데 사용되는 실행 파일 경로                                                     |

## VS Code 확장 vs. Claude Code CLI

Claude Code는 VS Code 확장 프로그램 (그래픽 패널)과 CLI (터미널의 커맨드라인 인터페이스) 모두로 사용 가능합니다. 일부 기능은 CLI에서만 사용 가능합니다. CLI 전용 기능이 필요하면 VS Code의 통합 터미널에서 `claude`를 실행하세요.

| 기능             | CLI                    | VS Code 확장 프로그램                        |
| ------------------- | ---------------------- | ---------------------------------------- |
| 명령어 및 스킬 | 모두                    | 일부 (`/`를 입력하여 사용 가능한 항목 확인)       |
| MCP 서버 구성   | 예                    | 아니요 (CLI로 구성, 확장 프로그램에서 사용) |
| 체크포인트         | 예                    | 예                                      |
| `!` bash 단축키   | 예                    | 아니요                                       |
| 탭 완성      | 예                    | 아니요                                       |

### 체크포인트로 되감기

VS Code 확장 프로그램은 체크포인팅을 지원하여 Claude의 파일 편집을 추적하고 이전 상태로 되감을 수 있습니다. 메시지 위에 마우스를 올려 되감기 버튼을 표시하고 세 가지 옵션 중 하나를 선택하세요:

* **여기서 대화 포크**: 모든 코드 변경을 유지하면서 이 메시지부터 새 대화 브랜치 시작
* **여기까지 코드 되감기**: 전체 대화 기록을 유지하면서 대화의 이 시점으로 파일 변경 사항 되돌리기
* **대화 포크 및 코드 되감기**: 새 대화 브랜치를 시작하고 파일 변경 사항을 이 시점으로 되돌리기

체크포인트 작동 방식과 제한 사항에 대한 자세한 내용은 체크포인팅을 참조하세요.

### VS Code에서 CLI 실행

VS Code에 머물면서 CLI를 사용하려면 통합 터미널 (Windows/Linux에서 `` Ctrl+` `` 또는 Mac에서 `` Cmd+` ``)을 열고 `claude`를 실행하세요. CLI는 diff 보기 및 진단 공유와 같은 기능을 위해 IDE와 자동으로 통합됩니다.

외부 터미널을 사용하는 경우 Claude Code 내에서 `/ide`를 실행하여 VS Code에 연결하세요.

### 확장 프로그램과 CLI 간 전환

확장 프로그램과 CLI는 동일한 대화 기록을 공유합니다. CLI에서 확장 프로그램 대화를 계속하려면 터미널에서 `claude --resume`을 실행하세요. 그러면 대화를 검색하고 선택할 수 있는 인터랙티브 선택기가 열립니다.

### 프롬프트에 터미널 출력 포함

`@terminal:name`을 사용하여 프롬프트에 터미널 출력을 참조하세요. 여기서 `name`은 터미널의 제목입니다. 이를 통해 Claude가 복사-붙여넣기 없이 명령어 출력, 오류 메시지, 로그를 볼 수 있습니다.

### 백그라운드 프로세스 모니터링

Claude가 장기 실행 명령어를 실행하면 확장 프로그램이 상태 표시줄에 진행 상황을 표시합니다. 그러나 백그라운드 작업에 대한 가시성은 CLI에 비해 제한됩니다. 더 나은 가시성을 위해 Claude가 명령어를 출력하도록 하여 VS Code의 통합 터미널에서 실행하세요.

### MCP로 외부 도구에 연결

MCP (Model Context Protocol) 서버는 Claude에게 외부 도구, 데이터베이스, API에 대한 접근을 제공합니다. CLI로 구성한 다음 확장 프로그램과 CLI 모두에서 사용하세요.

MCP 서버를 추가하려면 통합 터미널 (`` Ctrl+` `` 또는 `` Cmd+` ``)을 열고 실행하세요:

```bash
claude mcp add --transport http github https://api.githubcopilot.com/mcp/
```

구성되면 Claude에게 도구를 사용하도록 요청하세요 (예: "Review PR #456"). 일부 서버는 인증이 필요합니다: 터미널에서 `claude`를 실행한 다음 `/mcp`를 입력하여 인증하세요. 사용 가능한 서버는 MCP 문서를 참조하세요.

## Git 작업

Claude Code는 VS Code에서 직접 버전 관리 워크플로우를 도와주기 위해 git과 통합됩니다. Claude에게 변경 사항 커밋, pull request 생성, 또는 브랜치 작업을 요청하세요.

### 커밋 및 Pull Request 생성

Claude는 변경 사항을 스테이징하고, 커밋 메시지를 작성하고, 작업에 기반한 pull request를 생성할 수 있습니다:

```
> commit my changes with a descriptive message
> create a pr for this feature
> summarize the changes I've made to the auth module
```

Pull request를 생성할 때 Claude는 실제 코드 변경 사항을 기반으로 설명을 생성하고 테스트나 구현 결정에 대한 컨텍스트를 추가할 수 있습니다.

### 병렬 작업을 위한 Git Worktree 사용

Git worktree를 사용하면 여러 Claude Code 세션이 격리된 파일로 별도의 브랜치에서 동시에 작업할 수 있습니다:

```bash
# 새 기능을 위한 worktree 생성
git worktree add ../project-feature-a -b feature-a

# 각 worktree에서 Claude Code 실행
cd ../project-feature-a && claude
```

각 worktree는 git 기록을 공유하면서 독립적인 파일 상태를 유지합니다. 이는 Claude 인스턴스들이 다른 작업을 처리할 때 서로 간섭하지 않도록 합니다.

PR 검토 및 브랜치 관리를 포함한 자세한 git 워크플로우는 일반 워크플로우를 참조하세요.

## 서드파티 공급자 사용

기본적으로 Claude Code는 Anthropic의 API에 직접 연결됩니다. 조직에서 Amazon Bedrock, Google Vertex AI, 또는 Microsoft Foundry를 통해 Claude에 접근하는 경우 대신 공급자를 사용하도록 확장 프로그램을 구성하세요:

**단계:**

1. **로그인 프롬프트 비활성화**: 로그인 프롬프트 비활성화 설정을 열고 체크박스를 선택합니다. VS Code 설정 (`Cmd+,` Mac 또는 `Ctrl+,` Windows/Linux)을 열고 "Claude Code login"을 검색한 다음 **로그인 프롬프트 비활성화**를 체크할 수도 있습니다.
2. **공급자 구성**: 공급자의 설정 가이드를 따르세요:
   * Amazon Bedrock의 Claude Code
   * Google Vertex AI의 Claude Code
   * Microsoft Foundry의 Claude Code

   이 가이드에서는 `~/.claude/settings.json`에서 공급자를 구성하는 방법을 다루며, 설정이 VS Code 확장 프로그램과 CLI 간에 공유되도록 합니다.

## 보안 및 개인 정보

여러분의 코드는 비공개입니다. Claude Code는 지원을 제공하기 위해 코드를 처리하지만 모델 학습에 사용하지 않습니다. 데이터 처리 및 로깅을 해제하는 방법에 대한 자세한 내용은 데이터 및 개인 정보를 참조하세요.

자동 편집 권한이 활성화된 상태에서 Claude Code는 VS Code가 자동으로 실행할 수 있는 VS Code 구성 파일 (예: `settings.json` 또는 `tasks.json`)을 수정할 수 있습니다. 신뢰할 수 없는 코드로 작업할 때 위험을 줄이려면:

* 신뢰할 수 없는 워크스페이스에 VS Code 제한 모드 활성화
* 자동 수락 대신 수동 승인 모드 사용
* 변경 사항을 수락하기 전에 신중하게 검토

## 일반적인 문제 수정

### 확장 프로그램이 설치되지 않음

* VS Code의 호환 가능한 버전이 있는지 확인 (1.98.0 이상)
* VS Code에 확장 프로그램을 설치할 권한이 있는지 확인
* VS Code 마켓플레이스에서 직접 설치 시도

### 스파크 아이콘이 보이지 않음

스파크 아이콘은 파일이 열려 있을 때 **편집기 도구 모음** (편집기 오른쪽 상단)에 나타납니다. 보이지 않는 경우:

1. **파일 열기**: 아이콘에는 파일이 열려 있어야 합니다. 폴더만 열려 있어서는 충분하지 않습니다.
2. **VS Code 버전 확인**: 1.98.0 이상 필요 (도움말 → 정보)
3. **VS Code 다시 시작**: 명령 팔레트에서 "Developer: Reload Window" 실행
4. **충돌하는 확장 프로그램 비활성화**: 다른 AI 확장 프로그램 (Cline, Continue 등)을 일시적으로 비활성화
5. **워크스페이스 신뢰 확인**: 확장 프로그램은 제한 모드에서 작동하지 않습니다

또는 **상태 표시줄** (오른쪽 하단)에서 "✱ Claude Code"를 클릭하세요. 파일이 열려 있지 않아도 작동합니다. **명령 팔레트** (`Cmd+Shift+P` / `Ctrl+Shift+P`)를 사용하여 "Claude Code"를 입력할 수도 있습니다.

### Claude Code가 응답하지 않음

Claude Code가 프롬프트에 응답하지 않는 경우:

1. **인터넷 연결 확인**: 안정적인 인터넷 연결이 있는지 확인
2. **새 대화 시작**: 새 대화를 시작하여 문제가 지속되는지 확인
3. **CLI 시도**: 터미널에서 `claude`를 실행하여 더 자세한 오류 메시지 확인

문제가 지속되면 GitHub에 오류 내용과 함께 이슈를 제출하세요.

## 확장 프로그램 제거

Claude Code 확장 프로그램을 제거하려면:

1. 확장 프로그램 보기 열기 (`Cmd+Shift+X` Mac 또는 `Ctrl+Shift+X` Windows/Linux)
2. "Claude Code" 검색
3. **제거** 클릭

확장 프로그램 데이터도 제거하고 모든 설정을 초기화하려면:

```bash
rm -rf ~/.vscode/globalStorage/anthropic.claude-code
```

추가 도움은 문제 해결 가이드를 참조하세요.

## 다음 단계

VS Code에서 Claude Code를 설정했으니:

* Claude Code를 최대한 활용하기 위한 일반 워크플로우 탐색
* 외부 도구로 Claude의 기능을 확장하기 위한 MCP 서버 설정. CLI로 서버를 구성한 다음 확장 프로그램에서 사용하세요.
