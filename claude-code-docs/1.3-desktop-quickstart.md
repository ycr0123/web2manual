# 데스크탑 앱 시작하기

> 데스크탑에 Claude Code를 설치하고 첫 번째 코딩 세션을 시작합니다.

데스크탑 앱은 그래픽 인터페이스를 갖춘 Claude Code를 제공합니다: 시각적 diff 검토, Git worktree 격리를 통한 병렬 세션, 파일 첨부, 원격으로 오래 걸리는 작업 실행 기능. 로컬 파일 작업, SSH를 통한 원격 머신 연결, 클라우드에서의 세션 실행이 가능합니다. 터미널이 필요 없습니다.

데스크탑 앱에는 세 가지 탭이 있습니다:

- **Chat**: 파일 접근 없이 일반 대화, claude.ai와 유사합니다.
- **Cowork**: 자체 환경이 갖춰진 클라우드 VM에서 작업을 처리하는 자율 백그라운드 에이전트입니다. 다른 작업을 하는 동안에도 독립적으로 실행할 수 있습니다.
- **Code**: 로컬 파일에 직접 접근하는 인터랙티브 코딩 어시스턴트입니다. 실시간으로 각 변경사항을 검토하고 승인합니다.

> Claude Code를 사용하려면 Pro, Max, Teams, Enterprise 구독이 필요합니다.

## 설치

### 1단계: 앱 다운로드

플랫폼에 맞는 Claude를 다운로드하세요:

- **macOS**: Intel 및 Apple Silicon 범용 빌드
- **Windows**: x64 프로세서용

Windows ARM64의 경우 ARM64 버전을 다운로드하세요. ARM64 기기에서는 로컬 세션을 사용할 수 없으므로 원격 세션을 사용하세요.

현재 Linux는 지원되지 않습니다.

### 2단계: 로그인

macOS의 응용 프로그램 폴더 또는 Windows의 시작 메뉴에서 Claude를 실행합니다. Anthropic 계정으로 로그인하세요.

### 3단계: Code 탭 열기

상단 중앙의 **Code** 탭을 클릭합니다. Code를 클릭했을 때 업그레이드하라는 안내가 나오면 먼저 유료 플랜에 구독해야 합니다. 온라인에서 로그인하라는 안내가 나오면 로그인을 완료하고 앱을 재시작하세요. 403 오류가 발생하면 인증 문제 해결을 참고하세요.

## 첫 번째 세션 시작

Code 탭을 열었다면 프로젝트를 선택하고 Claude에게 작업을 지시합니다.

### 1단계: 환경 및 폴더 선택

**Local**을 선택하면 파일을 직접 사용하여 Claude를 자신의 머신에서 실행합니다. **폴더 선택**을 클릭하고 프로젝트 디렉토리를 선택합니다.

> 잘 알고 있는 작은 프로젝트부터 시작하세요. Claude Code가 무엇을 할 수 있는지 가장 빠르게 확인할 수 있습니다. 세션 격리를 위해 Git이 필요합니다. Git 없이는 동일한 폴더의 모든 세션이 같은 파일을 편집합니다.

다음도 선택할 수 있습니다:

- **Remote**: Anthropic의 클라우드 인프라에서 세션을 실행하며 앱을 닫아도 계속됩니다. 원격 세션은 웹의 Claude Code와 동일한 인프라를 사용합니다.
- **SSH**: SSH를 통해 원격 머신에 연결합니다 (자체 서버, 클라우드 VM, 개발 컨테이너). Claude는 해당 머신의 파일과 도구에 접근하여 원격 머신에서 실행됩니다.

### 2단계: 모델 선택

전송 버튼 옆의 드롭다운에서 모델을 선택합니다. 세션 시작 후에는 모델을 변경할 수 없습니다.

### 3단계: Claude에게 작업 지시

Claude가 무엇을 해야 하는지 입력하세요:

- `Find a TODO comment and fix it`
- `Add tests for the main function`
- `Create a CLAUDE.md with instructions for this codebase`

세션은 코드에 대한 Claude와의 대화입니다. 각 세션은 고유한 컨텍스트와 변경사항을 추적하므로 서로 간섭 없이 여러 작업을 수행할 수 있습니다.

### 4단계: 변경사항 검토 및 수락

기본적으로 Code 탭은 **Ask 모드**로 시작하며, Claude는 변경사항을 제안하고 적용하기 전에 승인을 기다립니다. 다음이 표시됩니다:

1. 각 파일에서 변경될 내용을 정확히 보여주는 diff 보기
2. 각 변경사항을 승인하거나 거부하는 수락/거부 버튼
3. Claude가 요청을 처리하는 동안 실시간 업데이트

변경사항을 거부하면 Claude는 다르게 진행하는 방법을 물어봅니다. 수락하기 전까지 파일은 수정되지 않습니다.

## 다음에는?

첫 번째 편집을 완료했습니다. 다음으로 시도해볼 것들입니다.

**중단하고 방향 전환하기.** 언제든지 Claude를 중단할 수 있습니다. 잘못된 방향으로 가고 있다면 중지 버튼을 클릭하거나 수정 사항을 입력하고 **Enter**를 누르세요. Claude는 하던 일을 멈추고 입력에 따라 조정합니다.

**Claude에게 더 많은 컨텍스트 제공하기.** 프롬프트 상자에 `@파일명`을 입력하여 특정 파일을 대화에 가져오거나, 첨부 버튼을 사용하여 이미지와 PDF를 첨부하거나, 파일을 프롬프트로 직접 드래그 앤 드롭하세요.

**반복 작업에 스킬 사용하기.** `/`를 입력하거나 **+** → **슬래시 명령어**를 클릭하여 내장 명령어, 커스텀 스킬, 플러그인 스킬을 탐색하세요. 스킬은 필요할 때 언제든지 호출할 수 있는 재사용 가능한 프롬프트입니다.

**커밋 전 변경사항 검토하기.** Claude가 파일을 편집한 후 `+12 -1` 표시기가 나타납니다. 클릭하면 diff 보기가 열리고 파일별로 수정 사항을 검토하며 특정 라인에 주석을 달 수 있습니다. Claude는 주석을 읽고 수정합니다.

**제어 수준 조정하기.** 권한 모드로 균형을 조정합니다:
- **Ask 모드** (기본값): 모든 편집 전에 승인 필요
- **Code 모드**: 더 빠른 반복을 위해 파일 편집 자동 수락
- **Plan 모드**: 파일을 건드리지 않고 Claude가 접근 방식을 계획하도록 함

**더 많은 기능을 위한 플러그인 추가하기.** 프롬프트 상자 옆의 **+** 버튼을 클릭하고 **플러그인**을 선택하여 스킬, 에이전트, MCP 서버 등을 추가하는 플러그인을 탐색하고 설치하세요.

**준비되면 확장하기.** 사이드바에서 병렬 세션을 열어 각각 자체 Git worktree에서 여러 작업을 동시에 처리하세요. 앱을 닫아도 계속되도록 오래 걸리는 작업을 클라우드로 보내세요.

## CLI에서 오셨나요?

데스크탑은 그래픽 인터페이스를 갖춘 CLI와 동일한 엔진으로 실행됩니다. 동일한 프로젝트에서 두 가지를 동시에 실행할 수 있으며, 구성 (CLAUDE.md 파일, MCP 서버, 훅, 스킬, 설정)을 공유합니다.

## 다음 단계

- **문제 해결**: 일반적인 오류 및 설정 문제 해결책
- **모범 사례**: 효과적인 프롬프트 작성과 Claude Code를 최대한 활용하는 팁
- **일반 워크플로우**: 디버깅, 리팩토링, 테스팅 등을 위한 튜토리얼
