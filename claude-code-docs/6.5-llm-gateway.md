# LLM 게이트웨이

> LLM 게이트웨이 솔루션과 함께 작동하도록 Claude Code를 구성하는 방법을 알아봅니다. 게이트웨이 요구 사항, 인증 구성, 모델 선택, 제공업체별 엔드포인트 설정을 다룹니다.

LLM 게이트웨이는 Claude Code와 모델 제공업체 사이에 중앙화된 프록시 레이어를 제공하며 종종 다음을 제공합니다:

* **중앙화된 인증** - API 키 관리의 단일 지점
* **사용량 추적** - 팀과 프로젝트 전반의 사용량 모니터링
* **비용 제어** - 예산 및 속도 제한 구현
* **감사 로깅** - 규정 준수를 위한 모든 모델 상호작용 추적
* **모델 라우팅** - 코드 변경 없이 제공업체 간 전환

## 게이트웨이 요구 사항

LLM 게이트웨이가 Claude Code와 함께 작동하려면 다음 요구 사항을 충족해야 합니다:

**API 형식**

게이트웨이는 클라이언트에게 다음 API 형식 중 하나 이상을 노출해야 합니다:

1. **Anthropic Messages**: `/v1/messages`, `/v1/messages/count_tokens`
   * 다음 요청 헤더를 전달해야 합니다: `anthropic-beta`, `anthropic-version`

2. **Bedrock InvokeModel**: `/invoke`, `/invoke-with-response-stream`
   * 다음 요청 본문 필드를 보존해야 합니다: `anthropic_beta`, `anthropic_version`

3. **Vertex rawPredict**: `:rawPredict`, `:streamRawPredict`, `/count-tokens:rawPredict`
   * 다음 요청 헤더를 전달해야 합니다: `anthropic-beta`, `anthropic-version`

헤더를 전달하지 않거나 본문 필드를 보존하지 않으면 기능이 저하되거나 Claude Code 기능을 사용할 수 없게 될 수 있습니다.

> Claude Code는 API 형식에 따라 활성화할 기능을 결정합니다. Bedrock 또는 Vertex와 함께 Anthropic Messages 형식을 사용하는 경우 환경 변수 `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1`을 설정해야 할 수 있습니다.

## 구성

### 모델 선택

기본적으로 Claude Code는 선택된 API 형식에 대한 표준 모델 이름을 사용합니다.

게이트웨이에서 커스텀 모델 이름을 구성한 경우 [모델 구성](/en/model-config)에 문서화된 환경 변수를 사용하여 커스텀 이름과 일치시키세요.

## LiteLLM 구성

> LiteLLM은 서드파티 프록시 서비스입니다. Anthropic은 LiteLLM의 보안 또는 기능을 보증하거나 유지 관리하거나 감사하지 않습니다. 이 가이드는 정보 제공 목적으로만 제공되며 최신 정보가 아닐 수 있습니다. 사용자의 재량에 따라 사용하세요.

### 사전 요구 사항

* 최신 버전으로 업데이트된 Claude Code
* 배포되고 접근 가능한 LiteLLM 프록시 서버
* 선택한 제공업체를 통한 Claude 모델 접근

### 기본 LiteLLM 설정

**Claude Code 구성:**

#### 인증 방법

##### 정적 API 키

고정 API 키를 사용하는 가장 간단한 방법:

```bash
# 환경에서 설정
export ANTHROPIC_AUTH_TOKEN=sk-litellm-static-key

# 또는 Claude Code 설정에서
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-litellm-static-key"
  }
}
```

이 값은 `Authorization` 헤더로 전송됩니다.

##### 헬퍼를 사용한 동적 API 키

순환 키 또는 사용자별 인증의 경우:

1. API 키 헬퍼 스크립트 만들기:

```bash
#!/bin/bash
# ~/bin/get-litellm-key.sh

# 예시: vault에서 키 가져오기
vault kv get -field=api_key secret/litellm/claude-code

# 예시: JWT 토큰 생성
jwt encode \
  --secret="${JWT_SECRET}" \
  --exp="+1h" \
  '{"user":"'${USER}'","team":"engineering"}'
```

2. 헬퍼를 사용하도록 Claude Code 설정 구성:

```json
{
  "apiKeyHelper": "~/bin/get-litellm-key.sh"
}
```

3. 토큰 새로 고침 간격 설정:

```bash
# 매 시간 새로 고침 (3600000 ms)
export CLAUDE_CODE_API_KEY_HELPER_TTL_MS=3600000
```

이 값은 `Authorization`과 `X-Api-Key` 헤더로 전송됩니다. `apiKeyHelper`는 `ANTHROPIC_AUTH_TOKEN` 또는 `ANTHROPIC_API_KEY`보다 우선순위가 낮습니다.

#### 통합 엔드포인트 (권장)

LiteLLM의 Anthropic 형식 엔드포인트 사용:

```bash
export ANTHROPIC_BASE_URL=https://litellm-server:4000
```

**패스스루 엔드포인트 대비 통합 엔드포인트의 이점:**

* 로드 밸런싱
* 폴백
* 비용 추적 및 사용자 추적에 대한 일관된 지원

#### 제공업체별 패스스루 엔드포인트 (대안)

##### LiteLLM을 통한 Claude API

```bash
export ANTHROPIC_BASE_URL=https://litellm-server:4000/anthropic
```

##### LiteLLM을 통한 Amazon Bedrock

```bash
export ANTHROPIC_BEDROCK_BASE_URL=https://litellm-server:4000/bedrock
export CLAUDE_CODE_SKIP_BEDROCK_AUTH=1
export CLAUDE_CODE_USE_BEDROCK=1
```

##### LiteLLM을 통한 Google Vertex AI

```bash
export ANTHROPIC_VERTEX_BASE_URL=https://litellm-server:4000/vertex_ai/v1
export ANTHROPIC_VERTEX_PROJECT_ID=your-gcp-project-id
export CLAUDE_CODE_SKIP_VERTEX_AUTH=1
export CLAUDE_CODE_USE_VERTEX=1
export CLOUD_ML_REGION=us-east5
```

더 자세한 정보는 [LiteLLM 문서](https://docs.litellm.ai/)를 참조하세요.
