# 인터랙티브 모드

> Claude Code 세션의 키보드 단축키, 입력 모드, 인터랙티브 기능에 대한 완전한 레퍼런스입니다.

## 키보드 단축키

> 키보드 단축키는 플랫폼과 터미널에 따라 다를 수 있습니다. 현재 환경에서 사용 가능한 단축키를 보려면 `?`를 누르세요.
>
> **macOS 사용자**: Option/Alt 키 단축키 (`Alt+B`, `Alt+F`, `Alt+Y`, `Alt+M`, `Alt+P`)는 터미널에서 Option을 Meta로 설정해야 합니다:
> - **iTerm2**: 설정 → 프로파일 → 키 → 왼쪽/오른쪽 Option 키를 "Esc+"로 설정
> - **Terminal.app**: 설정 → 프로파일 → 키보드 → "Option을 Meta 키로 사용" 체크
> - **VS Code**: 설정 → 프로파일 → 키 → 왼쪽/오른쪽 Option 키를 "Esc+"로 설정

| :------------------------------------------------- | :--------------------------------- | :----------------------------------------------------------------------------------------------------------------- |

| `Ctrl+O`                                           | 상세 출력 토글              | 자세한 도구 사용 및 실행 표시                                                                            |
| `Ctrl+R`                                           | 역방향 명령어 기록 검색     | 이전 명령어를 인터랙티브로 검색                                                                                     |
| `Ctrl+V` 또는 `Cmd+V` (iTerm2) 또는 `Alt+V` (Windows) | 클립보드에서 이미지 붙여넣기         | 이미지 또는 이미지 파일 경로 붙여넣기                                                                                           |
| `Ctrl+B`                                           | 실행 중인 작업 백그라운드 처리           | bash 명령어와 에이전트를 백그라운드로 전환. tmux 사용자는 두 번 누름                                                                       |
| `Ctrl+T`                                           | 작업 목록 토글                   | 터미널 상태 영역에서 작업 목록 표시 또는 숨기기                                                                             |
| `Left/Right arrows`                                | 다이얼로그 탭 순환          | 권한 다이얼로그와 메뉴의 탭 간 탐색                                                                              |
| `Up/Down arrows`                                   | 명령어 기록 탐색           | 이전 입력 불러오기                                                                                                    |
| `Esc` + `Esc`                                      | 되감기 또는 요약                | 이전 시점으로 코드 및/또는 대화 복원, 또는 선택한 메시지에서 요약                         |
| `Shift+Tab` 또는 `Alt+M`                             | 권한 모드 토글            | 자동 수락 모드, Plan Mode, 일반 모드 간 전환. 에이전트 팀 활성 시 Delegate Mode도 포함. |
| `Option+P` (macOS) 또는 `Alt+P` (Windows/Linux)     | 모델 전환                       | 프롬프트를 지우지 않고 모델 전환                                                                                         |
| `Option+T` (macOS) 또는 `Alt+T` (Windows/Linux)     | 확장 사고 토글           | 확장 사고 모드 활성화 또는 비활성화. 이 단축키를 활성화하려면 먼저 `/terminal-setup` 실행                     |

| :--------------- | :------------- | :------------------------------------------------------ |
| 빠른 이스케이프     | `\` + `Enter`  | 모든 터미널에서 작동                                  |

## 내장 명령어

내장 명령어는 일반적인 작업의 단축키입니다. 아래 표는 자주 사용되는 명령어를 다루지만 사용 가능한 모든 옵션을 포함하지는 않습니다. Claude Code에서 `/`를 입력하면 전체 목록을 볼 수 있으며, `/` 뒤에 문자를 입력하여 필터링할 수 있습니다.

`/`로 호출할 수 있는 커스텀 명령어를 만들려면 스킬을 참조하세요.

MCP 서버는 명령어로 표시되는 프롬프트를 노출할 수 있습니다. 이러한 프롬프트는 `/mcp__<server>__<prompt>` 형식을 사용하며 연결된 서버에서 동적으로 검색됩니다.

## 명령어 기록

Claude Code는 현재 세션의 명령어 기록을 유지합니다:

- 기록은 작업 디렉토리별로 저장됩니다
- `/clear` 명령어로 지워집니다
- 위/아래 화살표로 탐색합니다
- **참고**: 기록 확장 (`!`)은 기본적으로 비활성화됩니다

### Ctrl+R로 역방향 검색

`Ctrl+R`을 눌러 명령어 기록을 인터랙티브로 검색합니다:

1. **검색 시작**: `Ctrl+R`을 눌러 역방향 기록 검색 활성화
2. **쿼리 입력**: 이전 명령어에서 검색할 텍스트 입력 - 검색어가 일치하는 결과에서 강조 표시됩니다
3. **일치 항목 탐색**: `Ctrl+R`을 다시 눌러 오래된 일치 항목으로 순환
4. **일치 항목 수락**:
   - `Tab` 또는 `Esc`를 눌러 현재 일치 항목을 수락하고 계속 편집
   - `Enter`를 눌러 수락하고 즉시 명령어 실행
5. **검색 취소**:
   - `Ctrl+C`를 눌러 취소하고 원래 입력 복원
   - 빈 검색에서 `Backspace`를 눌러 취소

## 백그라운드 bash 명령어

Claude Code는 bash 명령어를 백그라운드에서 실행하는 기능을 지원하여 장기 실행 프로세스가 실행되는 동안 계속 작업할 수 있습니다.

### 백그라운드 처리 작동 방식

Claude Code가 명령어를 백그라운드에서 실행하면, 비동기적으로 명령어를 실행하고 즉시 백그라운드 작업 ID를 반환합니다. Claude Code는 명령어가 백그라운드에서 계속 실행되는 동안 새 프롬프트에 응답할 수 있습니다.

명령어를 백그라운드에서 실행하려면 다음 방법 중 하나를 사용하세요:

- Claude Code에게 명령어를 백그라운드에서 실행하도록 프롬프트
- `Ctrl+B`를 눌러 일반 Bash 도구 호출을 백그라운드로 전환. (tmux 사용자는 tmux의 prefix 키 때문에 `Ctrl+B`를 두 번 눌러야 합니다.)

**주요 기능:**

- 출력이 버퍼링되어 Claude가 TaskOutput 도구로 검색 가능
- 백그라운드 작업에는 추적 및 출력 검색을 위한 고유 ID 부여
- Claude Code 종료 시 백그라운드 작업이 자동으로 정리됨

모든 백그라운드 작업 기능을 비활성화하려면 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` 환경 변수를 `1`로 설정하세요.

**자주 백그라운드 처리되는 명령어:**

- 빌드 도구 (webpack, vite, make)
- 패키지 매니저 (npm, yarn, pnpm)
- 테스트 실행기 (jest, pytest)
- 개발 서버
- 장기 실행 프로세스 (docker, terraform)

### `!` 접두사를 사용한 Bash 모드

입력 앞에 `!`를 붙여 Claude를 거치지 않고 직접 bash 명령어를 실행합니다:

```bash
! npm test
! git status
! ls -la
```

Bash 모드:

- 명령어와 출력을 대화 컨텍스트에 추가
- 실시간 진행 상황 및 출력 표시
- 장기 실행 명령어에 동일한 `Ctrl+B` 백그라운드 처리 지원
- Claude가 명령어를 해석하거나 승인할 필요 없음
- 기록 기반 자동 완성 지원: 부분 명령어를 입력하고 **Tab**을 눌러 현재 프로젝트의 이전 `!` 명령어에서 완성

## 프롬프트 제안

세션을 처음 열면 시작에 도움을 주기 위해 프롬프트 입력창에 회색으로 표시된 예시 명령어가 나타납니다. Claude Code는 최근에 작업한 파일을 반영하도록 프로젝트의 git 기록에서 이를 선택합니다.

Claude가 응답한 후, 제안은 다단계 요청의 후속 단계나 워크플로우의 자연스러운 계속과 같이 대화 기록을 기반으로 계속 표시됩니다.

- **Tab**을 눌러 제안을 수락하거나, **Enter**를 눌러 수락 및 제출
- 입력을 시작하면 제안이 사라집니다

프롬프트 제안을 완전히 비활성화하려면 환경 변수를 설정하거나 `/config`에서 설정을 토글하세요:

```bash
export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

## 작업 목록

복잡한 다단계 작업을 처리할 때 Claude는 진행 상황을 추적하기 위한 작업 목록을 만듭니다. 작업은 터미널 상태 영역에 대기 중, 진행 중, 완료됨을 나타내는 표시기와 함께 나타납니다.

- `Ctrl+T`를 눌러 작업 목록 보기를 토글. 한 번에 최대 10개의 작업이 표시됩니다
- 모든 작업을 보거나 지우려면 Claude에게 직접 요청하세요: "show me all tasks" 또는 "clear all tasks"
- 작업은 컨텍스트 압축 간에 유지되어 Claude가 더 큰 프로젝트에서 체계적으로 유지
- 세션 간에 작업 목록을 공유하려면 `CLAUDE_CODE_TASK_LIST_ID`를 `~/.claude/tasks/`의 명명된 디렉토리에 설정:

```bash
CLAUDE_CODE_TASK_LIST_ID=my-project claude
```

- 이전 TODO 목록으로 되돌리려면 `CLAUDE_CODE_ENABLE_TASKS=false`로 설정하세요.

## PR 검토 상태

열린 pull request가 있는 브랜치에서 작업할 때, Claude Code는 푸터에 클릭 가능한 PR 링크를 표시합니다 (예: "PR #446"). 링크에는 검토 상태를 나타내는 색상 밑줄이 있습니다:

- 초록색: 승인됨
- 노란색: 검토 대기 중
- 빨간색: 변경 요청
- 회색: 초안
- 보라색: 병합됨

`Cmd+click` (Mac) 또는 `Ctrl+click` (Windows/Linux)으로 링크를 클릭하면 브라우저에서 pull request가 열립니다. 상태는 60초마다 자동으로 업데이트됩니다.

> PR 상태를 사용하려면 `gh` CLI가 설치되어 있고 인증되어 있어야 합니다 (`gh auth login`).
