# 마켓플레이스에서 빌드된 플러그인 검색 및 설치

> 마켓플레이스에서 플러그인을 찾아 설치하여 새 명령어, 에이전트, 기능으로 Claude Code를 확장합니다.

플러그인은 스킬, 에이전트, 훅, MCP 서버로 Claude Code를 확장합니다. 플러그인 마켓플레이스는 직접 만들지 않고도 이러한 확장 기능을 검색하고 설치하는 데 도움이 되는 카탈로그입니다.

## 마켓플레이스 작동 방식

마켓플레이스는 다른 사람이 만들어 공유한 플러그인의 카탈로그입니다. 마켓플레이스 사용은 두 단계로 이루어집니다:

1. **마켓플레이스 추가** - 사용 가능한 목록을 탐색할 수 있도록 카탈로그를 Claude Code에 등록합니다. 아직 플러그인이 설치되지 않습니다.
2. **개별 플러그인 설치** - 카탈로그를 탐색하고 원하는 플러그인을 설치합니다.

앱 스토어를 추가하는 것과 유사합니다: 스토어를 추가하면 컬렉션을 탐색할 수 있지만 다운로드할 앱은 개별적으로 선택합니다.

## 공식 Anthropic 마켓플레이스

공식 Anthropic 마켓플레이스(`claude-plugins-official`)는 Claude Code 시작 시 자동으로 사용 가능합니다. `/plugin`을 실행하고 **검색** 탭으로 이동하여 사용 가능한 항목을 탐색하세요.

공식 마켓플레이스에서 플러그인을 설치하려면:

```shell
/plugin install plugin-name@claude-plugins-official
```

공식 마켓플레이스에는 여러 카테고리의 플러그인이 포함됩니다:

### 코드 인텔리전스

코드 인텔리전스 플러그인은 Claude Code의 내장 LSP 도구를 활성화하여 Claude가 편집 직후 즉시 정의로 이동하고, 참조를 찾고, 유형 오류를 확인할 수 있게 합니다.

| 언어 | 플러그인 | 필요한 바이너리 |
| :--- | :--- | :--- |
| C/C++ | `clangd-lsp` | `clangd` |
| C# | `csharp-lsp` | `csharp-ls` |
| Go | `gopls-lsp` | `gopls` |
| Java | `jdtls-lsp` | `jdtls` |
| Kotlin | `kotlin-lsp` | `kotlin-language-server` |
| Lua | `lua-lsp` | `lua-language-server` |
| PHP | `php-lsp` | `intelephense` |
| Python | `pyright-lsp` | `pyright-langserver` |
| Rust | `rust-analyzer-lsp` | `rust-analyzer` |
| Swift | `swift-lsp` | `sourcekit-lsp` |
| TypeScript | `typescript-lsp` | `typescript-language-server` |

#### 코드 인텔리전스 플러그인으로 Claude가 얻는 것

코드 인텔리전스 플러그인이 설치되고 언어 서버 바이너리가 사용 가능하면, Claude는 두 가지 기능을 얻습니다:

* **자동 진단**: Claude가 파일을 편집할 때마다 언어 서버가 변경 사항을 분석하고 오류와 경고를 자동으로 보고합니다. Claude는 컴파일러나 린터를 실행할 필요 없이 유형 오류, 누락된 임포트, 구문 문제를 확인합니다.
* **코드 탐색**: Claude는 언어 서버를 사용하여 정의로 이동하고, 참조를 찾고, 호버 시 유형 정보를 가져오고, 심볼을 나열하고, 구현을 찾고, 호출 계층을 추적할 수 있습니다.

### 외부 통합

이러한 플러그인은 수동 설정 없이 Claude를 외부 서비스에 연결할 수 있도록 미리 구성된 MCP 서버를 번들로 제공합니다:

* **소스 제어**: `github`, `gitlab`
* **프로젝트 관리**: `atlassian` (Jira/Confluence), `asana`, `linear`, `notion`
* **디자인**: `figma`
* **인프라**: `vercel`, `firebase`, `supabase`
* **소통**: `slack`
* **모니터링**: `sentry`

### 개발 워크플로우

일반적인 개발 태스크를 위한 명령어와 에이전트를 추가하는 플러그인:

* **commit-commands**: 커밋, 푸시, PR 생성을 포함한 Git 커밋 워크플로우
* **pr-review-toolkit**: 풀 리퀘스트 검토를 위한 전문 에이전트
* **agent-sdk-dev**: Claude Agent SDK로 빌드하기 위한 도구
* **plugin-dev**: 자체 플러그인 만들기 도구킷

### 출력 스타일

Claude의 응답 방식 커스터마이즈:

* **explanatory-output-style**: 구현 선택에 대한 교육적 인사이트
* **learning-output-style**: 기술 향상을 위한 대화형 학습 모드

## 시도해 보기: 데모 마켓플레이스 추가

Anthropic은 예시 플러그인이 있는 데모 플러그인 마켓플레이스(`claude-code-plugins`)도 운영합니다.

**1단계: 마켓플레이스 추가**

```shell
/plugin marketplace add anthropics/claude-code
```

**2단계: 사용 가능한 플러그인 탐색**

`/plugin`을 실행하여 플러그인 관리자를 엽니다. 네 개의 탭이 있는 탭 인터페이스가 열립니다:

* **검색**: 모든 마켓플레이스에서 사용 가능한 플러그인 탐색
* **설치됨**: 설치된 플러그인 보기 및 관리
* **마켓플레이스**: 추가된 마켓플레이스 추가, 제거, 업데이트
* **오류**: 플러그인 로딩 오류 보기

**3단계: 플러그인 설치**

플러그인을 선택하여 세부 정보를 보고 설치 범위를 선택하세요:

* **사용자 범위**: 모든 프로젝트에서 본인만 설치
* **프로젝트 범위**: 이 저장소의 모든 협력자에게 설치
* **로컬 범위**: 이 저장소에서 본인만 설치

명령줄에서 직접 설치할 수도 있습니다:

```shell
/plugin install commit-commands@anthropics-claude-code
```

**4단계: 새 플러그인 사용**

설치 후 플러그인의 명령어를 즉시 사용할 수 있습니다. 플러그인 명령어는 플러그인 이름으로 네임스페이싱됩니다:

```shell
/commit-commands:commit
```

## 마켓플레이스 추가

`/plugin marketplace add` 명령을 사용하여 다양한 소스에서 마켓플레이스를 추가하세요.

**단축키**: `/plugin marketplace` 대신 `/plugin market`을, `remove` 대신 `rm`을 사용할 수 있습니다.

### GitHub에서 추가

```shell
/plugin marketplace add anthropics/claude-code
```

### 다른 Git 호스트에서 추가

HTTPS 사용:

```shell
/plugin marketplace add https://gitlab.com/company/plugins.git
```

SSH 사용:

```shell
/plugin marketplace add git@gitlab.com:company/plugins.git
```

특정 브랜치나 태그를 추가하려면:

```shell
/plugin marketplace add https://gitlab.com/company/plugins.git#v1.0.0
```

### 로컬 경로에서 추가

```shell
/plugin marketplace add ./my-marketplace
```

`marketplace.json` 파일의 직접 경로를 추가할 수도 있습니다:

```shell
/plugin marketplace add ./path/to/marketplace.json
```

### 원격 URL에서 추가

```shell
/plugin marketplace add https://example.com/marketplace.json
```

## 플러그인 설치

마켓플레이스를 추가한 후 플러그인을 설치할 수 있습니다:

```shell
/plugin install plugin-name@marketplace-name
```

> 플러그인을 설치하기 전에 신뢰할 수 있는지 확인하세요. Anthropic은 플러그인에 포함된 MCP 서버, 파일, 또는 기타 소프트웨어를 제어하지 않으며 의도대로 작동하는지 확인할 수 없습니다.

## 설치된 플러그인 관리

`/plugin`을 실행하고 **설치됨** 탭으로 이동하여 플러그인을 보고, 활성화하고, 비활성화하거나 제거하세요.

제거하지 않고 플러그인 비활성화:

```shell
/plugin disable plugin-name@marketplace-name
```

비활성화된 플러그인 다시 활성화:

```shell
/plugin enable plugin-name@marketplace-name
```

플러그인 완전히 제거:

```shell
/plugin uninstall plugin-name@marketplace-name
```

`--scope` 옵션으로 CLI 명령에서 특정 범위를 대상으로 지정할 수 있습니다:

```shell
claude plugin install formatter@your-org --scope project
claude plugin uninstall formatter@your-org --scope project
```

## 마켓플레이스 관리

### 대화형 인터페이스 사용

`/plugin`을 실행하고 **마켓플레이스** 탭으로 이동하여:

* 소스와 상태가 포함된 추가된 마켓플레이스 보기
* 새 마켓플레이스 추가
* 마켓플레이스 목록을 업데이트하여 최신 플러그인 가져오기
* 더 이상 필요하지 않은 마켓플레이스 제거

### CLI 명령어 사용

구성된 모든 마켓플레이스 나열:

```shell
/plugin marketplace list
```

마켓플레이스에서 플러그인 목록 새로 고침:

```shell
/plugin marketplace update marketplace-name
```

마켓플레이스 제거:

```shell
/plugin marketplace remove marketplace-name
```

> 마켓플레이스를 제거하면 해당 마켓플레이스에서 설치한 플러그인도 제거됩니다.

### 자동 업데이트 구성

Claude Code는 시작 시 마켓플레이스와 설치된 플러그인을 자동으로 업데이트할 수 있습니다. UI를 통해 개별 마켓플레이스의 자동 업데이트를 전환하세요:

1. `/plugin`을 실행하여 플러그인 관리자를 엽니다
2. **마켓플레이스**를 선택합니다
3. 목록에서 마켓플레이스를 선택합니다
4. **자동 업데이트 활성화** 또는 **자동 업데이트 비활성화**를 선택합니다

공식 Anthropic 마켓플레이스는 기본적으로 자동 업데이트가 활성화됩니다. 서드파티 및 로컬 개발 마켓플레이스는 기본적으로 자동 업데이트가 비활성화됩니다.

Claude Code 자동 업데이트를 비활성화하면서 플러그인 자동 업데이트를 유지하려면:

```shell
export DISABLE_AUTOUPDATER=true
export FORCE_AUTOUPDATE_PLUGINS=true
```

## 팀 마켓플레이스 구성

팀 관리자는 `.claude/settings.json`에 마켓플레이스 구성을 추가하여 프로젝트에 대한 자동 마켓플레이스 설치를 설정할 수 있습니다. 팀원들이 저장소 폴더를 신뢰하면 Claude Code가 이러한 마켓플레이스와 플러그인을 설치하라는 메시지를 표시합니다.

## 문제 해결

### /plugin 명령어가 인식되지 않음

"알 수 없는 명령어"가 표시되거나 `/plugin` 명령어가 나타나지 않는 경우:

1. **버전 확인**: `claude --version`을 실행하세요. 플러그인은 버전 1.0.33 이상이 필요합니다.
2. **Claude Code 업데이트**:
   * **Homebrew**: `brew upgrade claude-code`
   * **npm**: `npm update -g @anthropic-ai/claude-code`
3. **Claude Code 재시작**: 업데이트 후 터미널을 재시작하고 `claude`를 다시 실행하세요.

### 일반적인 문제

* **마켓플레이스가 로드되지 않음**: URL이 접근 가능하고 해당 경로에 `.claude-plugin/marketplace.json`이 있는지 확인하세요
* **플러그인 설치 실패**: 플러그인 소스 URL이 접근 가능하고 저장소가 공개(또는 접근 권한이 있는)인지 확인하세요
* **설치 후 파일을 찾을 수 없음**: 플러그인이 캐시에 복사되므로 플러그인 디렉터리 외부의 파일을 참조하는 경로는 작동하지 않습니다
* **플러그인 스킬이 나타나지 않음**: `rm -rf ~/.claude/plugins/cache`로 캐시를 삭제하고, Claude Code를 재시작한 후 플러그인을 다시 설치하세요.

### 코드 인텔리전스 문제

* **언어 서버가 시작되지 않음**: 바이너리가 설치되어 `$PATH`에 있는지 확인하세요. `/plugin` 오류 탭에서 세부 정보를 확인하세요.
* **높은 메모리 사용량**: `rust-analyzer`와 `pyright`와 같은 언어 서버는 대규모 프로젝트에서 상당한 메모리를 소비할 수 있습니다.
* **모노레포에서 거짓 양성 진단**: 언어 서버는 작업 공간이 올바르게 구성되지 않은 경우 내부 패키지에 대해 해결되지 않은 임포트 오류를 보고할 수 있습니다.

## 다음 단계

* **자체 플러그인 빌드**: 스킬, 에이전트, 훅을 만들려면 플러그인을 참조하세요
* **마켓플레이스 만들기**: 팀이나 커뮤니티에 플러그인을 배포하려면 플러그인 마켓플레이스 만들기를 참조하세요
* **기술 레퍼런스**: 완전한 사양은 플러그인 레퍼런스를 참조하세요
