# JetBrains IDE

> IntelliJ, PyCharm, WebStorm 등 JetBrains IDE에서 Claude Code를 사용합니다.

Claude Code는 전용 플러그인을 통해 JetBrains IDE와 통합되어 인터랙티브 diff 보기, 선택 컨텍스트 공유 등의 기능을 제공합니다.

## 지원되는 IDE

Claude Code 플러그인은 대부분의 JetBrains IDE에서 작동합니다:

* IntelliJ IDEA
* PyCharm
* Android Studio
* WebStorm
* PhpStorm
* GoLand

## 기능

* **빠른 실행**: `Cmd+Esc` (Mac) 또는 `Ctrl+Esc` (Windows/Linux)로 편집기에서 직접 Claude Code를 열거나 UI의 Claude Code 버튼을 클릭
* **Diff 보기**: 코드 변경 사항을 터미널 대신 IDE의 diff 뷰어에서 직접 표시
* **선택 컨텍스트**: IDE의 현재 선택 영역/탭이 Claude Code와 자동으로 공유됨
* **파일 참조 단축키**: `Cmd+Option+K` (Mac) 또는 `Alt+Ctrl+K` (Linux/Windows)로 파일 참조 삽입 (예: @File#L1-99)
* **진단 공유**: IDE의 진단 오류 (린트, 문법 등)가 작업하는 동안 Claude와 자동으로 공유됨

## 설치

### 마켓플레이스 설치

JetBrains 마켓플레이스에서 Claude Code 플러그인을 찾아 설치하고 IDE를 재시작하세요.

아직 Claude Code를 설치하지 않았다면 설치 지침은 빠른 시작 가이드를 참조하세요.

> 플러그인 설치 후 효과가 나타나려면 IDE를 완전히 재시작해야 할 수 있습니다.

## 사용법

### IDE에서 사용

IDE의 통합 터미널에서 `claude`를 실행하면 모든 통합 기능이 활성화됩니다.

### 외부 터미널에서 사용

외부 터미널에서 `/ide` 명령어를 사용하여 Claude Code를 JetBrains IDE에 연결하고 모든 기능을 활성화하세요:

```bash
claude
> /ide
```

IDE 프로젝트 루트와 동일한 디렉토리에서 Claude Code를 시작하면 IDE와 동일한 파일에 접근할 수 있습니다.

## 구성

### Claude Code 설정

Claude Code 설정을 통해 IDE 통합을 구성하세요:

1. `claude` 실행
2. `/config` 명령어 입력
3. diff 도구를 `auto`로 설정하여 자동 IDE 감지

### 플러그인 설정

**설정 → 도구 → Claude Code [Beta]**로 이동하여 Claude Code 플러그인을 구성하세요:

#### 일반 설정

* **Claude 명령어**: Claude를 실행하는 커스텀 명령어 지정 (예: `claude`, `/usr/local/bin/claude`, 또는 `npx @anthropic/claude`)
* **Claude 명령어를 찾을 수 없는 경우 알림 억제**: Claude 명령어를 찾을 수 없다는 알림 건너뜀
* **멀티라인 프롬프트에 Option+Enter 사용 활성화** (macOS만): 활성화하면 Option+Enter가 Claude Code 프롬프트에서 새 줄을 삽입합니다. Option 키가 예기치 않게 캡처되는 문제가 있으면 비활성화하세요 (터미널 재시작 필요)
* **자동 업데이트 활성화**: 플러그인 업데이트를 자동으로 확인하고 설치합니다 (재시작 시 적용)

> WSL 사용자의 경우: Claude 명령어로 `wsl -d Ubuntu -- bash -lic "claude"`를 설정하세요 (`Ubuntu`를 사용 중인 WSL 배포판 이름으로 교체)

#### ESC 키 구성

JetBrains 터미널에서 ESC 키가 Claude Code 작업을 중단하지 않는 경우:

1. **설정 → 도구 → 터미널**로 이동
2. 다음 중 하나를 수행:
   * "Escape로 편집기에 포커스 이동" 체크 해제, 또는
   * "터미널 키 바인딩 구성"을 클릭하고 "편집기로 포커스 전환" 단축키 삭제
3. 변경 사항 적용

이렇게 하면 ESC 키가 Claude Code 작업을 올바르게 중단할 수 있습니다.

## 특별 구성

### 원격 개발

> JetBrains 원격 개발을 사용할 때 **설정 → 플러그인 (호스트)**를 통해 원격 호스트에 플러그인을 설치해야 합니다.

플러그인은 로컬 클라이언트 컴퓨터가 아닌 원격 호스트에 설치되어야 합니다.

### WSL 구성

> WSL 사용자는 IDE 감지가 올바르게 작동하기 위해 추가 구성이 필요할 수 있습니다. 자세한 설정 지침은 WSL 문제 해결 가이드를 참조하세요.

WSL 구성에는 다음이 필요할 수 있습니다:

* 올바른 터미널 구성
* 네트워킹 모드 조정
* 방화벽 설정 업데이트

## 문제 해결

### 플러그인이 작동하지 않음

* 프로젝트 루트 디렉토리에서 Claude Code를 실행하고 있는지 확인
* IDE 설정에서 JetBrains 플러그인이 활성화되어 있는지 확인
* IDE를 완전히 재시작 (여러 번 해야 할 수 있음)
* 원격 개발의 경우 원격 호스트에 플러그인이 설치되어 있는지 확인

### IDE가 감지되지 않음

* 플러그인이 설치되어 활성화되어 있는지 확인
* IDE를 완전히 재시작
* 통합 터미널에서 Claude Code를 실행하고 있는지 확인
* WSL 사용자의 경우 WSL 문제 해결 가이드 참조

### 명령어를 찾을 수 없음

Claude 아이콘을 클릭했을 때 "command not found"가 표시되는 경우:

1. Claude Code 설치 확인: `npm list -g @anthropic-ai/claude-code`
2. 플러그인 설정에서 Claude 명령어 경로 구성
3. WSL 사용자의 경우 구성 섹션에서 언급된 WSL 명령어 형식 사용

## 보안 고려 사항

자동 편집 권한이 활성화된 상태로 JetBrains IDE에서 Claude Code가 실행될 때, IDE가 자동으로 실행할 수 있는 IDE 구성 파일을 수정할 수 있습니다. 이는 자동 편집 모드에서 Claude Code를 실행하는 위험을 증가시키고 bash 실행에 대한 Claude Code의 권한 프롬프트를 우회할 수 있습니다.

JetBrains IDE에서 실행할 때는 다음을 고려하세요:

* 편집에 수동 승인 모드 사용
* Claude가 신뢰할 수 있는 프롬프트로만 사용되도록 각별히 주의
* Claude Code가 수정할 수 있는 파일 인식

추가 도움은 문제 해결 가이드를 참조하세요.
