# Claude 메모리 관리

> 다양한 메모리 위치와 모범 사례를 통해 세션 간 Claude Code의 메모리를 관리하는 방법을 알아봅니다.

Claude Code에는 세션 간에 유지되는 두 가지 종류의 메모리가 있습니다:

- **자동 메모리**: Claude는 프로젝트 패턴, 주요 명령어, 사용자 선호도와 같은 유용한 컨텍스트를 자동으로 저장합니다. 세션 간에 유지됩니다.
- **CLAUDE.md 파일**: 여러분이 작성하고 관리하는 마크다운 파일로, Claude가 따를 지시, 규칙, 선호도가 담겨 있습니다.

두 가지 모두 매 세션 시작 시 Claude의 컨텍스트에 로드되지만, 자동 메모리는 주 파일의 처음 200줄만 로드합니다.

## 메모리 유형 결정

Claude Code는 각기 다른 목적을 가진 계층적 구조의 여러 메모리 위치를 제공합니다:

| 메모리 유형                | 위치                                                                                                                                                            | 목적                                             | 사용 사례 예시                                                    | 공유 대상                     |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------- | ------------------------------- |
| **관리 정책**         | macOS: `/Library/Application Support/ClaudeCode/CLAUDE.md`<br />Linux: `/etc/claude-code/CLAUDE.md`<br />Windows: `C:\Program Files\ClaudeCode\CLAUDE.md` | IT/DevOps가 관리하는 조직 전체 지시 | 회사 코딩 표준, 보안 정책, 규정 준수 요구 사항 | 조직의 모든 사용자       |
| **프로젝트 메모리**         | `./CLAUDE.md` 또는 `./.claude/CLAUDE.md`                                                                                                                          | 프로젝트에 대한 팀 공유 지시            | 프로젝트 아키텍처, 코딩 표준, 일반 워크플로우             | 소스 제어를 통한 팀원 |
| **프로젝트 규칙**          | `./.claude/rules/*.md`                                                                                                                                          | 모듈식, 주제별 프로젝트 지시        | 언어별 가이드라인, 테스트 관행, API 표준     | 소스 제어를 통한 팀원 |
| **사용자 메모리**            | `~/.claude/CLAUDE.md`                                                                                                                                           | 모든 프로젝트에 대한 개인 선호도               | 코드 스타일 선호도, 개인 도구 단축키                 | 본인만 (모든 프로젝트)         |
| **프로젝트 메모리 (로컬)** | `./CLAUDE.local.md`                                                                                                                                             | 개인 프로젝트별 선호도               | 샌드박스 URL, 선호하는 테스트 데이터                               | 본인만 (현재 프로젝트)      |
| **자동 메모리**            | `~/.claude/projects/<project>/memory/`                                                                                                                          | Claude의 자동 노트 및 학습              | 프로젝트 패턴, 디버깅 인사이트, 아키텍처 노트             | 본인만 (프로젝트별)          |

작업 디렉토리 위의 디렉토리 계층에 있는 CLAUDE.md 파일은 시작 시 전체 로드됩니다. 하위 디렉토리의 CLAUDE.md 파일은 Claude가 해당 디렉토리의 파일을 읽을 때 필요에 따라 로드됩니다. 자동 메모리는 `MEMORY.md`의 처음 200줄만 로드합니다. 더 구체적인 지시가 더 광범위한 지시보다 우선합니다.

> CLAUDE.local.md 파일은 자동으로 .gitignore에 추가되어 버전 관리에 체크인되어서는 안 되는 비공개 프로젝트별 선호도에 이상적입니다.

## 자동 메모리

자동 메모리는 Claude가 작업하면서 학습 내용, 패턴, 인사이트를 기록하는 영구 디렉토리입니다. Claude가 세션에서 발견한 것을 바탕으로 스스로를 위해 작성하는 노트를 포함합니다. Claude가 따를 지시를 여러분이 작성하는 CLAUDE.md 파일과는 달리, 자동 메모리에는 Claude가 스스로를 위해 작성한 노트가 포함됩니다.

> 자동 메모리는 점진적으로 배포되고 있습니다. 자동 메모리가 보이지 않으면 환경에서 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=0`을 설정하여 옵트인할 수 있습니다.

### Claude가 기억하는 것

Claude가 작업하면서 다음과 같은 것들을 저장할 수 있습니다:

- 프로젝트 패턴: 빌드 명령어, 테스트 관행, 코드 스타일 선호도
- 디버깅 인사이트: 까다로운 문제의 해결책, 일반적인 오류 원인
- 아키텍처 노트: 주요 파일, 모듈 관계, 중요한 추상화
- 사용자 선호도: 커뮤니케이션 스타일, 워크플로우 습관, 도구 선택

### 자동 메모리 저장 위치

각 프로젝트는 `~/.claude/projects/<project>/memory/`에 자체 메모리 디렉토리를 가집니다. `<project>` 경로는 git 저장소 루트에서 파생되므로 동일한 저장소 내의 모든 하위 디렉토리가 하나의 자동 메모리 디렉토리를 공유합니다. Git worktree는 별도의 메모리 디렉토리를 가집니다. git 저장소 외부에서는 작업 디렉토리가 대신 사용됩니다.

디렉토리에는 `MEMORY.md` 엔트리포인트와 선택적 주제 파일이 포함됩니다:

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          # 간결한 인덱스, 모든 세션에 로드됨
├── debugging.md       # 디버깅 패턴에 대한 상세 노트
├── api-conventions.md # API 설계 결정
└── ...                # Claude가 생성하는 기타 주제 파일
```

`MEMORY.md`는 메모리 디렉토리의 인덱스 역할을 합니다. Claude는 세션 동안 이 디렉토리의 파일을 읽고 씁니다.

### 작동 방식

- `MEMORY.md`의 처음 200줄은 매 세션 시작 시 Claude의 시스템 프롬프트에 로드됩니다. 200줄 이후의 내용은 자동으로 로드되지 않습니다.
- `debugging.md` 또는 `patterns.md`와 같은 주제 파일은 시작 시 로드되지 않습니다. Claude는 정보가 필요할 때 표준 파일 도구를 사용하여 필요에 따라 읽습니다.
- Claude는 세션 동안 메모리 파일을 읽고 쓰므로, 작업하면서 메모리 업데이트가 발생하는 것을 볼 수 있습니다.

### 자동 메모리 관리

자동 메모리 파일은 언제든지 편집할 수 있는 마크다운 파일입니다. `/memory`를 사용하여 파일 선택기를 열면 CLAUDE.md 파일과 함께 자동 메모리 엔트리포인트가 포함됩니다.

Claude에게 특정 내용을 저장하도록 요청하려면 직접 말하세요: "remember that we use pnpm, not npm" 또는 "save to memory that the API tests require a local Redis instance".

```bash
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=1  # 강제 비활성화
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=0  # 강제 활성화
```

## CLAUDE.md 가져오기

CLAUDE.md 파일은 `@path/to/import` 구문을 사용하여 추가 파일을 가져올 수 있습니다. 다음 예시는 3개의 파일을 가져옵니다:

```
See @README for project overview and @package.json for available npm commands for this project.

# Additional Instructions
- git workflow @docs/git-instructions.md
```

상대 경로와 절대 경로 모두 허용됩니다. 상대 경로는 작업 디렉토리가 아닌 가져오기를 포함하는 파일을 기준으로 해석됩니다. 버전 관리에 체크인되어서는 안 되는 비공개 프로젝트별 선호도를 위해서는 `CLAUDE.local.md`를 선호하세요: 자동으로 로드되고 `.gitignore`에 추가됩니다.

여러 git worktree에서 작업하는 경우, `CLAUDE.local.md`는 하나의 worktree에만 존재합니다. 대신 홈 디렉토리 가져오기를 사용하여 모든 worktree가 동일한 개인 지시를 공유하도록 하세요:

```
# Individual Preferences
- @~/.claude/my-project-instructions.md
```

> Claude Code가 처음으로 프로젝트에서 외부 가져오기를 만나면 특정 파일 목록이 포함된 승인 다이얼로그를 표시합니다. 승인하면 로드되고, 거부하면 건너뜁니다. 프로젝트당 한 번의 결정입니다: 거부하면 다이얼로그가 다시 나타나지 않고 가져오기가 비활성화됩니다.

충돌을 피하기 위해, 가져오기는 마크다운 코드 범위와 코드 블록 내에서는 평가되지 않습니다.

```
이 코드 범위는 가져오기로 처리되지 않습니다: `@anthropic-ai/claude-code`
```

가져온 파일은 재귀적으로 추가 파일을 가져올 수 있으며, 최대 깊이는 5홉입니다. `/memory` 명령어를 실행하여 로드된 메모리 파일을 확인할 수 있습니다.

## Claude의 메모리 조회 방식

Claude Code는 메모리를 재귀적으로 읽습니다: cwd에서 시작하여 루트 디렉토리 `/`까지 (포함하지 않고) 재귀적으로 올라가며 CLAUDE.md 또는 CLAUDE.local.md 파일을 읽습니다. 이는 특히 `foo/bar/`에서 Claude Code를 실행하고 `foo/CLAUDE.md`와 `foo/bar/CLAUDE.md` 모두에 메모리가 있는 대규모 저장소에서 편리합니다.

Claude는 현재 작업 디렉토리 아래의 하위 트리에 중첩된 CLAUDE.md도 발견합니다. 시작 시 로드하는 대신, Claude가 해당 하위 트리의 파일을 읽을 때만 포함됩니다.

### 추가 디렉토리에서 메모리 로드

`--add-dir` 플래그는 Claude에게 주 작업 디렉토리 외부의 추가 디렉토리에 대한 접근을 제공합니다. 기본적으로 이러한 디렉토리의 CLAUDE.md 파일은 로드되지 않습니다.

추가 디렉토리에서도 메모리 파일 (CLAUDE.md, .claude/CLAUDE.md, .claude/rules/\*.md)을 로드하려면 `CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD` 환경 변수를 설정하세요:

```bash
CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD=1 claude --add-dir ../shared-config
```

## `/memory`로 메모리 직접 편집

세션 중에 `/memory` 명령어를 사용하여 더 많은 추가나 구성을 위해 시스템 편집기에서 메모리 파일을 열 수 있습니다.

## 프로젝트 메모리 설정

중요한 프로젝트 정보, 관행, 자주 사용하는 명령어를 저장하기 위한 CLAUDE.md 파일을 설정하려고 한다고 가정합니다. 프로젝트 메모리는 `./CLAUDE.md` 또는 `./.claude/CLAUDE.md`에 저장할 수 있습니다.

다음 명령어로 코드베이스에 대한 CLAUDE.md를 부트스트랩하세요:

```
> /init
```

팁:
- 반복적인 검색을 피하기 위해 자주 사용하는 명령어 (빌드, 테스트, 린트)를 포함하세요
- 코드 스타일 선호도와 명명 관행을 문서화하세요
- 프로젝트에 특정한 중요한 아키텍처 패턴을 추가하세요
- CLAUDE.md 메모리는 팀과 공유하는 지시와 개인 선호도 모두에 사용할 수 있습니다.

## `.claude/rules/`를 사용한 모듈식 규칙

대규모 프로젝트의 경우, `.claude/rules/` 디렉토리를 사용하여 지시를 여러 파일로 구성할 수 있습니다. 이를 통해 팀이 하나의 큰 CLAUDE.md 대신 집중적이고 잘 정리된 규칙 파일을 유지할 수 있습니다.

### 기본 구조

프로젝트의 `.claude/rules/` 디렉토리에 마크다운 파일을 배치하세요:

```
your-project/
├── .claude/
│   ├── CLAUDE.md           # 주요 프로젝트 지시
│   └── rules/
│       ├── code-style.md   # 코드 스타일 가이드라인
│       ├── testing.md      # 테스트 관행
│       └── security.md     # 보안 요구 사항
```

`.claude/rules/`의 모든 `.md` 파일은 `.claude/CLAUDE.md`와 동일한 우선순위로 프로젝트 메모리로 자동 로드됩니다.

### 경로별 규칙

규칙은 YAML frontmatter의 `paths` 필드를 사용하여 특정 파일로 범위를 지정할 수 있습니다. 이러한 조건부 규칙은 Claude가 지정된 패턴과 일치하는 파일로 작업할 때만 적용됩니다.

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API 개발 규칙

- 모든 API 엔드포인트에 입력 유효성 검사 포함
- 표준 오류 응답 형식 사용
- OpenAPI 문서 주석 포함
```

`paths` 필드가 없는 규칙은 조건 없이 로드되어 모든 파일에 적용됩니다.

### 글로브 패턴

`paths` 필드는 표준 글로브 패턴을 지원합니다:

| 패턴                | 일치 항목                                  |
| ---------------------- | ---------------------------------------- |
| `**/*.ts`              | 모든 디렉토리의 TypeScript 파일    |
| `src/**/*`             | `src/` 디렉토리 아래의 모든 파일         |
| `*.md`                 | 프로젝트 루트의 마크다운 파일       |
| `src/components/*.tsx` | 특정 디렉토리의 React 컴포넌트 |

여러 패턴을 지정할 수 있습니다:

```markdown
---
paths:
  - "src/**/*.ts"
  - "lib/**/*.ts"
  - "tests/**/*.test.ts"
---
```

중괄호 확장이 지원됩니다:

```markdown
---
paths:
  - "src/**/*.{ts,tsx}"
  - "{src,lib}/**/*.ts"
---
```

### 하위 디렉토리

규칙을 더 나은 구조를 위해 하위 디렉토리로 구성할 수 있습니다:

```
.claude/rules/
├── frontend/
│   ├── react.md
│   └── styles.md
├── backend/
│   ├── api.md
│   └── database.md
└── general.md
```

모든 `.md` 파일이 재귀적으로 검색됩니다.

### 심볼릭 링크

`.claude/rules/` 디렉토리는 심볼릭 링크를 지원하여 여러 프로젝트 간에 공통 규칙을 공유할 수 있습니다:

```bash
# 공유 규칙 디렉토리 심볼릭 링크
ln -s ~/shared-claude-rules .claude/rules/shared

# 개별 규칙 파일 심볼릭 링크
ln -s ~/company-standards/security.md .claude/rules/security.md
```

심볼릭 링크는 해석되어 내용이 정상적으로 로드됩니다. 순환 심볼릭 링크는 감지되어 정상적으로 처리됩니다.

### 사용자 수준 규칙

`~/.claude/rules/`에 모든 프로젝트에 적용되는 개인 규칙을 만들 수 있습니다:

```
~/.claude/rules/
├── preferences.md    # 개인 코딩 선호도
└── workflows.md      # 선호하는 워크플로우
```
