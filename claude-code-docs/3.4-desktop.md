# Claude Code 데스크탑 사용

> Git 격리를 통한 병렬 세션, 시각적 diff 검토, 권한 모드, 커넥터, 엔터프라이즈 구성 등 Claude Code 데스크탑을 더 잘 활용하세요.

데스크탑 앱의 코드 탭에서 터미널 대신 그래픽 인터페이스를 통해 Claude Code를 사용할 수 있습니다. 인라인 댓글이 있는 시각적 diff 검토, Claude가 자율적으로 수행할 작업을 제어하는 권한 모드, 자동 Git 격리가 있는 병렬 세션, GitHub, Slack, Linear와 같은 도구를 통합하는 커넥터가 제공됩니다. 세션은 로컬, SSH를 통한 원격 머신, 또는 클라우드에서 실행할 수 있습니다.

> 데스크탑이 처음이신가요? 앱 설치 및 첫 번째 편집을 위해 시작하기를 참조하세요.

이 페이지는 코드 작업, 세션 관리, Claude Code 확장, 구성을 다룹니다. CLI 비교 및 문제 해결도 포함되어 있습니다.

## 세션 시작

첫 번째 메시지를 보내기 전에 프롬프트 영역에서 네 가지를 구성하세요:

* **환경**: Claude가 실행되는 위치 선택. 로컬 머신은 **로컬**, Anthropic 호스팅 세션은 **클라우드 환경**, 사용자가 관리하는 원격 머신은 **SSH 연결** 선택. 환경 구성 참조.
* **프로젝트 폴더**: Claude가 작업할 폴더 또는 저장소 선택. 원격 세션의 경우 여러 저장소 추가 가능.
* **모델**: 전송 버튼 옆 드롭다운에서 모델 선택. 세션이 시작되면 모델이 잠깁니다.
* **권한 모드**: 모드 선택기에서 Claude의 자율성 정도 선택. 세션 중에 변경 가능.

작업을 입력하고 **Enter**를 눌러 시작합니다. 각 세션은 독립적으로 컨텍스트와 변경 사항을 추적합니다.

## 코드 작업

Claude에게 적절한 컨텍스트를 제공하고, 자율 수행 정도를 제어하고, 변경 사항을 검토하세요.

### 프롬프트 상자 사용

Claude에게 수행할 작업을 입력하고 **Enter**를 눌러 전송합니다. Claude는 권한 모드에 따라 프로젝트 파일을 읽고, 변경하고, 명령어를 실행합니다. 언제든지 Claude를 중단할 수 있습니다: 중지 버튼을 클릭하거나 수정 사항을 입력하고 **Enter**를 누르세요. Claude는 하던 작업을 멈추고 입력에 따라 조정합니다.

프롬프트 상자 옆의 **+** 버튼으로 파일 첨부, 스킬, 커넥터, 플러그인에 접근할 수 있습니다.

### 프롬프트에 파일 및 컨텍스트 추가

프롬프트 상자는 외부 컨텍스트를 가져오는 두 가지 방법을 지원합니다:

* **@언급 파일**: `@`를 입력하고 파일 이름을 입력하여 대화 컨텍스트에 파일을 추가합니다. Claude가 해당 파일을 읽고 참조할 수 있습니다.
* **파일 첨부**: 첨부 버튼을 사용하거나 프롬프트에 직접 파일을 끌어다 놓아 이미지, PDF, 기타 파일을 프롬프트에 첨부합니다. 버그 스크린샷, 디자인 목업, 참조 문서를 공유하는 데 유용합니다.

### 권한 모드 선택

권한 모드는 세션 중 Claude의 자율성을 제어합니다: 파일 편집, 명령어 실행, 또는 둘 다 수행 전에 요청할지 여부. 전송 버튼 옆의 모드 선택기를 사용하여 언제든지 모드를 전환할 수 있습니다. Ask 모드로 시작하여 Claude의 작업을 정확히 확인한 다음 익숙해지면 Code 또는 Plan 모드로 이동하세요.

| 모드     | 설정 키        | 동작                                                                                                                                                                                                                                                                     |
| -------- | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Ask**  | `default`           | Claude는 각 파일 편집이나 명령어 전에 승인을 요청합니다. diff 보기를 통해 각 변경 사항을 수락하거나 거부할 수 있습니다. 새 사용자에게 권장됩니다.                                                                                                                         |
| **Code** | `acceptEdits`       | Claude는 파일 편집을 자동 수락하지만 터미널 명령어 전에 여전히 요청합니다. 파일 변경을 신뢰하고 더 빠른 반복을 원할 때 사용하세요.                                                                                                                              |
| **Plan** | `plan`              | Claude는 파일을 수정하거나 명령어를 실행하지 않고 코드를 분석하고 계획을 만듭니다. 먼저 접근 방식을 검토하고 싶은 복잡한 작업에 적합합니다.                                                                                                                |
| **Act**  | `bypassPermissions` | Claude는 권한 프롬프트 없이 실행됩니다. CLI의 `--dangerously-skip-permissions`와 동일합니다. 설정 → Claude Code의 "bypass permissions mode 허용"에서 활성화하세요. 샌드박스 컨테이너나 VM에서만 사용하세요. 엔터프라이즈 관리자는 이 옵션을 비활성화할 수 있습니다. |

`delegate` 및 `dontAsk` 권한 모드는 CLI에서만 사용 가능합니다.

> **모범 사례**: Claude가 변경하기 전에 접근 방식을 정리하도록 Plan 모드에서 복잡한 작업을 시작하세요. 계획을 승인하면 Code 또는 Ask 모드로 전환하여 실행하세요. 이 워크플로우에 대한 자세한 내용은 먼저 탐색하고, 계획하고, 그다음 코딩하기를 참조하세요.

원격 세션은 Code 모드와 Plan 모드를 지원합니다. Ask 모드는 원격 세션이 기본적으로 파일 편집을 자동 수락하기 때문에 사용할 수 없으며, Act 모드는 원격 환경이 이미 샌드박스화되어 있기 때문에 사용할 수 없습니다.

엔터프라이즈 관리자는 사용 가능한 권한 모드를 제한할 수 있습니다. 자세한 내용은 엔터프라이즈 구성을 참조하세요.

### Diff 보기로 변경 사항 검토

Claude가 코드를 변경한 후, diff 보기를 사용하면 pull request를 생성하기 전에 파일별로 수정 사항을 검토할 수 있습니다.

Claude가 파일을 변경하면 추가된 줄과 제거된 줄 수 (예: `+12 -1`)를 보여주는 diff 통계 표시기가 나타납니다. 이 표시기를 클릭하면 diff 뷰어가 열리며 왼쪽에 파일 목록, 오른쪽에 각 파일의 변경 사항이 표시됩니다.

특정 줄에 댓글을 달려면 diff의 줄을 클릭하여 댓글 상자를 엽니다. 피드백을 입력하고 **Enter**를 눌러 댓글을 추가합니다. 여러 줄에 댓글을 추가한 후 모든 댓글을 한꺼번에 제출합니다:

* **macOS**: **Cmd+Enter** 누르기
* **Windows**: **Ctrl+Enter** 누르기

Claude는 댓글을 읽고 요청된 변경을 수행하며, 검토할 수 있는 새 diff가 나타납니다.

## 세션 관리

각 세션은 고유한 컨텍스트와 변경 사항이 있는 독립적인 대화입니다. 여러 세션을 병렬로 실행하거나 클라우드에 작업을 전송할 수 있습니다.

### 세션으로 병렬 작업

사이드바에서 **+ 새 세션**을 클릭하여 여러 작업을 병렬로 처리합니다. Git 저장소의 경우 각 세션은 Git worktree를 사용하여 프로젝트의 격리된 복사본을 가지므로 한 세션의 변경 사항이 다른 세션에 커밋될 때까지 영향을 미치지 않습니다.

Worktree는 기본적으로 `<project-root>/.claude/worktrees/`에 저장됩니다. 설정 → Claude Code의 "Worktree 위치"에서 커스텀 디렉토리로 변경할 수 있습니다. 모든 worktree 브랜치 이름에 접두사를 붙이는 브랜치 접두사를 설정할 수도 있으며, 이는 Claude가 생성한 브랜치를 정리하는 데 유용합니다. worktree를 제거하려면 사이드바에서 세션 위에 마우스를 올려 보관 아이콘을 클릭하세요.

> 세션 격리에는 Git이 필요합니다. 대부분의 Mac은 기본적으로 Git이 포함되어 있습니다. 터미널에서 `git --version`을 실행하여 확인하세요. Windows에서는 없는 경우 Git을 다운로드하세요.
>
> Git 없이는 세션이 동일한 파일을 공유하고 한 세션의 변경 사항이 다른 세션에 즉시 표시됩니다.

사이드바 상단의 필터 아이콘을 사용하여 상태 (활성, 보관됨)와 환경 (로컬, 클라우드)으로 세션을 필터링하세요. 세션 이름을 변경하거나 컨텍스트 사용량을 확인하려면 활성 세션 상단 도구 모음에서 세션 제목을 클릭하세요. 컨텍스트가 꽉 차면 Claude는 자동으로 대화를 요약하고 계속 작업합니다. "compact this conversation"을 입력하여 요약을 조기에 트리거하고 컨텍스트 공간을 확보할 수도 있습니다.

### 장기 실행 작업을 원격으로 처리

대규모 리팩토링, 테스트 스위트, 마이그레이션 또는 기타 장기 실행 작업의 경우 세션을 시작할 때 **로컬** 대신 **원격**을 선택하세요. 원격 세션은 Anthropic의 클라우드 인프라에서 실행되며 앱을 닫거나 컴퓨터를 종료해도 계속됩니다. 언제든지 돌아와서 진행 상황을 확인하거나 다른 방향으로 Claude를 안내할 수 있습니다. claude.ai/code 또는 Claude iOS 앱에서 원격 세션을 모니터링할 수도 있습니다.

원격 세션은 여러 저장소도 지원합니다. 클라우드 환경을 선택한 후 저장소 약알 옆의 **+** 버튼을 클릭하여 세션에 추가 저장소를 추가합니다. 각 저장소는 자체 브랜치 선택기를 가집니다. 이는 공유 라이브러리와 그 소비자를 업데이트하는 것과 같이 여러 코드베이스에 걸친 작업에 유용합니다.

웹의 Claude Code에서 원격 세션 작동 방식에 대한 자세한 내용은 해당 문서를 참조하세요.

### 다른 환경에서 계속하기

세션 도구 모음 오른쪽 하단의 VS Code 아이콘에서 접근할 수 있는 **계속** 메뉴를 통해 세션을 다른 환경으로 이동할 수 있습니다:

* **웹의 Claude Code**: 로컬 세션을 원격으로 계속 실행되도록 전송합니다. 데스크탑이 브랜치를 푸시하고, 대화 요약을 생성하고, 전체 컨텍스트로 새 원격 세션을 만듭니다. 그런 다음 로컬 세션을 보관하거나 유지할 수 있습니다. 이를 위해 깨끗한 작업 트리가 필요하며 SSH 세션에서는 사용할 수 없습니다.
* **IDE**: 현재 작업 디렉토리에서 지원되는 IDE로 프로젝트를 엽니다.

## Claude Code 확장

외부 서비스를 연결하고, 재사용 가능한 워크플로우를 추가하고, 프로젝트에 맞게 Claude의 동작을 사용자 정의하세요.

### 외부 도구 연결

로컬 및 SSH 세션의 경우 프롬프트 상자 옆의 **+** 버튼을 클릭하고 **커넥터**를 선택하여 Google Calendar, Slack, GitHub, Linear, Notion 등의 통합을 추가하세요. 세션 전이나 세션 중에 커넥터를 추가할 수 있습니다. 원격 세션에서는 커넥터를 사용할 수 없습니다.

커넥터를 관리하거나 연결을 해제하려면 데스크탑 앱의 설정 → 커넥터로 이동하거나 프롬프트 상자의 커넥터 메뉴에서 **커넥터 관리**를 선택하세요.

연결되면 Claude가 일정을 읽고, 메시지를 보내고, 이슈를 생성하고, 도구와 직접 상호작용할 수 있습니다. 세션에 어떤 커넥터가 구성되어 있는지 Claude에게 물어볼 수 있습니다.

커넥터는 그래픽 설정 흐름이 있는 MCP 서버입니다. 지원되는 서비스와의 빠른 통합에 사용하세요. 커넥터 목록에 없는 통합의 경우 설정 파일을 통해 MCP 서버를 수동으로 추가하세요. 커스텀 커넥터를 만들 수도 있습니다.

### 스킬 사용

스킬은 Claude가 할 수 있는 것을 확장합니다. Claude는 관련될 때 자동으로 로드하거나 직접 호출할 수 있습니다: 프롬프트 상자에 `/`를 입력하거나 **+** 버튼을 클릭하고 **슬래시 명령어**를 선택하여 사용 가능한 항목을 탐색하세요. 여기에는 내장 명령어, 커스텀 스킬, 코드베이스의 프로젝트 스킬, 설치된 플러그인의 스킬이 포함됩니다. 하나를 선택하면 입력 필드에 강조 표시됩니다. 작업을 입력하고 평소처럼 전송하세요.

### 플러그인 설치

플러그인은 스킬, 에이전트, 훅, MCP 서버, LSP 구성을 Claude Code에 추가하는 재사용 가능한 패키지입니다. 터미널을 사용하지 않고 데스크탑 앱에서 플러그인을 설치할 수 있습니다.

로컬 및 SSH 세션의 경우 프롬프트 상자 옆의 **+** 버튼을 클릭하고 **플러그인**을 선택하여 설치된 플러그인과 해당 명령어를 확인하세요. 플러그인을 추가하려면 하위 메뉴에서 **플러그인 추가**를 선택하여 플러그인 브라우저를 열면 공식 Anthropic 마켓플레이스를 포함하여 구성된 마켓플레이스의 사용 가능한 플러그인이 표시됩니다. 플러그인을 활성화, 비활성화, 제거하려면 **플러그인 관리**를 선택하세요.

플러그인은 사용자 계정, 특정 프로젝트, 또는 로컬 전용으로 범위를 지정할 수 있습니다. 원격 세션에서는 플러그인을 사용할 수 없습니다. 자체 플러그인 생성을 포함한 전체 플러그인 참조는 플러그인을 참조하세요.

## 환경 구성

세션을 시작할 때 세 가지 환경 중에서 선택합니다:

* **로컬**: 파일에 직접 접근하여 컴퓨터에서 실행
* **원격**: Anthropic의 클라우드 인프라에서 실행. 앱을 닫아도 세션이 계속됩니다.
* **SSH**: SSH를 통해 연결하는 원격 머신에서 실행. 자체 서버, 클라우드 VM, 또는 개발 컨테이너

### 로컬 세션

로컬 세션은 셸에서 환경 변수를 상속합니다. 추가 변수가 필요한 경우 셸 프로파일 (`~/.zshrc` 또는 `~/.bashrc`)에 설정하고 데스크탑 앱을 다시 시작하세요. 지원되는 변수의 전체 목록은 환경 변수를 참조하세요.

확장 사고는 기본적으로 활성화되어 있으며, 복잡한 추론 작업에서 성능을 향상시키지만 추가 토큰을 사용합니다. 비활성화하거나 예산을 조정하려면 셸 프로파일에서 `MAX_THINKING_TOKENS`를 설정하세요. 비활성화는 `0`을 사용하세요.

### 원격 세션

원격 세션은 앱을 닫아도 백그라운드에서 계속됩니다. 사용량은 별도의 컴퓨팅 요금 없이 구독 플랜 한도에 계산됩니다.

다양한 네트워크 접근 수준과 환경 변수로 커스텀 클라우드 환경을 만들 수 있습니다. 원격 세션을 시작할 때 환경 드롭다운을 선택하고 **환경 추가**를 선택하세요. 네트워크 접근 구성 및 환경 변수에 대한 자세한 내용은 클라우드 환경을 참조하세요.

### SSH 세션

SSH 세션을 사용하면 데스크탑 앱을 인터페이스로 사용하면서 원격 머신에서 Claude Code를 실행할 수 있습니다. 클라우드 VM, 개발 컨테이너, 또는 특정 하드웨어나 종속성이 있는 서버에 있는 코드베이스를 작업하는 데 유용합니다.

SSH 연결을 추가하려면 세션을 시작하기 전에 환경 드롭다운을 클릭하고 **+ SSH 연결 추가**를 선택하세요. 다이얼로그에서 다음을 요청합니다:

* **이름**: 이 연결의 친근한 레이블
* **SSH 호스트**: `user@hostname` 또는 `~/.ssh/config`에 정의된 호스트
* **SSH 포트**: 비워두면 기본값 22 사용, 또는 SSH 구성의 포트 사용
* **신원 파일**: `~/.ssh/id_rsa`와 같은 개인 키 경로. 기본 키 또는 SSH 구성을 사용하려면 비워두세요.

추가되면 연결이 환경 드롭다운에 나타납니다. 선택하면 해당 머신에서 세션이 시작됩니다. Claude는 파일과 도구에 접근하여 원격 머신에서 실행됩니다.

SSH 세션은 권한 모드, 커넥터, 플러그인, MCP 서버를 지원합니다. 원격 머신에 Claude Code가 설치되어 있어야 합니다.

## 엔터프라이즈 구성

Teams 또는 Enterprise 플랜의 조직은 관리자 콘솔 제어, 관리형 설정 파일, 장치 관리 정책을 통해 데스크탑 앱 동작을 관리할 수 있습니다.

### 관리자 콘솔 제어

이 설정들은 관리자 설정 콘솔을 통해 구성됩니다:

* **코드 탭 활성화 또는 비활성화**: 조직의 사용자가 데스크탑 앱에서 Claude Code에 접근할 수 있는지 제어
* **Act 모드 비활성화**: 조직의 사용자가 bypass permissions 모드를 활성화하지 못하도록 방지
* **웹의 Claude Code 비활성화**: 조직의 원격 세션 활성화 또는 비활성화

### 관리형 설정

관리형 설정은 프로젝트 및 사용자 설정을 재정의하고 데스크탑이 CLI 세션을 생성할 때 적용됩니다. 조직의 관리형 설정 파일에서 이 키를 설정하거나 관리자 콘솔을 통해 원격으로 푸시할 수 있습니다.

| 키                                        | 설명                                                                                                      |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| `permissions.disableBypassPermissionsMode` | `"disable"`로 설정하면 사용자가 Act 모드를 활성화하지 못하게 합니다. 권한 참조. |

관리자 콘솔을 통해 업로드된 원격 관리형 설정은 현재 CLI 및 IDE 세션에만 적용됩니다. 데스크탑 특정 제한 사항은 위의 관리자 콘솔 제어를 사용하세요.

### 장치 관리 정책

IT 팀은 macOS의 MDM 또는 Windows의 그룹 정책을 통해 데스크탑 앱을 관리할 수 있습니다. 사용 가능한 정책으로는 Claude Code 기능 활성화 또는 비활성화, 자동 업데이트 제어, 커스텀 배포 URL 설정 등이 있습니다.

* **macOS**: Jamf 또는 Kandji와 같은 도구를 사용하여 `com.anthropic.Claude` 기본 설정 도메인을 통해 구성
* **Windows**: `SOFTWARE\Policies\Claude`의 레지스트리를 통해 구성

### 인증 및 SSO

엔터프라이즈 조직은 모든 사용자에게 SSO를 요구할 수 있습니다. 플랜별 세부 사항은 인증을 참조하고 SAML 및 OIDC 구성은 SSO 설정을 참조하세요.

### 데이터 처리

Claude Code는 로컬 세션에서는 로컬로, 원격 세션에서는 Anthropic의 클라우드 인프라에서 코드를 처리합니다. 대화와 코드 컨텍스트는 처리를 위해 Anthropic의 API로 전송됩니다. 데이터 보존, 개인 정보, 규정 준수에 대한 자세한 내용은 데이터 처리를 참조하세요.

### 배포

데스크탑은 엔터프라이즈 배포 도구를 통해 배포할 수 있습니다:

* **macOS**: `.dmg` 설치 프로그램을 사용하여 Jamf 또는 Kandji와 같은 MDM을 통해 배포
* **Windows**: MSIX 패키지 또는 `.exe` 설치 프로그램을 통해 배포. 자동 설치를 포함한 엔터프라이즈 배포 옵션은 Windows용 Claude 데스크탑 배포를 참조하세요.

프록시 설정, 방화벽 허용 목록, LLM 게이트웨이와 같은 네트워크 구성은 네트워크 구성을 참조하세요.

전체 엔터프라이즈 구성 참조는 엔터프라이즈 구성 가이드를 참조하세요.

## CLI를 사용해 오셨나요?

이미 Claude Code CLI를 사용 중이라면 데스크탑은 그래픽 인터페이스와 함께 동일한 기본 엔진을 실행합니다. 동일한 머신에서 심지어 동일한 프로젝트에서도 둘 다 동시에 실행할 수 있습니다. 각각은 별도의 세션 기록을 유지하지만 CLAUDE.md 파일을 통해 구성과 프로젝트 메모리를 공유합니다.

CLI 세션을 데스크탑으로 이동하려면 터미널에서 `/desktop`을 실행하세요. Claude가 세션을 저장하고 데스크탑 앱에서 열어 CLI를 종료합니다. 이 명령어는 macOS와 Windows에서만 사용 가능합니다.

> **데스크탑 vs CLI 사용 시기**: 시각적 diff 검토, 파일 첨부, 사이드바의 세션 관리를 원할 때 데스크탑을 사용하세요. 스크립팅, 자동화, 서드파티 공급자, 또는 터미널 워크플로우를 선호할 때 CLI를 사용하세요.

### CLI 플래그 동일 기능

이 표는 일반적인 CLI 플래그에 대한 데스크탑 앱 동일 기능을 보여줍니다. 목록에 없는 플래그는 스크립팅이나 자동화를 위해 설계되었기 때문에 데스크탑 동일 기능이 없습니다.

| CLI                                   | 데스크탑 동일 기능                                                                                            |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `--model sonnet`                      | 세션 시작 전 전송 버튼 옆 모델 드롭다운                                                                             |
| `--resume`, `--continue`              | 사이드바에서 세션 클릭                                                                                                |
| `--permission-mode`                   | 전송 버튼 옆 모드 선택기                                                                                         |
| `--dangerously-skip-permissions`      | 설정 → Claude Code → "bypass permissions mode 허용". 엔터프라이즈 관리자는 이 설정을 비활성화할 수 있습니다.         |
| `--add-dir`                           | 원격 세션에서 **+** 버튼으로 여러 저장소 추가                                                                   |
| `--allowedTools`, `--disallowedTools` | 데스크탑에서는 사용 불가                                                                                      |
| `--verbose`                           | 사용 불가. 시스템 로그 확인: macOS의 Console.app, Windows의 이벤트 뷰어 → 애플리케이션                 |
| `--print`, `--output-format`          | 사용 불가. 데스크탑은 인터랙티브 전용입니다.                                                                   |
| `ANTHROPIC_MODEL` 환경 변수             | 전송 버튼 옆 모델 드롭다운                                                                                        |
| `MAX_THINKING_TOKENS` 환경 변수         | 셸 프로파일에서 설정; 로컬 세션에 적용됩니다. 환경 구성 참조. |

### 공유 구성

데스크탑과 CLI는 동일한 구성 파일을 읽으므로 설정이 이어집니다:

* 프로젝트의 **CLAUDE.md** 및 **CLAUDE.local.md** 파일은 둘 다 사용합니다
* `~/.claude.json` 또는 `.mcp.json`에 구성된 **MCP 서버**는 둘 다 작동합니다
* 설정에 정의된 **훅**과 **스킬**은 둘 다 적용됩니다
* `~/.claude.json` 및 `~/.claude/settings.json`의 **설정**은 공유됩니다. `settings.json`의 권한 규칙, 허용 도구, 기타 설정은 데스크탑 세션에 적용됩니다.
* **모델**: Sonnet, Opus, Haiku는 둘 다 사용 가능합니다. 데스크탑에서는 세션 시작 전 전송 버튼 옆 드롭다운에서 모델을 선택하세요. 활성 세션 중에는 모델을 변경할 수 없습니다.

> **MCP 서버: 데스크탑 채팅 앱 vs Claude Code**: `claude_desktop_config.json`에서 Claude 데스크탑 채팅 앱용으로 구성된 MCP 서버는 Claude Code와 별도이며 코드 탭에 표시되지 않습니다. Claude Code에서 MCP 서버를 사용하려면 `~/.claude.json` 또는 프로젝트의 `.mcp.json` 파일에서 구성하세요. 자세한 내용은 MCP 구성을 참조하세요.

### 기능 비교

이 표는 CLI와 데스크탑 간의 핵심 기능을 비교합니다. CLI 플래그의 전체 목록은 CLI 레퍼런스를 참조하세요.

| 기능                                               | CLI                                                       | 데스크탑                                                      |
| ----------------------------------------------------- | --------------------------------------------------------- | ------------------------------------------------------------ |
| 권한 모드                                      | `delegate` 및 `dontAsk`를 포함한 모든 모드              | 설정을 통한 Ask, Code, Plan, Act                        |
| `--dangerously-skip-permissions`                      | CLI 플래그                                                  | 설정 → Claude Code → "bypass permissions mode 허용"     |
| 서드파티 공급자                                 | Bedrock, Vertex, Foundry                                  | 사용 불가. 데스크탑은 Anthropic API에 직접 연결됩니다. |
| MCP 서버                                           | 설정 파일에서 구성                               | 로컬 및 SSH 세션의 경우 커넥터 UI, 또는 설정 파일  |
| 플러그인                                               | `/plugin` 명령어                                         | 플러그인 매니저 UI                                            |
| @언급 파일                                        | 텍스트 기반                                                | 자동 완성 포함                                                 |
| 파일 첨부                                      | 사용 불가                                             | 이미지, PDF                                                 |
| 세션 격리                                     | git worktree를 통한 수동                                  | 자동 worktree                                                |
| 여러 세션                                     | 별도의 터미널                                        | 사이드바 탭                                                  |
| 스크립팅 및 자동화                                | `--print`, Agent SDK                                      | 사용 불가                                                |

### 데스크탑에서 사용할 수 없는 것

* **서드파티 공급자**: 데스크탑은 Anthropic API에 직접 연결됩니다. 대신 Bedrock, Vertex, 또는 Foundry와 함께 CLI를 사용하세요.
* **Linux**: 데스크탑 앱은 macOS와 Windows에서만 사용 가능합니다.
* **인라인 코드 제안**: 데스크탑은 자동 완성 스타일 제안을 제공하지 않습니다. 대화형 프롬프트와 명시적 코드 변경을 통해 작동합니다.
* **에이전트 팀 및 `delegate` 모드**: 멀티 에이전트 오케스트레이션과 `delegate` 권한 모드는 CLI와 Agent SDK를 통해 사용 가능하며 데스크탑에서는 사용할 수 없습니다.

## 문제 해결

### 버전 확인

실행 중인 데스크탑 앱 버전을 확인하려면:

* **macOS**: 메뉴 바에서 **Claude**를 클릭한 다음 **Claude 정보**
* **Windows**: **도움말**을 클릭한 다음 **정보**

버전 번호를 클릭하여 클립보드에 복사합니다.

### 코드 탭의 403 또는 인증 오류

코드 탭을 사용할 때 `Error 403: Forbidden` 또는 기타 인증 오류가 표시되는 경우:

1. 앱 메뉴에서 로그아웃 후 다시 로그인하세요. 가장 일반적인 해결책입니다.
2. 활성 유료 구독이 있는지 확인하세요: Pro, Max, Teams, 또는 Enterprise.
3. CLI는 작동하지만 데스크탑이 작동하지 않는 경우 데스크탑 앱을 완전히 종료 (창만 닫지 말고)하고 다시 열어 로그인하세요.
4. 인터넷 연결 및 프록시 설정을 확인하세요.

### 실행 시 빈 화면 또는 응답 없는 화면

앱이 열리지만 빈 화면이나 응답 없는 화면이 표시되는 경우:

1. 앱을 다시 시작하세요.
2. 보류 중인 업데이트를 확인하세요. 앱은 실행 시 자동으로 업데이트됩니다.
3. Windows에서 **Windows 로그 → 애플리케이션** 아래 이벤트 뷰어에서 충돌 로그를 확인하세요.

### "세션 로드 실패"

`Failed to load session`이 표시되면 선택한 폴더가 더 이상 존재하지 않거나, Git 저장소에 설치되지 않은 Git LFS가 필요하거나, 파일 권한이 접근을 방지할 수 있습니다. 다른 폴더를 선택하거나 앱을 다시 시작해보세요.

### 세션이 설치된 도구를 찾지 못함

Claude가 `npm`, `node`, 또는 기타 CLI 명령어를 찾지 못하는 경우, 일반 터미널에서 도구가 작동하는지 확인하고, 셸 프로파일이 PATH를 올바르게 설정하는지 확인하고, 환경 변수를 다시 로드하기 위해 데스크탑 앱을 다시 시작하세요.

### Git 및 Git LFS 오류

세션 격리와 worktree에는 Git이 필요합니다. "Git is required"가 표시되면 git-scm.com에서 Git을 설치하고 앱을 다시 시작하세요.

"Git LFS is required by this repository but is not installed"가 표시되면 git-lfs.com에서 Git LFS를 설치하고, `git lfs install`을 실행하고, 앱을 다시 시작하세요.

### Windows에서 MCP 서버가 작동하지 않음

Windows에서 MCP 서버 토글이 응답하지 않거나 서버가 연결에 실패하면, 서버가 설정에 올바르게 구성되어 있는지 확인하고, 앱을 다시 시작하고, 작업 관리자에서 서버 프로세스가 실행 중인지 확인하고, 서버 로그에서 연결 오류를 검토하세요.

### 앱이 종료되지 않음

* **macOS**: Cmd+Q를 누르세요. 앱이 응답하지 않으면 Cmd+Option+Esc로 강제 종료를 사용하고 Claude를 선택한 다음 강제 종료를 클릭하세요.
* **Windows**: Ctrl+Shift+Esc로 작업 관리자를 사용하여 Claude 프로세스를 종료하세요.

### Windows 특정 문제

* **설치 후 PATH가 업데이트되지 않음**: 새 터미널 창을 여세요. PATH 업데이트는 새 터미널 세션에만 적용됩니다.
* **동시 설치 오류**: 다른 설치가 진행 중이라는 오류가 표시되지만 실제로 없는 경우 관리자로 설치 프로그램을 실행해보세요.
* **ARM64 제한 사항**: Windows ARM64 장치는 데스크탑 앱을 실행할 수 있지만 로컬 세션은 지원하지 않습니다. 대신 **원격** 세션을 사용하세요.

### Intel Mac에서 Cowork 탭을 사용할 수 없음

Cowork 탭은 Apple Silicon, M1 이상이 필요합니다. 채팅 및 코드 탭은 Intel Mac에서 정상적으로 작동합니다.

### CLI에서 열 때 "브랜치가 아직 존재하지 않음"

원격 세션은 로컬 머신에 존재하지 않는 브랜치를 생성할 수 있습니다. 세션 도구 모음에서 브랜치 이름을 클릭하여 복사한 다음 로컬에서 가져오세요:

```bash
git fetch origin <branch-name>
git checkout <branch-name>
```
