# 빠른 모드로 응답 속도 향상

> 빠른 모드를 켜서 Claude Code에서 더 빠른 Opus 4.6 응답을 받으세요.

> **참고**: 빠른 모드는 리서치 프리뷰 중입니다. 피드백에 따라 기능, 가격, 가용성이 변경될 수 있습니다.

빠른 모드는 Claude Opus 4.6을 위한 고속 구성으로, 토큰당 비용은 높지만 모델을 2.5배 빠르게 만듭니다. 빠른 반복이나 실시간 디버깅과 같이 인터랙티브 작업에서 속도가 필요할 때 `/fast`로 켜고, 지연 시간보다 비용이 더 중요할 때 끄세요.

빠른 모드는 다른 모델이 아닙니다. 비용 효율보다 속도를 우선시하는 다른 API 구성의 동일한 Opus 4.6을 사용합니다. 동일한 품질과 기능을 제공하되, 응답이 더 빠릅니다.

알아야 할 사항:

- Claude Code CLI에서 `/fast`로 빠른 모드를 토글합니다. Claude Code VS Code 확장 프로그램에서도 `/fast`로 사용 가능합니다.
- Opus 4.6의 빠른 모드 가격은 $30/150 MTok부터 시작합니다. 빠른 모드는 2월 16일 오후 11:59 PT까지 모든 플랜에서 50% 할인으로 제공됩니다.
- 구독 플랜 (Pro/Max/Team/Enterprise) 및 Claude Console의 모든 Claude Code 사용자가 이용 가능합니다.
- 구독 플랜 (Pro/Max/Team/Enterprise)의 Claude Code 사용자의 경우, 빠른 모드는 추가 사용량을 통해서만 제공되며 구독 속도 제한에는 포함되지 않습니다.

## 빠른 모드 토글

다음 방법 중 하나로 빠른 모드를 토글합니다:

- `/fast`를 입력하고 Tab을 눌러 켜기 또는 끄기
- 사용자 설정 파일에서 `"fastMode": true` 설정

빠른 모드는 세션 간에 유지됩니다. 최상의 비용 효율을 위해 대화 중간이 아닌 세션 시작 시 빠른 모드를 활성화하세요. 자세한 내용은 아래 "비용 트레이드오프 이해하기"를 참조하세요.

빠른 모드를 활성화하면:

- 다른 모델을 사용 중인 경우 Claude Code가 자동으로 Opus 4.6으로 전환됩니다
- 확인 메시지가 표시됩니다: "Fast mode ON"
- 빠른 모드가 활성화된 동안 프롬프트 옆에 작은 `↯` 아이콘이 나타납니다
- 언제든지 `/fast`를 다시 실행하여 빠른 모드가 켜져 있는지 꺼져 있는지 확인할 수 있습니다

`/fast`로 빠른 모드를 비활성화해도 Opus 4.6에 유지됩니다. 이전 모델로 돌아가지 않습니다. 다른 모델로 전환하려면 `/model`을 사용하세요.

## 비용 트레이드오프 이해하기

빠른 모드는 표준 Opus 4.6보다 토큰당 가격이 높습니다:

| 모드                           | 입력 (MTok) | 출력 (MTok) |
| ------------------------------ | ------------ | ------------- |
| Opus 4.6의 빠른 모드 (&lt;200K)  | $30          | $150          |
| Opus 4.6의 빠른 모드 (>200K)  | $60          | $225          |

빠른 모드는 1M 토큰 확장 컨텍스트 창과 호환됩니다.

대화 중간에 빠른 모드로 전환하면 전체 대화 컨텍스트에 대해 전체 빠른 모드 캐시되지 않은 입력 토큰 가격이 청구됩니다. 처음부터 빠른 모드를 활성화한 경우보다 비용이 더 많이 듭니다.

## 빠른 모드 사용 시기 결정

빠른 모드는 비용보다 응답 지연 시간이 더 중요한 인터랙티브 작업에 적합합니다:

- 코드 변경에 대한 빠른 반복
- 실시간 디버깅 세션
- 촉박한 마감이 있는 시간에 민감한 작업

표준 모드가 더 적합한 경우:

- 속도가 덜 중요한 장기 자율 작업
- 배치 처리 또는 CI/CD 파이프라인
- 비용에 민감한 워크로드

### 빠른 모드 vs 노력 수준

빠른 모드와 노력 수준 모두 응답 속도에 영향을 주지만 다르게 작동합니다:

| 설정                | 효과                                                                           |
| ---------------------- | -------------------------------------------------------------------------------- |
| **빠른 모드**          | 동일한 모델 품질, 낮은 지연 시간, 높은 비용                                   |
| **낮은 노력 수준** | 적은 thinking 시간, 빠른 응답, 복잡한 작업에서 잠재적으로 낮은 품질 |

두 가지를 결합할 수 있습니다: 간단한 작업에서 최대 속도를 위해 낮은 노력 수준과 함께 빠른 모드를 사용하세요.

## 요구 사항

빠른 모드에는 다음 모든 항목이 필요합니다:

- **서드파티 클라우드 공급자에서는 사용 불가**: 빠른 모드는 Amazon Bedrock, Google Vertex AI, 또는 Microsoft Azure Foundry에서는 사용할 수 없습니다. 빠른 모드는 Anthropic Console API를 통해서 그리고 추가 사용량을 사용하는 Claude 구독 플랜에서 이용 가능합니다.
- **추가 사용량 활성화**: 계정에 추가 사용량이 활성화되어 있어야 합니다. 이는 플랜의 포함된 사용량을 초과하여 청구할 수 있게 합니다. 개인 계정의 경우 Console 결제 설정에서 활성화하세요. Teams 및 Enterprise의 경우 관리자가 조직의 추가 사용량을 활성화해야 합니다.

> 빠른 모드 사용량은 플랜에 남은 사용량이 있더라도 추가 사용량으로 직접 청구됩니다. 즉, 빠른 모드 토큰은 플랜의 포함된 사용량에 계산되지 않으며 첫 번째 토큰부터 빠른 모드 요금으로 청구됩니다.

- **Teams 및 Enterprise의 관리자 활성화**: 빠른 모드는 Teams 및 Enterprise 조직에서 기본적으로 비활성화되어 있습니다. 사용자가 접근하기 전에 관리자가 명시적으로 빠른 모드를 활성화해야 합니다.

> 조직의 관리자가 빠른 모드를 활성화하지 않은 경우, `/fast` 명령어는 "Fast mode has been disabled by your organization."을 표시합니다.

### 조직에 빠른 모드 활성화

관리자는 다음에서 빠른 모드를 활성화할 수 있습니다:

- **Console** (API 고객): Claude Code 기본 설정
- **Claude AI** (Teams 및 Enterprise): 관리자 설정 > Claude Code

## 속도 제한 처리

빠른 모드는 표준 Opus 4.6과 별도의 속도 제한이 있습니다. 빠른 모드 속도 제한에 도달하거나 추가 사용량 크레딧이 소진되면:

1. 빠른 모드가 자동으로 표준 Opus 4.6으로 폴백됩니다
2. 쿨다운을 나타내기 위해 `↯` 아이콘이 회색으로 변합니다
3. 표준 속도와 가격으로 계속 작업합니다
4. 쿨다운이 만료되면 빠른 모드가 자동으로 재활성화됩니다

쿨다운을 기다리는 대신 수동으로 빠른 모드를 비활성화하려면 `/fast`를 다시 실행하세요.

## 리서치 프리뷰

빠른 모드는 리서치 프리뷰 기능입니다. 이는 다음을 의미합니다:

- 피드백에 따라 기능이 변경될 수 있습니다
- 가용성과 가격은 변경될 수 있습니다
- 기반이 되는 API 구성이 발전할 수 있습니다

일반적인 Anthropic 지원 채널을 통해 문제나 피드백을 보고하세요.

## 관련 항목

- **모델 구성**: 모델 전환 및 노력 수준 조정
- **비용 효율적으로 관리**: 토큰 사용량 추적 및 비용 절감
- **상태 표시줄 구성**: 모델 및 컨텍스트 정보 표시
