# 스킬로 Claude 확장하기

> Claude Code에서 Claude의 역량을 확장하는 스킬을 만들고 관리하며 공유합니다. 커스텀 슬래시 명령 포함.

스킬은 Claude가 할 수 있는 일을 확장합니다. `SKILL.md` 파일에 지침을 작성하면 Claude가 이를 자신의 도구 모음에 추가합니다. Claude는 관련성이 있을 때 스킬을 사용하거나, `/skill-name`으로 직접 호출할 수 있습니다.

> **참고:** 커스텀 슬래시 명령이 스킬에 통합되었습니다. `.claude/commands/review.md`의 파일과 `.claude/skills/review/SKILL.md`의 스킬은 모두 `/review`를 만들고 동일하게 작동합니다. 기존 `.claude/commands/` 파일은 계속 작동합니다. 스킬은 선택적 기능을 추가합니다: 지원 파일을 위한 디렉토리, 사용자 또는 Claude가 호출할지 제어하는 frontmatter, Claude가 관련성이 있을 때 자동으로 로드하는 기능.

Claude Code 스킬은 [Agent Skills](https://agentskills.io) 오픈 표준을 따릅니다.

## 시작하기

### 첫 번째 스킬 만들기

이 예시는 Claude가 시각적 다이어그램과 비유를 사용하여 코드를 설명하도록 가르치는 스킬을 만듭니다.

**1단계: 스킬 디렉토리 만들기**

개인 스킬 폴더에 스킬을 위한 디렉토리를 만듭니다. 개인 스킬은 모든 프로젝트에서 사용할 수 있습니다.

```bash
mkdir -p ~/.claude/skills/explain-code
```

**2단계: SKILL.md 작성**

모든 스킬에는 두 부분으로 구성된 `SKILL.md` 파일이 필요합니다: Claude에게 스킬을 사용할 시점을 알려주는 YAML frontmatter (`---` 마커 사이)와 스킬이 호출될 때 Claude가 따르는 지침이 담긴 markdown 콘텐츠. `name` 필드가 `/slash-command`가 되고, `description`은 Claude가 자동으로 로드할 시기를 결정하는 데 도움을 줍니다.

`~/.claude/skills/explain-code/SKILL.md` 만들기:

```yaml
---
name: explain-code
description: Explains code with visual diagrams and analogies. Use when explaining how code works, teaching about a codebase, or when the user asks "how does this work?"
---

When explaining code, always include:

1. **Start with an analogy**: Compare the code to something from everyday life
2. **Draw a diagram**: Use ASCII art to show the flow, structure, or relationships
3. **Walk through the code**: Explain step-by-step what happens
4. **Highlight a gotcha**: What's a common mistake or misconception?

Keep explanations conversational. For complex concepts, use multiple analogies.
```

**3단계: 스킬 테스트**

두 가지 방법으로 테스트할 수 있습니다:

**설명과 일치하는 질문으로 Claude가 자동으로 호출하도록 하기:**

```
How does this code work?
```

**또는 스킬 이름으로 직접 호출하기:**

```
/explain-code src/auth/login.ts
```

### 스킬이 저장되는 위치

스킬을 저장하는 위치에 따라 사용할 수 있는 사람이 결정됩니다:

| 위치 | 경로 | 적용 대상 |
| :--- | :--- | :--- |
| Enterprise | Managed 설정 참조 | 조직의 모든 사용자 |
| Personal | `~/.claude/skills/<skill-name>/SKILL.md` | 모든 프로젝트 |
| Project | `.claude/skills/<skill-name>/SKILL.md` | 이 프로젝트만 |
| Plugin | `<plugin>/skills/<skill-name>/SKILL.md` | 플러그인이 활성화된 곳 |

동일한 이름의 스킬이 여러 수준에 있는 경우, 우선순위가 높은 위치가 승리합니다: enterprise > personal > project. 플러그인 스킬은 `plugin-name:skill-name` 네임스페이스를 사용합니다.

#### 중첩된 디렉토리에서 자동 검색

하위 디렉토리의 파일로 작업할 때, Claude Code는 중첩된 `.claude/skills/` 디렉토리에서 스킬을 자동으로 검색합니다. 이는 패키지마다 자체 스킬이 있는 모노레포 설정을 지원합니다.

각 스킬은 `SKILL.md`를 진입점으로 가진 디렉토리입니다:

```
my-skill/
├── SKILL.md           # 주 지침 (필수)
├── template.md        # Claude가 채울 템플릿
├── examples/
│   └── sample.md      # 예상 형식을 보여주는 예시 출력
└── scripts/
    └── validate.sh    # Claude가 실행할 수 있는 스크립트
```

#### 추가 디렉토리의 스킬

`--add-dir`을 통해 추가된 디렉토리의 `.claude/skills/`에 정의된 스킬은 자동으로 로드되고 실시간 변경 감지에 의해 선택됩니다. 따라서 재시작 없이 세션 중에 편집할 수 있습니다.

## 스킬 구성

### 스킬 콘텐츠 유형

**참조 콘텐츠**는 Claude가 현재 작업에 적용하는 지식을 추가합니다. 규칙, 패턴, 스타일 가이드, 도메인 지식. 이 콘텐츠는 인라인으로 실행되므로 Claude가 대화 컨텍스트와 함께 사용할 수 있습니다.

```yaml
---
name: api-conventions
description: API design patterns for this codebase
---

When writing API endpoints:
- Use RESTful naming conventions
- Return consistent error formats
- Include request validation
```

**작업 콘텐츠**는 Claude에게 특정 동작을 위한 단계별 지침을 제공합니다. 이는 종종 Claude가 결정하도록 하는 것보다 `/skill-name`으로 직접 호출하려는 동작입니다.

```yaml
---
name: deploy
description: Deploy the application to production
context: fork
disable-model-invocation: true
---

Deploy the application:
1. Run the test suite
2. Build the application
3. Push to the deployment target
```

### Frontmatter 레퍼런스

모든 필드는 선택 사항입니다. Claude가 스킬을 언제 사용할지 알 수 있도록 `description`만 권장됩니다.

| 필드 | 필수 | 설명 |
| :--- | :--- | :--- |
| `name` | 아니요 | 스킬의 표시 이름. 생략하면 디렉토리 이름을 사용합니다. 소문자, 숫자, 하이픈만 사용 (최대 64자). |
| `description` | 권장 | 스킬이 하는 일과 사용 시점. |
| `argument-hint` | 아니요 | 자동 완성 중에 표시되는 힌트. 예: `[issue-number]` 또는 `[filename] [format]`. |
| `disable-model-invocation` | 아니요 | Claude가 이 스킬을 자동으로 로드하지 못하도록 `true`로 설정합니다. 기본값: `false`. |
| `user-invocable` | 아니요 | `/` 메뉴에서 숨기려면 `false`로 설정합니다. 기본값: `true`. |
| `allowed-tools` | 아니요 | 이 스킬이 활성화되어 있을 때 권한 요청 없이 Claude가 사용할 수 있는 도구. |
| `model` | 아니요 | 이 스킬이 활성화되어 있을 때 사용할 모델. |
| `context` | 아니요 | `fork`로 설정하면 분기된 서브에이전트 컨텍스트에서 실행됩니다. |
| `agent` | 아니요 | `context: fork`가 설정된 경우 사용할 서브에이전트 유형. |
| `hooks` | 아니요 | 이 스킬의 라이프사이클에 범위가 지정된 Hooks. |

#### 사용 가능한 문자열 치환

| 변수 | 설명 |
| :--- | :--- |
| `$ARGUMENTS` | 스킬을 호출할 때 전달된 모든 인수. |
| `$ARGUMENTS[N]` | 0 기반 인덱스로 특정 인수에 접근합니다. |
| `$N` | `$ARGUMENTS[N]`의 축약형. |
| `${CLAUDE_SESSION_ID}` | 현재 세션 ID. |

**치환을 사용한 예시:**

```yaml
---
name: session-logger
description: Log activity for this session
---

Log the following to logs/${CLAUDE_SESSION_ID}.log:

$ARGUMENTS
```

### 지원 파일 추가

스킬에는 디렉토리에 여러 파일이 포함될 수 있습니다. 이렇게 하면 `SKILL.md`가 핵심 내용에 집중하면서 Claude가 필요할 때만 상세한 참조 자료에 접근할 수 있습니다.

```
my-skill/
├── SKILL.md (필수 - 개요 및 탐색)
├── reference.md (상세 API 문서 - 필요 시 로드)
├── examples.md (사용 예시 - 필요 시 로드)
└── scripts/
    └── helper.py (유틸리티 스크립트 - 실행되지만 로드되지 않음)
```

Claude가 각 파일에 무엇이 있는지 알 수 있도록 `SKILL.md`에서 지원 파일을 참조하세요:

```markdown
## 추가 리소스

- 완전한 API 세부 정보는 [reference.md](reference.md)를 참조하세요
- 사용 예시는 [examples.md](examples.md)를 참조하세요
```

`SKILL.md`를 500줄 이하로 유지하세요. 상세한 참조 자료는 별도 파일로 이동하세요.

### 스킬을 호출할 사람 제어

기본적으로 사용자와 Claude 모두 어떤 스킬이든 호출할 수 있습니다.

* **`disable-model-invocation: true`**: 사용자만 스킬을 호출할 수 있습니다. `/commit`, `/deploy`, `/send-slack-message`와 같은 부작용이 있는 워크플로우에 사용하세요.

* **`user-invocable: false`**: Claude만 스킬을 호출할 수 있습니다. 명령으로 실행 가능하지 않은 배경 지식에 사용하세요.

| Frontmatter | 사용자 호출 가능 | Claude 호출 가능 | 컨텍스트에 로드되는 시점 |
| :--- | :--- | :--- | :--- |
| (기본값) | 예 | 예 | 설명이 항상 컨텍스트에 있고, 호출 시 전체 스킬 로드 |
| `disable-model-invocation: true` | 예 | 아니요 | 설명이 컨텍스트에 없고, 사용자가 호출할 때 전체 스킬 로드 |
| `user-invocable: false` | 아니요 | 예 | 설명이 항상 컨텍스트에 있고, 호출 시 전체 스킬 로드 |

### 도구 접근 제한

`allowed-tools` 필드를 사용하여 스킬이 활성화되어 있을 때 Claude가 사용할 수 있는 도구를 제한합니다:

```yaml
---
name: safe-reader
description: Read files without making changes
allowed-tools: Read, Grep, Glob
---
```

### 스킬에 인수 전달

사용자와 Claude 모두 스킬을 호출할 때 인수를 전달할 수 있습니다. 인수는 `$ARGUMENTS` 플레이스홀더를 통해 사용할 수 있습니다.

```yaml
---
name: fix-issue
description: Fix a GitHub issue
disable-model-invocation: true
---

Fix GitHub issue $ARGUMENTS following our coding standards.

1. Read the issue description
2. Understand the requirements
3. Implement the fix
4. Write tests
5. Create a commit
```

`/fix-issue 123`을 실행하면 Claude는 "Fix GitHub issue 123 following our coding standards..."를 받습니다.

위치별로 개별 인수에 접근하려면:

```yaml
---
name: migrate-component
description: Migrate a component from one framework to another
---

Migrate the $ARGUMENTS[0] component from $ARGUMENTS[1] to $ARGUMENTS[2].
Preserve all existing behavior and tests.
```

`/migrate-component SearchBar React Vue`를 실행하면 `$ARGUMENTS[0]`이 `SearchBar`로, `$ARGUMENTS[1]`이 `React`로, `$ARGUMENTS[2]`가 `Vue`로 교체됩니다.

## 고급 패턴

### 동적 컨텍스트 주입

`` !`command` `` 구문은 스킬 콘텐츠가 Claude에게 전송되기 전에 셸 명령을 실행합니다. 명령 출력이 플레이스홀더를 대체하므로 Claude는 명령 자체가 아닌 실제 데이터를 받습니다.

```yaml
---
name: pr-summary
description: Summarize changes in a pull request
context: fork
agent: Explore
allowed-tools: Bash(gh *)
---

## Pull request context
- PR diff: !`gh pr diff`
- PR comments: !`gh pr view --comments`
- Changed files: !`gh pr diff --name-only`

## Your task
Summarize this pull request...
```

이 스킬이 실행될 때:

1. 각 `` !`command` ``가 즉시 실행됩니다 (Claude가 보기 전에)
2. 출력이 스킬 콘텐츠의 플레이스홀더를 대체합니다
3. Claude는 실제 PR 데이터가 포함된 완전히 렌더링된 프롬프트를 받습니다

스킬에서 확장 사고를 활성화하려면 스킬 콘텐츠 어딘가에 "ultrathink"를 포함하세요.

### 서브에이전트에서 스킬 실행

스킬을 격리된 환경에서 실행하려면 frontmatter에 `context: fork`를 추가하세요. 스킬 콘텐츠가 서브에이전트를 구동하는 프롬프트가 됩니다. 대화 기록에 접근할 수 없습니다.

> **경고:** `context: fork`는 명시적인 지침이 있는 스킬에만 의미가 있습니다. 스킬에 작업 없이 지침만 포함된 경우, 서브에이전트는 지침을 받지만 실행 가능한 프롬프트가 없습니다.

스킬과 서브에이전트는 두 방향으로 함께 작동합니다:

| 접근 방식 | 시스템 프롬프트 | 작업 | 추가 로드 항목 |
| :--- | :--- | :--- | :--- |
| `context: fork`가 있는 스킬 | 에이전트 유형에서 (`Explore`, `Plan` 등) | SKILL.md 콘텐츠 | CLAUDE.md |
| `skills` 필드가 있는 서브에이전트 | 서브에이전트의 markdown 본문 | Claude의 위임 메시지 | 사전 로드된 스킬 + CLAUDE.md |

#### 예시: Explore 에이전트를 사용한 리서치 스킬

```yaml
---
name: deep-research
description: Research a topic thoroughly
context: fork
agent: Explore
---

Research $ARGUMENTS thoroughly:

1. Find relevant files using Glob and Grep
2. Read and analyze the code
3. Summarize findings with specific file references
```

`agent` 필드는 사용할 서브에이전트 구성을 지정합니다. 옵션에는 내장 에이전트 (`Explore`, `Plan`, `general-purpose`) 또는 `.claude/agents/`의 커스텀 서브에이전트가 포함됩니다. 생략하면 `general-purpose`를 사용합니다.

### Claude의 스킬 접근 제한

Claude가 호출할 수 있는 스킬을 제어하는 세 가지 방법:

**`/permissions`에서 Skill 도구를 거부하여 모든 스킬 비활성화**.

**특정 스킬을 허용하거나 거부**하기 위해 권한 규칙 사용:

```
# 특정 스킬만 허용
Skill(commit)
Skill(review-pr *)

# 특정 스킬 거부
Skill(deploy *)
```

권한 구문: 정확한 일치를 위해 `Skill(name)`, 인수가 있는 접두사 매칭을 위해 `Skill(name *)`.

**개별 스킬 숨기기**: frontmatter에 `disable-model-invocation: true`를 추가합니다.

## 스킬 공유

스킬은 대상 청중에 따라 다양한 범위에서 배포될 수 있습니다:

* **프로젝트 스킬**: `.claude/skills/`를 버전 컨트롤에 커밋
* **플러그인**: 플러그인에 `skills/` 디렉토리 생성
* **Managed**: Managed 설정을 통해 조직 전체에 배포

### 시각적 출력 생성

스킬은 모든 언어의 스크립트를 번들하고 실행할 수 있어, 단일 프롬프트에서 가능한 것 이상의 기능을 Claude에게 제공합니다.

스킬 디렉토리 만들기:

```bash
mkdir -p ~/.claude/skills/codebase-visualizer/scripts
```

`~/.claude/skills/codebase-visualizer/SKILL.md` 만들기:

````yaml
---
name: codebase-visualizer
description: Generate an interactive collapsible tree visualization of your codebase. Use when exploring a new repo, understanding project structure, or identifying large files.
allowed-tools: Bash(python *)
---

# Codebase Visualizer

Generate an interactive HTML tree view that shows your project's file structure with collapsible directories.

## Usage

Run the visualization script from your project root:

```bash
python ~/.claude/skills/codebase-visualizer/scripts/visualize.py .
```

This creates `codebase-map.html` in the current directory and opens it in your default browser.
````

## 문제 해결

### 스킬이 트리거되지 않는 경우

Claude가 예상대로 스킬을 사용하지 않는 경우:

1. 설명에 사용자가 자연스럽게 사용할 키워드가 포함되어 있는지 확인
2. `What skills are available?`에서 스킬이 나타나는지 확인
3. 설명에 더 잘 맞도록 요청을 다시 표현해보세요
4. 스킬이 사용자 호출 가능한 경우 `/skill-name`으로 직접 호출하세요

### 스킬이 너무 자주 트리거되는 경우

원하지 않을 때 Claude가 스킬을 사용하는 경우:

1. 설명을 더 구체적으로 만드세요
2. 수동 호출만 원하는 경우 `disable-model-invocation: true`를 추가하세요

### Claude가 모든 스킬을 보지 못하는 경우

Claude가 사용 가능한 항목을 알 수 있도록 스킬 설명이 컨텍스트에 로드됩니다. 스킬이 많으면 문자 예산을 초과할 수 있습니다. 예산은 컨텍스트 창의 2%에 따라 동적으로 확장되며, 16,000자의 폴백이 있습니다.

제한을 재정의하려면 `SLASH_COMMAND_TOOL_CHAR_BUDGET` 환경 변수를 설정하세요.
