# 데이터 사용 정책

> Claude에 대한 Anthropic의 데이터 사용 정책에 대해 알아봅니다.

## 데이터 정책

### 데이터 교육 정책

**소비자 사용자 (Free, Pro, Max 플랜)**:
데이터를 미래 Claude 모델 개선에 사용하도록 허용할 선택권을 제공합니다. 이 설정이 켜져 있는 경우 Free, Pro, Max 계정의 데이터를 사용하여 새 모델을 교육합니다 (이러한 계정에서 Claude Code를 사용하는 경우 포함).

**상업 사용자** (Teams, Enterprise 플랜, API, 서드파티 플랫폼, Claude Gov): 기존 정책을 유지합니다. Anthropic은 고객이 모델 개선을 위해 데이터를 제공하도록 선택한 경우를 제외하고 (예: [개발자 파트너 프로그램](https://support.claude.com/en/articles/11174108-about-the-development-partner-program)) 상업 약관에 따라 Claude Code에 전송된 코드 또는 프롬프트를 사용하여 생성 모델을 교육하지 않습니다.

### 개발자 파트너 프로그램

[개발자 파트너 프로그램](https://support.claude.com/en/articles/11174108-about-the-development-partner-program)과 같이 교육할 자료를 제공하는 방법에 명시적으로 옵트인하는 경우 제공된 자료를 모델 교육에 사용할 수 있습니다. 조직 관리자는 조직의 개발자 파트너 프로그램에 명시적으로 옵트인할 수 있습니다. 이 프로그램은 Anthropic 1차 API에만 사용 가능하며 Bedrock 또는 Vertex 사용자에게는 제공되지 않습니다.

### `/bug` 명령을 사용한 피드백

`/bug` 명령을 사용하여 Claude Code에 대한 피드백을 보내도록 선택하는 경우, 피드백을 제품 및 서비스 개선에 사용할 수 있습니다. `/bug`를 통해 공유된 대화록은 5년간 보관됩니다.

### 세션 품질 설문 조사

Claude Code에서 "Claude는 이번 세션에서 어떻게 수행하고 있나요?" 프롬프트를 볼 때, 이 설문 조사에 응답하면 (Dismiss 선택 포함) 숫자 평점 (1, 2, 3 또는 dismiss)만 기록됩니다. 이 설문 조사의 일부로 대화 기록, 입력, 출력 또는 기타 세션 데이터는 수집하거나 저장하지 않습니다. 좋아요/싫어요 피드백이나 `/bug` 보고서와 달리 이 세션 품질 설문 조사는 단순한 제품 만족도 지표입니다. 이 설문 조사에 대한 응답은 데이터 교육 선호도에 영향을 미치지 않으며 AI 모델을 교육하는 데 사용될 수 없습니다.

이 설문 조사를 비활성화하려면 `CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1`을 설정하세요. 서드파티 프로바이더(Bedrock, Vertex, Foundry)를 사용하거나 원격 측정이 비활성화된 경우에도 설문 조사가 자동으로 비활성화됩니다.

### 데이터 보존

Anthropic은 계정 유형과 선호도에 따라 Claude Code 데이터를 보관합니다.

**소비자 사용자 (Free, Pro, Max 플랜)**:

* 모델 개선을 위해 데이터 사용을 허용하는 사용자: 모델 개발 및 안전 개선을 지원하기 위한 5년 보존 기간
* 모델 개선을 위한 데이터 사용을 허용하지 않는 사용자: 30일 보존 기간
* 개인 정보 설정은 언제든지 [claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls)에서 변경 가능합니다.

**상업 사용자 (Teams, Enterprise, API)**:

* 표준: 30일 보존 기간
* Zero data retention: 적절하게 구성된 API 키로 사용 가능 - Claude Code는 서버에 채팅 기록을 보관하지 않습니다.
* 로컬 캐싱: Claude Code 클라이언트는 세션 재개를 위해 최대 30일 동안 로컬로 세션을 저장할 수 있습니다 (구성 가능).

데이터 보존 관행에 대한 자세한 내용은 [Privacy Center](https://privacy.anthropic.com/)를 참조하세요.

전체 세부 사항은 [상업 서비스 약관](https://www.anthropic.com/legal/commercial-terms) (Teams, Enterprise, API 사용자) 또는 [소비자 약관](https://www.anthropic.com/legal/consumer-terms) (Free, Pro, Max 사용자) 및 [개인 정보 보호 정책](https://www.anthropic.com/legal/privacy)을 검토하세요.

## 데이터 액세스

모든 1차 사용자의 경우 로컬 Claude Code 및 원격 Claude Code에 대해 기록되는 데이터에 대해 자세히 알아볼 수 있습니다. 원격 Claude Code의 경우 Claude는 Claude Code 세션을 시작한 저장소에 액세스합니다. Claude는 연결되어 있지만 세션을 시작하지 않은 저장소에는 액세스하지 않습니다.

## 로컬 Claude Code: 데이터 흐름 및 종속성

Claude Code는 [NPM](https://www.npmjs.com/package/@anthropic-ai/claude-code)에서 설치됩니다. Claude Code는 로컬에서 실행됩니다. LLM과 상호 작용하기 위해 Claude Code는 네트워크를 통해 데이터를 전송합니다. 이 데이터에는 모든 사용자 프롬프트와 모델 출력이 포함됩니다. 데이터는 TLS를 통해 전송 중에 암호화되며 저장 시에는 암호화되지 않습니다. Claude Code는 대부분의 인기 있는 VPN 및 LLM 프록시와 호환됩니다.

Claude Code는 Anthropic의 API를 기반으로 구축됩니다. API 로깅 절차를 포함한 API의 보안 제어에 대한 자세한 내용은 [Anthropic Trust Center](https://trust.anthropic.com)에서 제공되는 규정 준수 아티팩트를 참조하세요.

### 클라우드 실행: 데이터 흐름 및 종속성

[웹에서 Claude Code를 사용](/en/claude-code-on-the-web)할 때 세션은 로컬이 아닌 Anthropic이 관리하는 가상 머신에서 실행됩니다. 클라우드 환경에서:

* **코드 및 데이터 저장:** 저장소가 격리된 VM에 복제됩니다. 코드 및 세션 데이터는 계정 유형의 보존 및 사용 정책을 따릅니다 (위의 데이터 보존 섹션 참조).
* **자격 증명:** GitHub 인증은 보안 프록시를 통해 처리되며 GitHub 자격 증명은 샌드박스에 들어가지 않습니다.
* **네트워크 트래픽:** 모든 아웃바운드 트래픽은 감사 로깅 및 남용 방지를 위해 보안 프록시를 통해 흐릅니다.
* **세션 데이터:** 프롬프트, 코드 변경 사항 및 출력은 로컬 Claude Code 사용과 동일한 데이터 정책을 따릅니다.

클라우드 실행의 보안 세부 정보는 [보안](/en/security#cloud-execution-security)을 참조하세요.

## 원격 측정 서비스

Claude Code는 지연 시간, 안정성, 사용 패턴과 같은 운영 지표를 기록하기 위해 사용자 머신에서 Statsig 서비스에 연결합니다. 이 로깅에는 코드 또는 파일 경로가 포함되지 않습니다. 데이터는 TLS를 사용하여 전송 중에 암호화되고 저장 시 256비트 AES 암호화를 사용합니다. 자세한 내용은 [Statsig 보안 문서](https://www.statsig.com/trust/security)를 참조하세요. Statsig 원격 측정을 거부하려면 `DISABLE_TELEMETRY` 환경 변수를 설정하세요.

Claude Code는 운영 오류 로깅을 위해 사용자 머신에서 Sentry에 연결합니다. 데이터는 TLS를 사용하여 전송 중에 암호화되고 저장 시 256비트 AES 암호화를 사용합니다. 자세한 내용은 [Sentry 보안 문서](https://sentry.io/security/)를 참조하세요. 오류 로깅을 거부하려면 `DISABLE_ERROR_REPORTING` 환경 변수를 설정하세요.

사용자가 `/bug` 명령을 실행하면 코드를 포함한 전체 대화 기록의 사본이 Anthropic으로 전송됩니다. 데이터는 전송 중 및 저장 시 암호화됩니다. 선택적으로 공개 저장소에 Github 이슈가 생성됩니다. 버그 보고를 거부하려면 `DISABLE_BUG_COMMAND` 환경 변수를 설정하세요.

## API 공급자별 기본 동작

기본적으로 Bedrock, Vertex 또는 Foundry를 사용할 때 모든 비필수 트래픽(오류 보고, 원격 측정, 버그 보고 기능, 세션 품질 설문 조사 포함)을 비활성화합니다. `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 환경 변수를 설정하여 한 번에 모두 거부할 수도 있습니다. 전체 기본 동작은 다음과 같습니다.

| 서비스 | Claude API | Vertex API | Bedrock API | Foundry API |
|---------|-----------|------------|-------------|-------------|
| **Statsig (지표)** | 기본값 켜짐. `DISABLE_TELEMETRY=1`로 비활성화. | 기본값 꺼짐. `CLAUDE_CODE_USE_VERTEX`가 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_BEDROCK`이 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_FOUNDRY`가 1이어야 함. |
| **Sentry (오류)** | 기본값 켜짐. `DISABLE_ERROR_REPORTING=1`로 비활성화. | 기본값 꺼짐. `CLAUDE_CODE_USE_VERTEX`가 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_BEDROCK`이 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_FOUNDRY`가 1이어야 함. |
| **Claude API (`/bug` 보고)** | 기본값 켜짐. `DISABLE_BUG_COMMAND=1`로 비활성화. | 기본값 꺼짐. `CLAUDE_CODE_USE_VERTEX`가 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_BEDROCK`이 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_FOUNDRY`가 1이어야 함. |
| **세션 품질 설문 조사** | 기본값 켜짐. `CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1`로 비활성화. | 기본값 꺼짐. `CLAUDE_CODE_USE_VERTEX`가 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_BEDROCK`이 1이어야 함. | 기본값 꺼짐. `CLAUDE_CODE_USE_FOUNDRY`가 1이어야 함. |

모든 환경 변수는 `settings.json`에 체크인할 수 있습니다 ([자세히 보기](/en/settings)).
