# 출력 스타일

> 소프트웨어 엔지니어링 이외의 용도에 맞게 Claude Code를 조정합니다.

출력 스타일을 사용하면 로컬 스크립트 실행, 파일 읽기/쓰기, TODO 추적과 같은 핵심 기능을 유지하면서 Claude Code를 모든 유형의 에이전트로 사용할 수 있습니다.

## 내장 출력 스타일

Claude Code의 **기본** 출력 스타일은 소프트웨어 엔지니어링 작업을 효율적으로 완료하는 데 도움이 되도록 설계된 기존 시스템 프롬프트입니다.

코드베이스와 Claude의 작동 방식을 가르치는 데 집중된 두 가지 추가 내장 출력 스타일이 있습니다:

- **설명형 (Explanatory)**: 소프트웨어 엔지니어링 작업을 돕는 사이에 교육적인 "인사이트"를 제공합니다. 구현 선택과 코드베이스 패턴을 이해하는 데 도움이 됩니다.

- **학습형 (Learning)**: 협력적인 배우면서 하기 모드로, Claude가 코딩하면서 "인사이트"를 공유할 뿐만 아니라 여러분이 작은 전략적 코드 조각을 직접 기여하도록 요청합니다. Claude Code가 여러분이 구현할 코드에 `TODO(human)` 마커를 추가합니다.

## 출력 스타일 작동 방식

출력 스타일은 Claude Code의 시스템 프롬프트를 직접 수정합니다.

- 모든 출력 스타일은 효율적인 출력을 위한 지시 (간결하게 응답하는 것 등)를 제외합니다.
- 커스텀 출력 스타일은 `keep-coding-instructions`가 true가 아닌 한 코딩을 위한 지시 (테스트로 코드 검증하는 것 등)를 제외합니다.
- 모든 출력 스타일은 시스템 프롬프트 끝에 자체 커스텀 지시가 추가됩니다.
- 모든 출력 스타일은 대화 중에 Claude가 출력 스타일 지시를 따르도록 리마인더를 트리거합니다.

## 출력 스타일 변경

다음 방법 중 하나를 사용할 수 있습니다:

- `/output-style`을 실행하여 메뉴에 접근하고 출력 스타일을 선택합니다 (`/config` 메뉴에서도 접근 가능)
- `/output-style [style]`을 실행합니다. 예를 들어 `/output-style explanatory`로 특정 스타일로 직접 전환합니다

이러한 변경 사항은 로컬 프로젝트 수준에 적용되어 `.claude/settings.local.json`에 저장됩니다. 다른 수준의 설정 파일에서 `outputStyle` 필드를 직접 편집할 수도 있습니다.

## 커스텀 출력 스타일 만들기

커스텀 출력 스타일은 frontmatter와 시스템 프롬프트에 추가될 텍스트로 구성된 마크다운 파일입니다:

```markdown
---
name: My Custom Style
description:
  A brief description of what this style does, to be displayed to the user
---

# Custom Style Instructions

You are an interactive CLI tool that helps users with software engineering
tasks. [Your custom instructions here...]

## Specific Behaviors

[Define how the assistant should behave in this style...]
```

이러한 파일을 사용자 수준 (`~/.claude/output-styles`) 또는 프로젝트 수준 (`.claude/output-styles`)에 저장할 수 있습니다.

### Frontmatter

출력 스타일 파일은 명령어에 대한 메타데이터를 지정하는 데 유용한 frontmatter를 지원합니다:

| Frontmatter                | 목적                                                                     | 기본값                 |
| :------------------------- | :-------------------------------------------------------------------------- | :---------------------- |
| `name`                     | 파일 이름이 아닌 경우의 출력 스타일 이름                              | 파일 이름에서 상속 |
| `description`              | 출력 스타일 설명. `/output-style`의 UI에서만 사용     | 없음                    |
| `keep-coding-instructions` | 코딩과 관련된 Claude Code의 시스템 프롬프트 부분을 유지할지 여부. | false                   |

## 관련 기능 비교

### 출력 스타일 vs. CLAUDE.md vs. --append-system-prompt

출력 스타일은 소프트웨어 엔지니어링에 특정한 Claude Code의 기본 시스템 프롬프트 부분을 완전히 "끕니다". CLAUDE.md와 `--append-system-prompt`는 Claude Code의 기본 시스템 프롬프트를 편집하지 않습니다. CLAUDE.md는 Claude Code의 기본 시스템 프롬프트 *다음에* 사용자 메시지로 내용을 추가합니다. `--append-system-prompt`는 시스템 프롬프트에 내용을 추가합니다.

### 출력 스타일 vs. 에이전트

출력 스타일은 주 에이전트 루프에 직접 영향을 미치며 시스템 프롬프트에만 영향을 줍니다. 에이전트는 특정 작업을 처리하기 위해 호출되며 사용할 모델, 사용 가능한 도구, 에이전트 사용 시기에 대한 컨텍스트와 같은 추가 설정을 포함할 수 있습니다.

### 출력 스타일 vs. 스킬

출력 스타일은 Claude가 응답하는 방식 (형식, 톤, 구조)을 수정하며 선택되면 항상 활성화됩니다. 스킬은 `/skill-name`으로 호출하거나 Claude가 관련될 때 자동으로 로드하는 작업별 프롬프트입니다. 일관된 형식 선호도에는 출력 스타일을 사용하고, 재사용 가능한 워크플로우와 작업에는 스킬을 사용하세요.
