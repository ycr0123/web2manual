# Claude Code 작동 원리

> 에이전트 루프, 내장 도구, Claude Code가 프로젝트와 상호작용하는 방식을 이해합니다.

Claude Code는 터미널에서 실행되는 에이전트형 어시스턴트입니다. 코딩에 탁월하지만, 커맨드라인에서 할 수 있는 모든 작업을 도울 수 있습니다: 문서 작성, 빌드 실행, 파일 검색, 주제 조사 등.

## 에이전트 루프

Claude에게 작업을 주면 세 단계를 거쳐 진행합니다: **컨텍스트 수집**, **행동 수행**, **결과 검증**. 이 단계들은 서로 혼합됩니다. Claude는 코드를 이해하기 위한 파일 검색, 변경을 위한 편집, 작업 확인을 위한 테스트 실행 등 전 과정에서 도구를 사용합니다.

루프는 요청에 따라 적응합니다. 코드베이스에 대한 질문은 컨텍스트 수집만 필요할 수 있습니다. 버그 수정은 세 단계를 반복적으로 순환합니다. 리팩토링은 광범위한 검증이 필요할 수 있습니다. Claude는 이전 단계에서 배운 내용을 바탕으로 각 단계에서 필요한 것을 결정하며, 수십 가지 행동을 연결하고 과정에서 방향을 수정합니다.

이 루프에는 사용자도 포함됩니다. 언제든지 개입하여 Claude를 다른 방향으로 이끌거나, 추가 컨텍스트를 제공하거나, 다른 접근 방식을 시도하도록 요청할 수 있습니다. Claude는 자율적으로 작동하지만 사용자 입력에 반응합니다.

에이전트 루프는 두 가지 구성 요소로 구동됩니다: 추론하는 **모델**과 행동하는 **도구**. Claude Code는 Claude 주변의 **에이전트 하네스** 역할을 합니다: 도구, 컨텍스트 관리, 실행 환경을 제공하여 언어 모델을 능력 있는 코딩 에이전트로 만들어 줍니다.

### 모델

Claude Code는 Claude 모델을 사용하여 코드를 이해하고 작업에 대해 추론합니다. Claude는 모든 언어의 코드를 읽고, 구성 요소가 어떻게 연결되는지 이해하며, 목표를 달성하기 위해 무엇을 변경해야 하는지 파악합니다. 복잡한 작업의 경우 작업을 단계로 나누고, 실행하며, 배운 내용을 바탕으로 조정합니다.

다양한 트레이드오프를 가진 여러 모델을 사용할 수 있습니다. Sonnet은 대부분의 코딩 작업을 잘 처리합니다. Opus는 복잡한 아키텍처 결정에 더 강력한 추론을 제공합니다. 세션 중 `/model`로 전환하거나 `claude --model <name>`으로 시작하세요.

### 도구

도구는 Claude Code를 에이전트형으로 만드는 것입니다. 도구 없이는 Claude가 텍스트로만 응답할 수 있습니다. 도구가 있으면 Claude는 행동할 수 있습니다: 코드 읽기, 파일 편집, 명령 실행, 웹 검색, 외부 서비스와 상호작용. 각 도구 사용은 루프에 피드백되는 정보를 반환하여 Claude의 다음 결정을 알려줍니다.

내장 도구는 일반적으로 네 가지 범주로 나뉩니다:

| 범주                  | Claude가 할 수 있는 것                                                                                              |
| --------------------- | --------------------------------------------------------------------------------------------------------------- |
| **파일 작업**         | 파일 읽기, 코드 편집, 새 파일 생성, 이름 변경 및 재구성                                                          |
| **검색**              | 패턴으로 파일 찾기, 정규식으로 콘텐츠 검색, 코드베이스 탐색                                                      |
| **실행**              | 셸 명령 실행, 서버 시작, 테스트 실행, git 사용                                                                   |
| **웹**                | 웹 검색, 문서 가져오기, 오류 메시지 조회                                                                         |
| **코드 인텔리전스**   | 편집 후 타입 오류와 경고 확인, 정의로 이동, 참조 찾기 (코드 인텔리전스 플러그인 필요)                            |

Claude는 프롬프트와 진행 중에 배운 내용을 바탕으로 사용할 도구를 선택합니다. "실패하는 테스트를 수정해줘"라고 하면 Claude는 다음을 수행할 수 있습니다:

1. 테스트 스위트를 실행하여 실패 항목 확인
2. 오류 출력 읽기
3. 관련 소스 파일 검색
4. 코드를 이해하기 위해 파일 읽기
5. 문제를 수정하기 위해 파일 편집
6. 검증을 위해 테스트 재실행

각 도구 사용은 다음 단계에 정보를 제공하는 새로운 정보를 Claude에게 줍니다. 이것이 에이전트 루프의 작동 방식입니다.

**기본 기능 확장:** 내장 도구는 기반입니다. 스킬로 Claude가 알고 있는 것을 확장하고, MCP로 외부 서비스에 연결하며, 훅으로 워크플로우를 자동화하고, 서브에이전트에 작업을 위임할 수 있습니다.

## Claude가 접근할 수 있는 것

디렉토리에서 `claude`를 실행하면 Claude Code는 다음에 접근할 수 있습니다:

- **프로젝트.** 디렉토리와 하위 디렉토리의 파일, 허가를 받으면 다른 위치의 파일도 접근 가능합니다.
- **터미널.** 실행할 수 있는 모든 명령: 빌드 도구, git, 패키지 관리자, 시스템 유틸리티, 스크립트.
- **Git 상태.** 현재 브랜치, 커밋되지 않은 변경사항, 최근 커밋 기록.
- **CLAUDE.md.** 매 세션마다 Claude가 알아야 할 프로젝트별 지침, 규칙, 컨텍스트를 저장하는 마크다운 파일.
- **구성한 확장 기능.** 외부 서비스를 위한 MCP 서버, 워크플로우를 위한 스킬, 위임된 작업을 위한 서브에이전트, 브라우저 상호작용을 위한 Chrome의 Claude.

Claude가 프로젝트 전체를 파악하므로 프로젝트 전반에 걸쳐 작업할 수 있습니다. Claude에게 "인증 버그를 수정해줘"라고 요청하면 관련 파일을 검색하고, 컨텍스트를 이해하기 위해 여러 파일을 읽고, 여러 파일에 걸쳐 조율된 편집을 수행하며, 수정을 검증하기 위해 테스트를 실행하고, 요청 시 변경사항을 커밋합니다.

## 세션으로 작업하기

Claude Code는 작업하는 동안 대화를 로컬에 저장합니다. 각 메시지, 도구 사용, 결과가 저장되어 세션을 되감고, 재개하거나, 분기할 수 있습니다. Claude가 코드 변경을 수행하기 전에는 필요시 되돌릴 수 있도록 영향받는 파일의 스냅샷도 저장합니다.

**세션은 독립적입니다.** 새 세션은 이전 세션의 대화 기록 없이 새로운 컨텍스트 창으로 시작됩니다. Claude는 자동 메모리를 사용하여 세션 전반에 걸쳐 학습 내용을 유지할 수 있으며, CLAUDE.md에 자체 지속적인 지침을 추가할 수 있습니다.

### 브랜치 간 작업

각 Claude Code 대화는 현재 디렉토리에 연결된 세션입니다. 재개할 때 해당 디렉토리의 세션만 볼 수 있습니다.

Claude는 현재 브랜치의 파일을 봅니다. 브랜치를 전환하면 Claude는 새 브랜치의 파일을 보지만, 대화 기록은 동일하게 유지됩니다. 전환 후에도 Claude는 논의한 내용을 기억합니다.

세션이 디렉토리에 연결되어 있으므로, 개별 브랜치를 위한 별도 디렉토리를 생성하는 git worktree를 사용하여 병렬 Claude 세션을 실행할 수 있습니다.

### 세션 재개 또는 분기

`claude --continue` 또는 `claude --resume`으로 세션을 재개하면 동일한 세션 ID를 사용하여 중단한 곳에서 계속합니다. 새 메시지가 기존 대화에 추가됩니다. 전체 대화 기록이 복원되지만, 세션 범위의 권한은 복원되지 않습니다.

원본 세션에 영향을 주지 않고 다른 접근 방식을 시도하려면 `--fork-session` 플래그를 사용하세요:

```bash
claude --continue --fork-session
```

이렇게 하면 해당 시점까지의 대화 기록을 보존하면서 새 세션 ID가 생성됩니다. 원본 세션은 변경되지 않습니다.

**여러 터미널에서 동일한 세션**: 여러 터미널에서 동일한 세션을 재개하면 두 터미널 모두 동일한 세션 파일에 씁니다. 두 곳의 메시지가 혼합됩니다. 동일한 시작점에서 병렬 작업을 위해 `--fork-session`을 사용하여 각 터미널에 자체 새 세션을 부여하세요.

### 컨텍스트 창

Claude의 컨텍스트 창에는 대화 기록, 파일 내용, 명령 출력, CLAUDE.md, 로드된 스킬, 시스템 지침이 저장됩니다. 작업하면서 컨텍스트가 채워집니다. Claude는 자동으로 압축하지만, 대화 초반의 지침이 손실될 수 있습니다. 지속적인 규칙은 CLAUDE.md에 넣고, `/context`를 실행하여 무엇이 공간을 차지하는지 확인하세요.

#### 컨텍스트가 가득 찼을 때

Claude Code는 한도에 가까워지면 자동으로 컨텍스트를 관리합니다. 먼저 오래된 도구 출력을 지우고, 필요하면 대화를 요약합니다. 요청과 주요 코드 스니펫은 보존됩니다. 대화 초반의 상세 지침은 손실될 수 있습니다. 대화 기록에 의존하지 말고 지속적인 규칙은 CLAUDE.md에 넣으세요.

압축 중 보존할 항목을 제어하려면 CLAUDE.md에 "Compact Instructions" 섹션을 추가하거나 `/compact`를 특정 초점과 함께 실행하세요 (예: `/compact focus on the API changes`).

`/context`를 실행하여 무엇이 공간을 차지하는지 확인하세요. MCP 서버는 모든 요청에 도구 정의를 추가하므로, 서버 몇 개만으로도 작업 시작 전에 상당한 컨텍스트를 소비할 수 있습니다. `/mcp`를 실행하여 서버별 비용을 확인하세요.

#### 스킬과 서브에이전트로 컨텍스트 관리

압축 외에도 컨텍스트에 로드되는 것을 제어하는 다른 기능을 사용할 수 있습니다.

**스킬**은 필요 시 로드됩니다. Claude는 세션 시작 시 스킬 설명을 보지만, 전체 내용은 스킬이 사용될 때만 로드됩니다. 수동으로 호출하는 스킬의 경우 `disable-model-invocation: true`를 설정하여 필요할 때까지 설명이 컨텍스트에 나타나지 않도록 하세요.

**서브에이전트**는 메인 대화와 완전히 별개인 자체 새 컨텍스트를 갖습니다. 서브에이전트의 작업은 컨텍스트를 부풀리지 않습니다. 완료되면 요약을 반환합니다. 이 격리가 서브에이전트가 긴 세션에 도움이 되는 이유입니다.

## 체크포인트와 권한으로 안전하게 유지하기

Claude에는 두 가지 안전 메커니즘이 있습니다: 체크포인트는 파일 변경을 되돌릴 수 있게 하고, 권한은 Claude가 묻지 않고 할 수 있는 것을 제어합니다.

### 체크포인트로 변경사항 되돌리기

**모든 파일 편집은 되돌릴 수 있습니다.** Claude가 파일을 편집하기 전에 현재 내용의 스냅샷을 저장합니다. 문제가 발생하면 `Esc`를 두 번 눌러 이전 상태로 되돌리거나 Claude에게 실행 취소를 요청하세요.

체크포인트는 세션에 로컬이며, git과 별도입니다. 파일 변경사항만 다룹니다. 원격 시스템 (데이터베이스, API, 배포)에 영향을 미치는 작업은 체크포인트할 수 없으므로, Claude가 외부 부작용이 있는 명령을 실행하기 전에 확인하는 이유입니다.

### Claude가 할 수 있는 것 제어하기

`Shift+Tab`을 눌러 권한 모드를 순환합니다:

- **기본**: Claude가 파일 편집 및 셸 명령 전에 확인 요청
- **편집 자동 수락**: Claude가 확인 없이 파일 편집, 명령에 대해서는 여전히 확인 요청
- **Plan 모드**: Claude가 읽기 전용 도구만 사용하여 실행 전에 승인할 수 있는 계획 작성
- **Delegate 모드**: Claude가 직접 구현 없이 에이전트 팀원을 통해서만 작업 조율. 에이전트 팀이 활성화된 경우에만 사용 가능.

`.claude/settings.json`에서 특정 명령을 허용하여 Claude가 매번 묻지 않도록 할 수도 있습니다. `npm test` 또는 `git status`와 같은 신뢰할 수 있는 명령에 유용합니다.

## Claude Code로 효과적으로 작업하기

이 팁들은 Claude Code에서 더 나은 결과를 얻는 데 도움이 됩니다.

### Claude Code에게 도움 요청하기

Claude Code는 사용 방법을 가르쳐 줄 수 있습니다. "훅을 어떻게 설정하나요?" 또는 "CLAUDE.md를 구성하는 가장 좋은 방법은?"과 같은 질문을 하면 Claude가 설명해 줍니다.

내장 명령어도 설정 과정을 안내해 줍니다:

- `/init`은 프로젝트를 위한 CLAUDE.md 생성 과정을 안내합니다
- `/agents`는 커스텀 서브에이전트 구성을 도와줍니다
- `/doctor`는 설치의 일반적인 문제를 진단합니다

### 대화입니다

Claude Code는 대화형입니다. 완벽한 프롬프트가 필요하지 않습니다. 원하는 것으로 시작한 다음 세밀하게 조정하세요:

```
> Fix the login bug

[Claude investigates, tries something]

> That's not quite right. The issue is in the session handling.

[Claude adjusts approach]
```

첫 번째 시도가 맞지 않을 때 처음부터 다시 시작하지 않습니다. 반복합니다.

#### 중단하고 방향 전환하기

언제든지 Claude를 중단할 수 있습니다. 잘못된 방향으로 가고 있다면 수정 사항을 입력하고 Enter를 누르세요. Claude는 하던 일을 멈추고 입력에 따라 접근 방식을 조정합니다.

### 처음부터 구체적으로 하기

초기 프롬프트가 정확할수록 수정이 적게 필요합니다. 특정 파일을 참조하고, 제약 조건을 언급하며, 예제 패턴을 가리키세요.

```
> The checkout flow is broken for users with expired cards.
> Check src/payments/ for the issue, especially token refresh.
> Write a failing test first, then fix it.
```

### Claude에게 검증할 것 주기

Claude는 자신의 작업을 확인할 수 있을 때 더 잘 수행합니다. 테스트 케이스를 포함하거나, 예상 UI의 스크린샷을 붙여넣거나, 원하는 출력을 정의하세요.

```
> Implement validateEmail. Test cases: 'user@example.com' → true,
> 'invalid' → false, 'user@.com' → false. Run the tests after.
```

### 구현 전에 탐색하기

복잡한 문제의 경우 연구와 코딩을 분리하세요. 계획 모드 (`Shift+Tab` 두 번)를 사용하여 먼저 코드베이스를 분석하세요:

```
> Read src/auth/ and understand how we handle sessions.
> Then create a plan for adding OAuth support.
```

계획을 검토하고 대화를 통해 세밀하게 조정한 다음 Claude가 구현하도록 하세요.

### 지시하지 말고 위임하기

유능한 동료에게 위임하는 것처럼 생각하세요. 컨텍스트와 방향을 제공하고 Claude가 세부 사항을 파악하도록 신뢰하세요:

```
> The checkout flow is broken for users with expired cards.
> The relevant code is in src/payments/. Can you investigate and fix it?
```

어떤 파일을 읽거나 어떤 명령을 실행할지 지정할 필요가 없습니다. Claude가 알아서 처리합니다.
