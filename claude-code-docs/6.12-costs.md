# 비용 관리

> 토큰 사용량을 추적하고, 팀 지출 한도를 설정하고, 컨텍스트 관리 및 모델 선택으로 Claude Code 비용을 절감합니다.

Claude Code는 각 상호 작용마다 토큰을 사용합니다. 비용은 코드베이스 크기, 쿼리 복잡성 및 대화 길이에 따라 달라집니다. 평균 비용은 개발자당 하루 $6이며, 사용자의 90%가 하루 비용이 $12 미만입니다.

팀 사용의 경우 Claude Code는 API 토큰 소비를 기준으로 요금을 청구합니다. 평균적으로 Sonnet 4.6을 사용하는 Claude Code는 개발자당 월 ~$100-200이 소요되지만, 사용자가 실행하는 인스턴스 수와 자동화에 사용하는지 여부에 따라 큰 편차가 있습니다.

이 페이지에서는 [비용 추적](#track-your-costs), [팀 비용 관리](#managing-costs-for-teams), [토큰 사용 줄이기](#reduce-token-usage)에 대해 다룹니다.

## 비용 추적

### `/cost` 명령 사용

> `/cost` 명령은 API 토큰 사용량을 표시하며 API 사용자를 위한 것입니다. Claude Max 및 Pro 구독자는 구독에 사용량이 포함되어 있으므로 `/cost` 데이터는 청구 목적으로 관련이 없습니다. 구독자는 `/stats`를 사용하여 사용 패턴을 확인할 수 있습니다.

`/cost` 명령은 현재 세션에 대한 자세한 토큰 사용량 통계를 제공합니다.

```
Total cost:            $0.55
Total duration (API):  6m 19.7s
Total duration (wall): 6h 33m 10.2s
Total code changes:    0 lines added, 0 lines removed
```

## 팀 비용 관리

Claude API를 사용할 때 전체 Claude Code 워크스페이스 지출에 대한 [워크스페이스 지출 한도를 설정](https://platform.claude.com/docs/en/build-with-claude/workspaces#workspace-limits)할 수 있습니다. 관리자는 Console에서 [비용 및 사용 보고서를 볼 수 있습니다](https://platform.claude.com/docs/en/build-with-claude/workspaces#usage-and-cost-tracking).

> Claude Console 계정으로 Claude Code를 처음 인증하면 "Claude Code"라는 워크스페이스가 자동으로 생성됩니다. 이 워크스페이스는 조직의 모든 Claude Code 사용에 대한 중앙 집중식 비용 추적 및 관리를 제공합니다. 이 워크스페이스의 API 키는 생성할 수 없으며, Claude Code 인증 및 사용에만 사용됩니다.

Bedrock, Vertex, Foundry에서는 Claude Code가 클라우드에서 지표를 전송하지 않습니다. 비용 지표를 얻으려면 여러 대기업에서 [키별 지출을 추적하는](https://docs.litellm.ai/docs/proxy/virtual_keys#tracking-spend) 오픈 소스 도구인 [LiteLLM](/en/llm-gateway#litellm-configuration)을 사용한다고 보고했습니다. 이 프로젝트는 Anthropic과 무관하며 보안 감사를 받지 않았습니다.

### 속도 제한 권장 사항

팀용 Claude Code를 설정할 때 조직 규모에 따라 다음 사용자당 TPM(분당 토큰) 및 RPM(분당 요청) 권장 사항을 고려하세요.

| 팀 규모 | 사용자당 TPM | 사용자당 RPM |
|-----------|--------------|--------------|
| 1-5명 | 200k-300k | 5-7 |
| 5-20명 | 100k-150k | 2.5-3.5 |
| 20-50명 | 50k-75k | 1.25-1.75 |
| 50-100명 | 25k-35k | 0.62-0.87 |
| 100-500명 | 15k-20k | 0.37-0.47 |
| 500명+ | 10k-15k | 0.25-0.35 |

예를 들어 사용자가 200명인 경우 각 사용자에게 20k TPM을 요청하거나 총 4백만 TPM (200*20,000 = 4백만)을 요청할 수 있습니다.

사용자당 TPM은 대규모 조직에서 더 적은 수의 사용자가 Claude Code를 동시에 사용하는 경향이 있으므로 팀 규모가 커짐에 따라 감소합니다. 이러한 속도 제한은 개별 사용자가 아닌 조직 수준에서 적용되므로 다른 사용자가 활발하게 서비스를 사용하지 않을 때 개별 사용자가 일시적으로 계산된 몫보다 더 많이 사용할 수 있습니다.

> 대규모 그룹을 대상으로 한 실시간 교육 세션과 같이 동시 사용이 비정상적으로 높은 시나리오가 예상되는 경우 사용자당 더 높은 TPM 할당이 필요할 수 있습니다.

### 에이전트 팀 토큰 비용

[에이전트 팀](/en/agent-teams)은 각각 고유한 컨텍스트 창이 있는 여러 Claude Code 인스턴스를 생성합니다. 토큰 사용량은 활성 팀원 수와 각 팀원이 실행하는 시간에 따라 확장됩니다.

에이전트 팀 비용을 관리 가능하게 유지하려면:

* 팀원에게 Sonnet을 사용하세요. 조정 작업에서 능력과 비용의 균형을 맞춥니다.
* 팀을 작게 유지하세요. 각 팀원은 자체 컨텍스트 창을 실행하므로 토큰 사용량은 팀 규모에 대략 비례합니다.
* 스폰 프롬프트를 집중적으로 유지하세요. 팀원은 CLAUDE.md, MCP 서버, 스킬을 자동으로 로드하지만 스폰 프롬프트의 모든 내용은 처음부터 컨텍스트에 추가됩니다.
* 작업이 완료되면 팀을 정리하세요. 활성 팀원은 유휴 상태에서도 계속 토큰을 사용합니다.
* 에이전트 팀은 기본적으로 비활성화되어 있습니다. 활성화하려면 [settings.json](/en/settings) 또는 환경에서 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`을 설정하세요. [에이전트 팀 활성화](/en/agent-teams#enable-agent-teams)를 참조하세요.

## 토큰 사용 줄이기

토큰 비용은 컨텍스트 크기에 따라 달라집니다: Claude가 처리하는 컨텍스트가 많을수록 더 많은 토큰을 사용합니다. Claude Code는 프롬프트 캐싱(시스템 프롬프트와 같은 반복 콘텐츠에 대한 비용 절감)과 자동 압축(컨텍스트 제한에 근접할 때 대화 기록 요약)을 통해 비용을 자동으로 최적화합니다.

다음 전략은 컨텍스트를 작게 유지하고 메시지당 비용을 줄이는 데 도움이 됩니다.

### 능동적인 컨텍스트 관리

`/cost`를 사용하여 현재 토큰 사용량을 확인하거나, [상태 표시줄을 구성](/en/statusline#context-window-usage)하여 지속적으로 표시하세요.

* **작업 간에 지우기**: 관련 없는 작업으로 전환할 때 `/clear`를 사용하여 새로 시작하세요. 오래된 컨텍스트는 이후의 모든 메시지에서 토큰을 낭비합니다. 지우기 전에 `/rename`을 사용하여 세션을 나중에 쉽게 찾을 수 있도록 하고, `/resume`을 사용하여 돌아올 수 있습니다.
* **사용자 지정 압축 지침 추가**: `/compact Focus on code samples and API usage`는 요약 중에 보존할 내용을 Claude에게 알려줍니다.

CLAUDE.md에서 압축 동작을 사용자 지정할 수도 있습니다.

```markdown
# Compact instructions

When you are using compact, please focus on test output and code changes
```

### 올바른 모델 선택

Sonnet은 대부분의 코딩 작업을 잘 처리하며 Opus보다 비용이 적습니다. 복잡한 아키텍처 결정이나 다단계 추론을 위해 Opus를 예약하세요. `/model`을 사용하여 세션 중에 모델을 전환하거나 `/config`에서 기본값을 설정하세요. 간단한 하위 에이전트 작업의 경우 [하위 에이전트 구성](/en/sub-agents#choose-a-model)에서 `model: haiku`를 지정하세요.

### MCP 서버 오버헤드 줄이기

각 MCP 서버는 유휴 상태에서도 컨텍스트에 도구 정의를 추가합니다. `/context`를 실행하여 공간을 사용하는 항목을 확인하세요.

* **가능한 경우 CLI 도구 선호**: `gh`, `aws`, `gcloud`, `sentry-cli`와 같은 도구는 지속적인 도구 정의를 추가하지 않으므로 MCP 서버보다 컨텍스트 효율이 높습니다. Claude는 오버헤드 없이 CLI 명령을 직접 실행할 수 있습니다.
* **사용하지 않는 서버 비활성화**: `/mcp`를 실행하여 구성된 서버를 확인하고 활발하게 사용하지 않는 서버를 비활성화하세요.
* **도구 검색이 자동**: MCP 도구 설명이 컨텍스트 창의 10%를 초과하면 Claude Code는 자동으로 도구를 지연하고 [도구 검색](/en/mcp#scale-with-mcp-tool-search)을 통해 온디맨드로 로드합니다. 지연된 도구는 실제로 사용될 때만 컨텍스트에 들어가므로 임계값이 낮을수록 유휴 도구 정의가 공간을 차지하는 일이 줄어듭니다. `ENABLE_TOOL_SEARCH=auto:<N>`으로 낮은 임계값을 설정하세요 (예: `auto:5`는 도구가 컨텍스트 창의 5%를 초과하면 트리거).

### 형식화된 언어용 코드 인텔리전스 플러그인 설치

[코드 인텔리전스 플러그인](/en/discover-plugins#code-intelligence)은 Claude에게 텍스트 기반 검색 대신 정확한 심볼 탐색을 제공하여 낯선 코드를 탐색할 때 불필요한 파일 읽기를 줄입니다. "정의로 이동" 호출 하나로 grep 후 여러 후보 파일을 읽어야 하는 작업을 대체할 수 있습니다. 설치된 언어 서버는 편집 후 유형 오류를 자동으로 보고하므로 Claude가 컴파일러를 실행하지 않고도 실수를 잡아낼 수 있습니다.

### 훅 및 스킬에 처리 위임

사용자 지정 [훅](/en/hooks)은 Claude가 보기 전에 데이터를 전처리할 수 있습니다. Claude가 10,000줄의 로그 파일을 읽어 오류를 찾는 대신 훅이 `ERROR`를 grep하고 일치하는 줄만 반환하여 컨텍스트를 수만 개의 토큰에서 수백 개로 줄일 수 있습니다.

[스킬](/en/skills)은 Claude에게 도메인 지식을 제공하여 탐색할 필요가 없도록 합니다. 예를 들어 "codebase-overview" 스킬은 프로젝트 아키텍처, 주요 디렉토리, 명명 규칙을 설명할 수 있습니다. Claude가 스킬을 호출하면 구조를 이해하기 위해 여러 파일을 읽는 데 토큰을 낭비하는 대신 즉시 이 컨텍스트를 얻습니다.

예를 들어 이 PreToolUse 훅은 테스트 출력을 필터링하여 실패만 표시합니다.

settings.json에서 모든 Bash 명령 전에 훅을 실행하도록 추가하세요.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/filter-test-output.sh"
          }
        ]
      }
    ]
  }
}
```

훅은 명령이 테스트 러너인지 확인하고 실패만 표시하도록 수정하는 스크립트를 호출합니다.

```bash
#!/bin/bash
input=$(cat)
cmd=$(echo "$input" | jq -r '.tool_input.command')

# 테스트를 실행하는 경우 실패만 표시하도록 필터링
if [[ "$cmd" =~ ^(npm test|pytest|go test) ]]; then
  filtered_cmd="$cmd 2>&1 | grep -A 5 -E '(FAIL|ERROR|error:)' | head -100"
  echo "{\"hookSpecificOutput\":{\"hookEventName\":\"PreToolUse\",\"permissionDecision\":\"allow\",\"updatedInput\":{\"command\":\"$filtered_cmd\"}}}"
else
  echo "{}"
fi
```

### CLAUDE.md의 지침을 스킬로 이동

[CLAUDE.md](/en/memory) 파일은 세션 시작 시 컨텍스트에 로드됩니다. PR 검토 또는 데이터베이스 마이그레이션과 같은 특정 워크플로우에 대한 자세한 지침이 포함된 경우 관련 없는 작업을 수행할 때도 해당 토큰이 존재합니다. [스킬](/en/skills)은 호출될 때만 온디맨드로 로드되므로 특수화된 지침을 스킬로 이동하면 기본 컨텍스트가 작아집니다. 필수 항목만 포함하여 CLAUDE.md를 ~500줄 이하로 유지하세요.

### 확장 사고 조정

확장 사고는 복잡한 계획 및 추론 작업에서 성능을 크게 향상시키므로 기본적으로 31,999 토큰 예산으로 활성화되어 있습니다. 그러나 사고 토큰은 출력 토큰으로 청구되므로 심층 추론이 필요하지 않은 간단한 작업의 경우 Opus 4.6의 경우 `/model`에서 [노력 수준을 낮추거나](/en/model-config#adjust-effort-level), `/config`에서 사고를 비활성화하거나, 예산을 낮춰 (`MAX_THINKING_TOKENS=8000`) 비용을 줄일 수 있습니다.

### 하위 에이전트에 장황한 작업 위임

테스트 실행, 문서 가져오기 또는 로그 파일 처리는 상당한 컨텍스트를 소비할 수 있습니다. 이를 [하위 에이전트](/en/sub-agents#isolate-high-volume-operations)에 위임하면 장황한 출력이 하위 에이전트의 컨텍스트에 머물고 요약만 주요 대화로 돌아옵니다.

### 에이전트 팀 비용 관리

에이전트 팀은 팀원이 계획 모드로 실행될 때 표준 세션보다 약 7배 더 많은 토큰을 사용합니다. 각 팀원이 자체 컨텍스트 창을 유지하고 별도의 Claude 인스턴스로 실행되기 때문입니다. 팀원당 토큰 사용을 제한하려면 팀 작업을 작고 독립적으로 유지하세요. 자세한 내용은 [에이전트 팀](/en/agent-teams)을 참조하세요.

### 구체적인 프롬프트 작성

"이 코드베이스를 개선하세요"와 같은 모호한 요청은 광범위한 스캔을 유발합니다. "auth.ts의 로그인 함수에 입력 유효성 검사를 추가하세요"와 같은 구체적인 요청은 Claude가 최소한의 파일 읽기로 효율적으로 작업할 수 있게 합니다.

### 복잡한 작업에서 효율적으로 작업

더 길거나 복잡한 작업의 경우 다음 습관이 잘못된 방향으로 인한 낭비된 토큰을 방지하는 데 도움이 됩니다.

* **복잡한 작업에 계획 모드 사용**: 구현 전에 Shift+Tab을 눌러 [계획 모드](/en/common-workflows#use-plan-mode-for-safe-code-analysis)로 진입하세요. Claude가 코드베이스를 탐색하고 승인을 위한 접근 방식을 제안하여 초기 방향이 잘못되었을 때의 비용이 많이 드는 재작업을 방지합니다.
* **조기에 궤도 수정**: Claude가 잘못된 방향으로 시작하면 즉시 Escape를 눌러 중지하세요. `/rewind`를 사용하거나 Escape를 두 번 탭하여 대화 및 코드를 이전 체크포인트로 복원하세요.
* **검증 목표 제공**: 프롬프트에 테스트 케이스를 포함하거나 스크린샷을 붙여넣거나 예상 출력을 정의하세요. Claude가 자체적으로 작업을 검증할 수 있으면 수정을 요청하기 전에 문제를 잡아냅니다.
* **점진적으로 테스트**: 파일 하나를 작성하고 테스트한 다음 계속하세요. 이렇게 하면 수정 비용이 적을 때 조기에 문제를 잡아낼 수 있습니다.

## 백그라운드 토큰 사용

Claude Code는 유휴 상태에서도 일부 백그라운드 기능을 위해 토큰을 사용합니다.

* **대화 요약**: `claude --resume` 기능을 위해 이전 대화를 요약하는 백그라운드 작업
* **명령 처리**: `/cost`와 같은 일부 명령이 상태를 확인하기 위해 요청을 생성할 수 있음

이러한 백그라운드 프로세스는 활성 상호 작용 없이도 소량의 토큰을 사용합니다 (일반적으로 세션당 $0.04 미만).

## Claude Code 동작 변경 이해

Claude Code는 비용 보고를 포함하여 기능 작동 방식을 변경할 수 있는 업데이트를 정기적으로 받습니다. `claude --version`을 실행하여 현재 버전을 확인하세요. 구체적인 청구 질문은 [Console 계정](https://platform.claude.com/login)을 통해 Anthropic 지원팀에 문의하세요. 팀 배포의 경우 더 광범위한 배포 전에 사용 패턴을 설정하기 위해 소규모 파일럿 그룹으로 시작하세요.
